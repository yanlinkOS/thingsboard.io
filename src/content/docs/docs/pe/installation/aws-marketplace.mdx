This guide walks you through installing ThingsBoard Professional Edition from AWS Marketplace using the <a href="https://docs.aws.amazon.com/marketplace/latest/userguide/pricing.html#ami-pricing-models" target="_blank">BYOL (Bring Your Own License)</a> model. You get the license directly from ThingsBoard, Inc, and purchase the server instances and infrastructure from AWS.

## Prerequisites

- Active <a href="https://aws.amazon.com/" target="_blank">Amazon AWS</a> account

## Step 1. Subscribe to ThingsBoard PE BYOL

Navigate to the <a href="https://aws.amazon.com/marketplace/pp/B07V8S6JLG" target="_blank">ThingsBoard Professional Edition BYOL</a> product page on AWS Marketplace and click **View purchase options**.

Before completing your subscription, review:

- **Product details** — BYOL model means a valid ThingsBoard license must be purchased separately to activate the software.
- **Pricing details** — The software subscription cost is $0.00. AWS infrastructure usage charges apply separately.
- **Terms and conditions** — By subscribing, you agree to the **End User License Agreement (EULA)** and the **AWS Customer Agreement**.

<ImageGallery images={[
  { src: '/src/assets/images/docs/installation/aws-marketplace-pe/BYOL-subscribe.png', alt: 'AWS Marketplace BYOL subscription page', caption: 'Click Subscribe after reviewing the terms.' },
]} />

Click **Subscribe** to proceed.

## Step 2. Launch ThingsBoard PE instance

After subscribing, click **Launch your software**.

### 2.1 Setup configuration

Configure the deployment options:

- **Service** — Select **Amazon EC2** for on-demand scalability and full control over your instance.
- **Launch method** — Select **Launch from EC2 Console**.
- **Region** — Choose your preferred AWS region.

<ImageGallery images={[
  { src: '/src/assets/images/docs/installation/aws-marketplace-pe/service-setup.png', alt: 'Service setup configuration', caption: 'Select EC2 service, launch method, and region.' },
]} />

Click **Launch from EC2** to continue.

### 2.2 Name and AMI

Assign a name to your instance and verify the correct **ThingsBoard Professional Edition (BYOL) AMI** is selected.

<ImageGallery images={[
  { src: '/src/assets/images/docs/installation/aws-marketplace-pe/setup-configuration.png', alt: 'Name and AMI configuration', caption: 'Verify the AMI and assign an instance name.' },
]} />

### 2.3 Instance type and key pair

Select an EC2 instance type and key pair.

<Aside type="note">
ThingsBoard requires an instance with at least **4 GB of RAM**. We recommend **8 GB of RAM** and **2 vCPUs** for optimal performance.
</Aside>

<ImageGallery images={[
  { src: '/src/assets/images/docs/installation/aws-marketplace-pe/instance-type-and-key-pair-configuration.png', alt: 'Instance type and key pair configuration', caption: 'Select instance type and create or choose a key pair.' },
]} />

Select an existing **Key Pair** or create a new one. Make sure you have access to the key file — you will need it to connect via SSH.

### 2.4 Configure security group

We recommend using the **Create security group** option. If you use an existing security group, ensure it allows inbound traffic on these ports:

| Port | Protocol | Description |
|---|---|---|
| 8080 | TCP | HTTP access to ThingsBoard web interface |
| 1883 | TCP | MQTT protocol |
| 8883 | TCP | MQTT over SSL |
| 7070 | TCP | Edge RPC service |
| 9090 | TCP | Remote Integration service |
| 5683–5688 | UDP | CoAP and LwM2M protocols |

<ImageGallery images={[
  { src: '/src/assets/images/docs/installation/aws-marketplace-pe/network-settings-configuration.png', alt: 'Security group configuration', caption: 'Configure inbound rules for ThingsBoard ports.' },
]} />

### 2.5 Storage configuration

Specify storage options for the instance. The default configuration uses 20 GiB of gp2 storage.

<ImageGallery images={[
  { src: '/src/assets/images/docs/installation/aws-marketplace-pe/storage-configuration.png', alt: 'Storage configuration', caption: 'Default 20 GiB gp2 storage.' },
]} />

### 2.6 Launch instance

Review the settings in the **Summary** panel and click **Launch instance**.

<ImageGallery images={[
  { src: '/src/assets/images/docs/installation/aws-marketplace-pe/summary-launch.png', alt: 'Launch summary', caption: 'Review and launch the instance.' },
]} />

### 2.7 Obtain your Public IP and Instance ID

Wait until the instance state changes to **Running** and all status checks pass. Go to the **EC2 Instances** page and click the **Instance ID** to view the **Instance Summary**.

<ImageGallery images={[
  { src: '/src/assets/images/docs/installation/aws-marketplace-pe/instanceId.png', alt: 'EC2 instances list', caption: 'Click the Instance ID to view details.' },
  { src: '/src/assets/images/docs/installation/aws-marketplace-pe/instance-summary.png', alt: 'Instance summary', caption: 'Save the Public DNS name and Instance ID.' },
]} />

Save the **Public DNS** name (e.g. `ec2-54-163-50-21.compute-1.amazonaws.com`) and the **Instance ID** (e.g. `i-0e614977a8e0f4f3b`). You will need them in the next steps.

## Step 3. Obtain your license secret

Register on the <a href="https://license.thingsboard.io/signup" target="_blank">ThingsBoard License Portal</a> to get your license. See <a href="https://www.youtube.com/watch?v=dK-QDFGxWek" target="_blank">How to get a pay-as-you-go subscription</a> or <a href="https://www.youtube.com/watch?v=GPe0lHolWek" target="_blank">How to get a perpetual license</a> for details.

Save your license secret — you will need it in the next step.

## Step 4. Configure your license secret

### 4.1 Connect to your instance via SSH

Use the official AWS guides:

- <a href="https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/connect-linux-inst-from-windows.html" target="_blank">Connect from Windows using PuTTY</a>
- <a href="https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/connect-linux-inst-ssh.html" target="_blank">Connect from Linux/Mac using SSH</a>

You will need the instance **Public DNS** name from Step 2.7 and the key file from Step 2.3.

### 4.2 Add your license secret

Open the configuration file:

```bash
sudo nano /etc/thingsboard/conf/thingsboard.conf
```

Locate the license configuration block and uncomment the export statement. Replace `YOUR_LICENSE_SECRET_HERE` with your actual license secret:

```bash
# License secret obtained from ThingsBoard License Portal (https://license.thingsboard.io)
# UNCOMMENT NEXT LINE AND PUT YOUR LICENSE SECRET:
export TB_LICENSE_SECRET=YOUR_LICENSE_SECRET_HERE
```

## Step 5. Launch ThingsBoard service

```bash
sudo service thingsboard start
```

<Aside type="note">
Allow up to 120 seconds for the Web UI to start on slower machines (1–2 CPUs or 1–2 GB RAM).
</Aside>

## Step 6. Connect to ThingsBoard UI

Open `http://PUBLIC_DNS_NAME/login` in your browser.

Use the following default credentials:

- **Login**: sysadmin@thingsboard.org
- **Password**: your **Instance ID** from Step 2.7 (e.g. `i-0e614977a8e0f4f3b`)

## Post-installation steps

<Aside type="caution">
By default, AWS Marketplace instances **do not assign a static public IP**. The public IP may change when the instance restarts. Assign an <a href="https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/working-with-eips.html" target="_blank">Elastic IP</a> for a persistent address.
</Aside>

### Configure HTTPS

<Steps>
1. Assign a valid DNS record to your instance's external IP address.
2. Connect to your instance via SSH (see Step 4.1).
3. Run the following commands, replacing the domain and email with your own:

   ```bash
   sudo certbot-certonly --domain smth.yourcompany.com --email support@yourcompany.com
   sudo haproxy-refresh
   ```
</Steps>

## FAQ

**What is the Total Cost of Ownership (TCO)?**

| Cost Component | Description |
|---|---|
| **ThingsBoard License fee** | See <a href="/pricing/" target="_blank">pricing</a> |
| **AWS EC2 instance price** | Compare at <a href="https://www.ec2instances.info/" target="_blank">ec2instances.info</a> |
| **Additional EC2 costs** | Network traffic, disk space, optional services (e.g. CloudWatch) |

**Example A: Maker Subscription (t3.medium, 20 GB disk)**

| Component | Monthly Cost (USD) |
|---|---|
| ThingsBoard PE Maker subscription | $10.00 |
| EC2 t3.medium (1 month) | $30.37 |
| 20 GB EBS volume | $2.00 |
| **Total** | **≈ $42.40** |

**Example B: Prototype Subscription (m5.large, 100 GB disk)**

| Component | Monthly Cost (USD) |
|---|---|
| ThingsBoard PE Prototype subscription | $99.00 |
| EC2 m5.large (1 month) | $70.08 |
| 100 GB EBS volume | $10.00 |
| **Total** | **≈ $179.00** |

**How do I upgrade my instance type?**

<Steps>
1. Stop the ThingsBoard service:

   ```bash
   sudo service thingsboard stop
   ```

2. Stop the EC2 instance from the AWS Console: **Instance state → Stop instance**.
3. Change the instance type: **Actions → Instance settings → Change instance type**.
4. Start the instance from the AWS Console.
</Steps>

**Where are the ThingsBoard logs?** Logs are stored in `/var/log/thingsboard`.
