    <p class="config-def-desc">Size of the thread pool that executes transport API callbacks (session registration, telemetry/attribute and RPC responses, entity update notifications, and the tenant profile fetch on a cache miss). Bounds how many such callbacks - including those that block on a backend round-trip - can run concurrently.</p>
  </div>
  <div class="config-def-item">
    <p class="config-def-meta"><code class="config-def-env">HTTP&#95;REQUEST&#95;TIMEOUT</code> · <span class="config-def-label">Default</span> <code>60000</code></p>

  <div class="config-def-item">
    <p class="config-def-meta"><code class="config-def-env">HTTP&#95;MAX&#95;REQUEST&#95;TIMEOUT</code> · <span class="config-def-label">Default</span> <code>300000</code></p>
    <p class="config-def-desc">HTTP maximum request processing timeout in milliseconds</p>
  </div>
  <div class="config-def-item">
    <p class="config-def-meta"><code class="config-def-env">HTTP&#95;TRANSPORT&#95;MAX&#95;PAYLOAD&#95;SIZE&#95;LIMIT&#95;CONFIGURATION</code> · <span class="config-def-label">Default</span> <code>/api/v1/&#42;/rpc/&#42;&#42;=65536;/api/v1/&#42;&#42;=52428800</code></p>
    <p class="config-def-desc">Semi-colon-separated list of urlPattern=maxPayloadSize pairs that define max http request size for specified url pattern. After first match all other will be skipped</p>
  </div>
  <div class="config-def-item">
    <p class="config-def-meta"><code class="config-def-env">TB&#95;TRANSPORT&#95;SESSIONS&#95;INACTIVITY&#95;TIMEOUT</code> · <span class="config-def-label">Default</span> <code>600000</code></p>
    <p class="config-def-desc">Session inactivity timeout is a global configuration parameter that defines how long the device transport session will be opened after the last message arrives from the device.  The parameter value is in milliseconds.  The last activity time of the device session is updated if the device sends any message, including keepalive messages  If there is no activity, the session will be closed, and all subscriptions will be deleted.  We recommend this parameter to be in sync with device inactivity timeout ("state.defaultInactivityTimeoutInSec" or DEFAULT&#95;INACTIVITY&#95;TIMEOUT) parameter  which is responsible for detection of the device connectivity status in the core service of the platform.  The value of the session inactivity timeout parameter should be greater or equal to the device inactivity timeout.  Note that the session inactivity timeout is set in milliseconds while device inactivity timeout is in seconds.</p>
  </div>
  <div class="config-def-item">
    <p class="config-def-meta"><code class="config-def-env">TB&#95;TRANSPORT&#95;SESSIONS&#95;REPORT&#95;TIMEOUT</code> · <span class="config-def-label">Default</span> <code>3000</code></p>
    <p class="config-def-desc">Interval of periodic check for expired sessions and report of the changes to session last activity time</p>
  </div>
  <div class="config-def-item">
    <p class="config-def-meta"><code class="config-def-env">JSON&#95;TYPE&#95;CAST&#95;ENABLED</code> · <span class="config-def-label">Default</span> <code>true</code></p>
    <p class="config-def-desc">Cast String data types to Numeric if possible when processing Telemetry/Attributes JSON</p>
  </div>
  <div class="config-def-item">
    <p class="config-def-meta"><code class="config-def-env">JSON&#95;MAX&#95;STRING&#95;VALUE&#95;LENGTH</code> · <span class="config-def-label">Default</span> <code>0</code></p>
    <p class="config-def-desc">Maximum allowed string value length when processing Telemetry/Attributes JSON (0 value disables string value length check)</p>
  </div>
  <div class="config-def-item">
    <p class="config-def-meta"><code class="config-def-env">TB&#95;TRANSPORT&#95;LOG&#95;ENABLED</code> · <span class="config-def-label">Default</span> <code>true</code></p>
    <p class="config-def-desc">Enable/Disable log of transport messages to telemetry. For example, logging of LwM2M registration update</p>
