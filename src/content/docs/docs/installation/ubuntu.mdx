This guide walks you through installing ThingsBoard Community Edition on Ubuntu Server.

## Prerequisites

- Ubuntu 22.04 LTS or 24.04 LTS
- A server with at least **4 GB of RAM** (8 GB recommended for hybrid database mode)

Verify that required font libraries are installed:

```bash
dpkg -l libharfbuzz0b fontconfig fonts-dejavu-core
```

Expected result: installed packages appear in the output with the status **ii**. If any are missing, install them:

```bash
sudo apt update && sudo apt install -y libharfbuzz0b fontconfig fonts-dejavu-core
```

## Step 3. Configure database

ThingsBoard uses PostgreSQL as the main database. For high-throughput scenarios (> 5K messages/sec), you can use a hybrid setup with Cassandra for time-series data.

<Tabs defaultTab="postgres" urlParam="database">
<TabItem label="PostgreSQL (recommended)" id="postgres">

PostgreSQL is recommended for most installations handling up to 5K messages per second.

Install PostgreSQL 16:

```bash
sudo apt install -y postgresql-common
sudo /usr/share/postgresql-common/pgdg/apt.postgresql.org.sh
sudo apt update
sudo apt -y install postgresql-16
sudo service postgresql start
```

Set a password for the `postgres` user:

```bash
sudo -u postgres psql -c "\password"
```

Enter and confirm the password when prompted. Connect to PostgreSQL and create the ThingsBoard database:

```bash
psql -U postgres -d postgres -h 127.0.0.1 -W
```

```sql
CREATE DATABASE thingsboard;
```

Press `Ctrl+D` twice to exit.

Configure ThingsBoard to use PostgreSQL. Edit the configuration file:

```bash
sudo nano /etc/thingsboard/conf/thingsboard.conf
```

Add the following lines. Replace `PUT_YOUR_POSTGRESQL_PASSWORD_HERE` with your actual postgres password:

```bash
# DB Configuration
export DATABASE_TS_TYPE=sql
export SPRING_DATASOURCE_URL=jdbc:postgresql://localhost:5432/thingsboard
export SPRING_DATASOURCE_USERNAME=postgres
export SPRING_DATASOURCE_PASSWORD=PUT_YOUR_POSTGRESQL_PASSWORD_HERE
```

</TabItem>
<TabItem label="Hybrid (PostgreSQL + Cassandra)" id="hybrid">

<Aside type="note">
The hybrid approach stores time-series data in Cassandra while using PostgreSQL for main entities (devices, assets, dashboards, customers). Recommended for deployments with 1M+ devices or high data ingestion rate (> 5K msg/sec).
</Aside>

**Install PostgreSQL 16:**

```bash
sudo apt install -y postgresql-common
sudo /usr/share/postgresql-common/pgdg/apt.postgresql.org.sh
sudo apt update
sudo apt -y install postgresql-16
sudo service postgresql start
```

Set a password for the `postgres` user:

```bash
sudo -u postgres psql -c "\password"
```

Create the ThingsBoard database:

```bash
psql -U postgres -d postgres -h 127.0.0.1 -W
```

```sql
CREATE DATABASE thingsboard;
```

Press `Ctrl+D` twice to exit.

**Install Cassandra:**

```bash
echo "deb [signed-by=/etc/apt/keyrings/apache-cassandra.asc] https://debian.cassandra.apache.org 50x main" | sudo tee /etc/apt/sources.list.d/cassandra.sources.list
sudo curl -o /etc/apt/keyrings/apache-cassandra.asc https://downloads.apache.org/cassandra/KEYS
sudo apt-get update
sudo apt-get install -y cassandra cassandra-tools
```

Verify Cassandra is running:

```bash
nodetool status
```

The status column should report `UN` (Up/Normal). Wait a few moments for Cassandra to initialize if needed.

**Configure ThingsBoard:**

Edit the configuration file:

```bash
sudo nano /etc/thingsboard/conf/thingsboard.conf
```

Add the following lines. Replace `PUT_YOUR_POSTGRESQL_PASSWORD_HERE` with your actual postgres password:

```bash
# DB Configuration
export DATABASE_TS_TYPE=cassandra
export SPRING_DATASOURCE_URL=jdbc:postgresql://localhost:5432/thingsboard
export SPRING_DATASOURCE_USERNAME=postgres
export SPRING_DATASOURCE_PASSWORD=PUT_YOUR_POSTGRESQL_PASSWORD_HERE
```

Optionally, configure external Cassandra connection:

```bash
export CASSANDRA_CLUSTER_NAME=Thingsboard Cluster
export CASSANDRA_KEYSPACE_NAME=thingsboard
export CASSANDRA_URL=127.0.0.1:9042
export CASSANDRA_USE_CREDENTIALS=false
export CASSANDRA_USERNAME=
export CASSANDRA_PASSWORD=
```

## Step 4. Choose queue service

ThingsBoard uses a message broker for internal communication between services. Choose one of the options below:

- **In Memory** — built-in, default. Suitable for development and PoC environments. Not recommended for production or cluster deployments.
- **Kafka** — recommended for production. Used on most ThingsBoard production environments. Works for on-prem and private cloud deployments. Some cloud providers offer managed Kafka (e.g. <a href="https://aws.amazon.com/msk/" target="_blank">AWS MSK</a>).
- **Confluent Cloud** — fully managed streaming platform based on Kafka. Useful for cloud-agnostic deployments.

<Tabs>
<TabItem label="In Memory (default)">

In Memory queue is built-in and enabled by default. No configuration needed.

</TabItem>
<TabItem label="Kafka (production)">

<a href="https://kafka.apache.org/" target="_blank">Apache Kafka</a> is an open-source stream-processing software platform.

**Install Docker:**

See <a href="https://docs.docker.com/engine/install/ubuntu/" target="_blank">Docker for Ubuntu</a>.

<Aside type="caution">
Add your user to the `docker` group so you can run commands without `sudo`.
See <a href="https://docs.docker.com/engine/install/linux-postinstall/#manage-docker-as-a-non-root-user" target="_blank">Manage Docker as a non-root user</a>.
</Aside>

Create the `docker-compose-kafka.yml` file with the following content:

<Code code={kafkaYaml} lang="yml" meta="maxLines=20 collapsible download='docker-compose-kafka.yml'" />

Start the Kafka container:

```bash
docker compose -f docker-compose-kafka.yml up -d
```

Configure ThingsBoard to use Kafka. Edit the configuration file:

```bash

```

Add the following lines:

```bash

export TB_KAFKA_SERVERS=localhost:9092
```

</TabItem>

To use <a href="https://confluent.cloud/login" target="_blank">Confluent Cloud</a>, first create an account, then create a <a href="https://docs.confluent.io/current/quickstart/cloud-quickstart/index.html#step-1-create-a-ak-cluster-in-ccloud" target="_blank">Kafka cluster</a> and obtain your <a href="https://docs.confluent.io/current/quickstart/cloud-quickstart/index.html#step-4-create-an-api-key" target="_blank">API Key</a>.

## Step 5. [Optional] Memory configuration

Edit the ThingsBoard configuration file:

```bash
sudo nano /etc/thingsboard/conf/thingsboard.conf
```

Add or update the following environment variable. Set the value to half of your total RAM (e.g. `2G` for a 4 GB machine, `4G` for an 8 GB machine):

```bash
export JAVA_OPTS="$JAVA_OPTS -Xms2G -Xmx2G"
```

## Step 6. Run installation script

```bash
sudo /usr/share/thingsboard/bin/install/install.sh --loadDemo
```

## Step 7. Start ThingsBoard

```bash
sudo service thingsboard start
```

<Aside type="note">
Please allow up to 90 seconds for the Web UI to start.
</Aside>

Once started, open `http://localhost:8080` in your browser. You should see the ThingsBoard login page. Use the following default credentials:

- **System Administrator**: sysadmin@thingsboard.org / sysadmin
- **Tenant Administrator**: tenant@thingsboard.org / tenant
- **Customer User**: customer@thingsboard.org / customer

You can change passwords for each account in the account profile page.

See <DocLink product={Products.CE} path="getting-started">Getting Started</DocLink> for your next steps after login.

## Configure HAProxy (HTTPS)

You may want to configure HTTPS access using HAProxy. This is possible if you are hosting ThingsBoard in the cloud and have a valid DNS name assigned to your instance.

## Troubleshooting

ThingsBoard logs are stored in `/var/log/thingsboard`. Check for errors with:

```bash
cat /var/log/thingsboard/thingsboard.log | grep ERROR
```
