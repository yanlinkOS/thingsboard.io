This guide walks you through setting up ThingsBoard Community Edition in cluster mode using Kubernetes and Minikube with microservices architecture.

For a simpler single-node installation, see <DocLink product={Products.CE} path="installation/docker">Docker (Linux, macOS)</DocLink>. For production cluster deployments, see <DocLink product={Products.CE} path="installation/docker-compose-setup">Docker Compose (Cluster)</DocLink>.

## Prerequisites

ThingsBoard microservices run on a Kubernetes cluster. You need:

- <a href="https://kubernetes.io/docs/setup/learning-environment/minikube/" target="_blank">Minikube</a> installed
- `kubectl` command-line tool configured to communicate with your cluster

### Enable ingress addon

By default the ingress addon is disabled in Minikube. Enable it:

```bash
minikube addons enable ingress
```

## Step 2. Configure database

Edit the `.env` file to set the database type:

```bash
nano .env
```

Set the `DATABASE` variable to one of:

| Value | Description |
|---|---|
| `postgres` | Use PostgreSQL for all data |
| `hybrid` | Use PostgreSQL for entities and Cassandra for time-series data |

<Aside type="note">
The corresponding Kubernetes resources are defined in `postgres.yml` and `cassandra.yml`.
</Aside>

<Aside type="note">
If you selected `hybrid`, you can also configure the number of Cassandra nodes (`StatefulSet.spec.replicas` in `cassandra.yml`) and the `CASSANDRA_REPLICATION_FACTOR` in `.env`. It is recommended to have 3 Cassandra nodes with replication factor 2.
</Aside>

## Step 3. Install and start ThingsBoard

<Steps>
1. Run the installation script:

   ```bash
   ./k8s-install-tb.sh --loadDemo
   ```

   The `--loadDemo` flag loads sample tenant account, dashboards, and devices for evaluation and testing.

2. Deploy third-party resources:

   ```bash
   ./k8s-deploy-thirdparty.sh
   ```

   Type **yes** when prompted if you are running ThingsBoard in `high-availability` deployment type for the first time or don't have a configured Redis cluster.

3. Deploy ThingsBoard resources:

   ```bash
   ./k8s-deploy-resources.sh
   ```
</Steps>

After all resources start, get the cluster IP:

```bash
minikube ip
```

Open `http://{your-cluster-ip}` in your browser. You should see the ThingsBoard login page. Use the following default credentials:

- **System Administrator**: sysadmin@thingsboard.org / sysadmin
- **Tenant Administrator**: tenant@thingsboard.org / tenant
- **Customer User**: customer@thingsboard.org / customer

You can change passwords for each account in the account profile page.

See <DocLink product={Products.CE} path="getting-started">Getting Started</DocLink> for your next steps after login.

## Inspect logs and manage resources

List running ThingsBoard node pods:

```bash
kubectl get pods -l app=tb-node
```

Stream logs of a specific pod:

```bash
kubectl logs -f TB_NODE_POD_NAME
```

Replace `TB_NODE_POD_NAME` with the pod name from the list above.

Other useful commands:

| Command | Description |
|---|---|
| `kubectl get pods` | List all pods |
| `kubectl get services` | List all services |
| `kubectl get deployments` | List all deployments |

See the <a href="https://kubernetes.io/docs/reference/kubectl/cheatsheet/" target="_blank">kubectl Cheat Sheet</a> for more commands.

### Cleanup

Delete ThingsBoard microservices:

```bash
./k8s-delete-resources.sh
```

Delete third-party services:

```bash
./k8s-delete-thirdparty.sh
```

Delete all resources including database:

```bash
./k8s-delete-all.sh
```

## Upgrading

Pull the latest changes from the repository:

```bash
git pull origin master
```

Then run the upgrade:

```bash
./k8s-delete-resources.sh
./k8s-upgrade-tb.sh
./k8s-deploy-resources.sh
```

<Aside type="note">
Perform upgrades sequentially (e.g. v4.0.0 → v4.0.1 → v4.1.0).
</Aside>

<Aside type="note">
The `--fromVersion` flag is required for upgrades prior to v3.9.1. For example: `./k8s-upgrade-tb.sh --fromVersion=3.8.1`. See <DocLink product={Products.CE} path="installation/upgrade-instructions/" target="_blank">Upgrade Instructions</DocLink> for valid `fromVersion` values.
</Aside>

### Migration to Professional Edition

<Steps>
1. Upgrade to the latest CE version.

2. Stop ThingsBoard resources:

   ```bash
   ./k8s-delete-resources.sh
   ```

3. Merge your configuration with the <a href="https://github.com/thingsboard/thingsboard-pe-k8s" target="_blank">latest PE Minikube scripts</a>. Configure the license key as described in the <DocLink product={Products.PE} path="installation/minikube-cluster-setup">PE Minikube guide</DocLink>.

4. Run the migration script:

   ```bash
   ./k8s-upgrade-tb.sh --fromVersion=CE
   ```

5. Deploy ThingsBoard resources:

   ```bash
   ./k8s-deploy-resources.sh
   ```
</Steps>
