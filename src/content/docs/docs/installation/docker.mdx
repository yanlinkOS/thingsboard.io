This guide walks you through installing and starting ThingsBoard Community Edition using Docker on Linux or macOS.

## Prerequisites

Install Docker:
- <a href="https://docs.docker.com/engine/install/ubuntu/" target="_blank">Docker for Ubuntu</a>
- <a href="https://docs.docker.com/engine/install/centos/" target="_blank">Docker for CentOS</a>
- <a href="https://docs.docker.com/desktop/setup/install/mac-install/" target="_blank">Docker Desktop for macOS</a>

A server with at least **4 GB of RAM** (8 GB recommended) and minimal load (a few messages per second).

<Aside type="caution">
On Linux, add your user to the `docker` group so you can run commands without `sudo`.
See <a href="https://docs.docker.com/engine/install/linux-postinstall/#manage-docker-as-a-non-root-user" target="_blank">Manage Docker as a non-root user</a>.
</Aside>

## Step 1. Create docker compose file

ThingsBoard uses a message broker for internal communication between services. Choose one of the options below:

- **In Memory** — built-in, default. Suitable for development and PoC environments. Not recommended for production or cluster deployments.

- **Confluent Cloud** — fully managed streaming platform based on Kafka. Useful for cloud-agnostic deployments.

### Docker compose parameters

| Parameter | Description |
|---|---|
| `8080:8080` | HTTP port |
| `1883:1883` | MQTT port |
| `8883:8883` | MQTT over SSL port |
| `7070:7070` | Edge RPC port |
| `5683-5688:5683-5688/udp` | CoAP and LwM2M ports |
| `tb-postgres-data` | Docker volume for PostgreSQL data |
| `thingsboard-ce` | Container name |
| `restart: always` | Auto-restart on reboot or failure |
| `image: "thingsboard/tb-node:..."` | ThingsBoard CE Docker image |

## Step 2. Initialize database schema and system assets

Before starting ThingsBoard, initialize the database schema and load built-in assets:

```bash
docker compose run --rm -e INSTALL_TB=true -e LOAD_DEMO=true thingsboard-ce
```

| Variable | Description |
|---|---|
| `INSTALL_TB=true` | Installs the core database schema and system resources (widgets, images, rule chains, etc.) |
| `LOAD_DEMO=true` | Loads sample tenant account, dashboards, and devices for evaluation and testing |

## Step 3. Start the platform

Bring up all containers in detached mode, then follow the ThingsBoard logs:

```bash
docker compose up -d && docker compose logs -f thingsboard-ce
```

Once started, open `http://localhost:8080` in your browser. You should see the ThingsBoard login page. Use the following default credentials:

- **System Administrator**: sysadmin@thingsboard.org / sysadmin
- **Tenant Administrator**: tenant@thingsboard.org / tenant
- **Customer User**: customer@thingsboard.org / customer

You can change passwords for each account in the account profile page.

You can safely detach from the log stream (Ctrl+C) — containers will continue running.

See <DocLink product={Products.CE} path="getting-started">Getting Started</DocLink> for your next steps after login.

## Inspect logs and control containers

Stream the ThingsBoard container logs:

```bash
docker compose logs -f thingsboard-ce
```

Stop all containers:

```bash
docker compose down
```

Start all containers:

```bash
docker compose up -d
```

## Upgrading

When a new ThingsBoard release becomes available, update your installation to benefit from the latest features and security patches.

<Aside type="note">
To maintain database integrity and system stability, perform upgrades sequentially (e.g. v4.0.2 → v4.1.0 → v4.2.0).
</Aside>

See the <DocLink product={Products.CE} path="installation/upgrade-instructions/" target="_blank">Upgrade Instructions</DocLink> for detailed steps.

## Troubleshooting

### DNS issues

If you observe errors related to DNS issues, for example:

```bash
127.0.1.1:53: cannot unmarshal DNS message
```

Configure your system to use Google public DNS servers. See <a href="https://developers.google.com/speed/public-dns/docs/using#linux" target="_blank">Linux</a> and <a href="https://developers.google.com/speed/public-dns/docs/using#mac_os" target="_blank">macOS</a> instructions.
