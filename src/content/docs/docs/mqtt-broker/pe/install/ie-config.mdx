This page lists all TBMQ Integration Executor configuration parameters and their default values.

The Integration Executor is configured through environment variables.
In a **Docker Compose** deployment, set them in the `docker-compose.yml` or `.env` file (see the <DocLink product={Products.TBMQ_PE} path="install/docker">Docker installation guide</DocLink>).
In a **Kubernetes** deployment, set them in Helm values or the Deployment/StatefulSet manifests (see the <DocLink product={Products.TBMQ_PE} path="install/installation-options">installation options</DocLink>).

## Configuration file format

The `tbmq-integration-executor.yml` configuration file is written in YAML. Each parameter has an environment variable name and a default value:

```yaml
server:
  address: "${HTTP_BIND_ADDRESS:0.0.0.0}"
```

Here `HTTP_BIND_ADDRESS` is the environment variable and `0.0.0.0` is the default. To override a value, set the corresponding environment variable:

```bash
HTTP_BIND_PORT=8084
```
