<DocLink product={props.product} path='user-guide/dashboards/' target="_blank">Dashboard</DocLink> is a centralized, configurable user interface used for monitoring, analyzing, and managing IoT systems in real time.

A dashboard consists of <DocLink product={props.product} path='user-guide/widgets/' target="_blank">widgets</DocLink> — configurable components such as charts, cards, tables, or maps — used to visualize data, display alarms and events, and interact with devices or processes.

In this step, you will create a dashboard and add three widgets:
- a widget that displays the latest temperature value;
- a chart widget that shows temperature changes over time;
- an alarms widget that displays alarms triggered by high temperature conditions.

## Step 2.2 Add a Value Card Widget

Add a <DocLink product={props.product} path='reference/widgets/cards/value-card/' target="_blank">Value card</DocLink> widget to display the latest temperature value.

<Steps>
  <ol>
    <li>In the dashboard edit mode, click <strong>&#43; Add new widget</strong> in the center of the screen.</li>
    <li>Select <strong>Cards &#8702; Value card</strong>.</li>
    <li>
      Configure the widget:
      <ul style={{ marginTop: 0 }}>
        <li>Set <strong>Thermometer</strong> device as the datasource.</li>
        <li>Use <strong>temperature</strong> as the data key.</li>
      </ul>
    </li>
    <li>Click <strong>Add</strong>.</li>
    <li>Resize the widget if necessary.</li>
  </ol>
</Steps>

The widget displays the latest temperature reported by the device.

<Aside type="tip">
  This guide uses a single device as the widget datasource.
  For dynamic or multi-entity dashboards, use <DocLink product={props.product} path='user-guide/aliases/' target="_blank">entity aliases</DocLink>.
</Aside>

## Step 2.3 Add a Time Series Chart Widget

A <DocLink product={props.product} path='reference/widgets/charts/time-series-chart/' target="_blank">Time series chart</DocLink> visualizes how telemetry values change over time.

Add a chart to display temperature history:

<Steps>
1. In the dashboard edit mode, click **&#43; Add widget**.
2. Select **Charts &#8702; Time series chart**.
3. Set **Thermometer** device as the datasource.
4. In **Series**, specify **temperature** as the series key.
5. Click **Add**.
6. Adjust the widget size if needed.
</Steps>

The chart displays all recorded temperature values with timestamps for the last hour.

Send several new telemetry updates (see <DocLink product={props.product} path='getting-started/1/#step-12-verify-device-connectivity' target="_blank">Step 1.2</DocLink>). The chart updates automatically as new data arrives, providing real-time visualization of temperature changes.

<Aside type="tip">
  Explore the dashboard <DocLink product={props.product} path='user-guide/dashboards' target="_blank">time window</DocLink> settings, as well as data aggregation and grouping options, to customize how telemetry is displayed.
</Aside>

## Step 2.4 Add an Alarms Table Widget

An <DocLink product={props.product} path='reference/widgets/alarm-widgets/alarms-table/' target="_blank">Alarms table</DocLink> widget displays alarms associated with selected entities, including their severity and status.

Add an alarms table to the dashboard:

<Steps>
1. In edit mode, click **&#43; Add widget**.
2. Select **Alarm widgets &#8702; Alarms table**.
3. Set **Thermometer** device as the datasource.
4. Configure alarm severity and status filters.  
  The widget displays alarms matching the selected properties. If none are selected, all alarms will be displayed, regardless of their status or severity.
5. Click **Add**.
6. Resize and position the widget if needed.
</Steps>

##### Save the Dashboard

Click **Save** in the top-right corner to save the dashboard.

The dashboard now provides real-time visibility into telemetry data and active alarms.
