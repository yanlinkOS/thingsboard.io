<DocLink product={props.product} path='user-guide/alarms' target="_blank">Alarm</DocLink> in ThingsBoard represents an abnormal condition or critical event related to an entity, such as a device. Alarms are generated automatically when conditions defined in <DocLink product={props.product} path='user-guide/alarm-rules' target="_blank">alarm rules</DocLink> are met (for example, when a telemetry value exceeds a threshold).

In this step, you will configure an alarm rule that detects when the temperature exceeds a defined threshold.
Then, you will simulate this condition by publishing telemetry data above the threshold to trigger the alarm and generate a notification.

---

## Step 3.1 Configure Alarm Rule

Create an alarm rule that triggers when the temperature exceeds **25 °C**.

<Aside type="tip">
  Configure the alarm rule at the **device profile** level to ensure the same logic is applied to all devices using this profile.
</Aside>

#### Add the Alarm Rule

<Steps>
1. Navigate to **Alarms &#8702; Alarm rules**.
2. In the upper-right corner, click **&#43; Add alarm rule &#8702; Create new alarm rule**.
</Steps>

In the alarm rule configuration dialog, specify the following:

##### General Settings

<Steps>
1. Alarm type: `High temperature`
2. Target entity: **default**
</Steps>

##### Add a Telemetry Argument

<Steps>
  <ol>
    <li>Click <strong>Add argument</strong>.</li>
    <li>
      Configure the argument:
      <ul style={{ marginTop: 0 }}>
        <li>Entity type: <strong>Current entity</strong></li>
        <li>Argument type: <strong>Latest telemetry</strong></li>
        <li>Time series key: <code>temperature</code></li>
        <li>Argument name: <code>temperature</code></li>
      </ul>
    </li>
    <li>Click <strong>Add</strong>.</li>
  </ol>
</Steps>

This creates the <span class="code-light">temperature</span> variable used in the alarm condition.

##### Define the Trigger Condition

<Steps>
1. Click **Add trigger condition**.
2. Ensure that **Severity** is set to **Critical**.
3. **Add a condition** in **Script** mode:
   ```javascript
   // Triggers an alarm when temperature is above 25 degrees
   return temperature > 25;
   ```
4. Click **Save**.
</Steps>

##### Save the Alarm Rule

Click **Add** to save and activate the alarm rule.

The alarm will now be triggered whenever the device temperature exceeds **25 °C**.

---

## Step 3.2 Trigger an Alarm

Publish telemetry with a value greater than **25 °C** (for example, `26`).
Use the Check connectivity feature as described in <DocLink product={props.product} path='getting-started/1/#step-12-verify-device-connectivity' target="_blank">Step 1.2</DocLink> to send the test telemetry data.

Once the threshold is exceeded, the new active alarm can be viewed, acknowledged, or cleared from:
- the **Alarms** page;
- the dashboard using the **Alarms widget**;
- the **Alarms** tab in the device details page.

You can also view the new alarm notification by clicking the **bell icon** in the top-right corner of the screen or by opening the **Notification center**.

This demonstrates how ThingsBoard detects and reacts to critical conditions in real time.

## Notifications

ThingsBoard <DocLink product={props.product} path='user-guide/notifications' target="_blank">Notification center</DocLink> allows you to send personalized notifications to end users about device activity, alarms, and system events within your IoT ecosystem.

Notifications can be delivered via web, email, SMS, or third-party integrations.

<Aside type="tip">
  To learn how to configure email notifications for alarms, see the <DocLink product={props.product} path='recipes/send-email-alarm/' target="_blank">Send email on alarm</DocLink> guide.
</Aside>

You can also use the <DocLink product={props.product} path='user-guide/mobile-app-center' target="_blank">ThingsBoard Mobile Application</DocLink> to receive push notifications about critical events directly on your smartphone—wherever you are.
