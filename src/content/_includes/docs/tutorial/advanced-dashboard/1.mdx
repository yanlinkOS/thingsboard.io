## Step 1.1 Import Assets

<DocLink bold={false} product={props.product} path='user-guide/assets' target="_blank">Assets</DocLink> represent real-world objects such as buildings and office spaces.

In this step, create four assets: Building A, Building B, Office A, and Office B.  
Each asset should also include <DocLink bold={false} product={props.product} path='user-guide/digital-twins/attributes' target="_blank">attributes</DocLink> such as building coordinates, address, floor information, office phone number, and other relevant details that will be used in the next step.

You can create each asset manually or import multiple assets at once using <DocLink bold={false} product={props.product} path='user-guide/provisioning#bulk-provisioning' target="_blank">bulk provisioning</DocLink> feature. After import, all buildings and offices will have the required attribute data.

### Import Buildings

<Steps>
  1. Download <a href="/resources/docs/tutorial/advanced-dashboard-guide-1/building-assets.csv" download="building-assets.csv">building-assets.csv</a>
  2. Go to **Entities &#8702; Assets**.
  3. Click **+ Add asset &#8702; Import asset**.
  4. Drag the `building-assets.csv` file into the import window and click **Continue**.
  5. Verify the delimiter `,`, then click **Continue**.
  6. Map the data between the uploaded file columns and the ThingsBoard data types using the table below.
  7. Click **Continue**, then **OK**.
</Steps>

| Key | Column type | Building A | Building B |
|-----|------|-------|-------|
| `name` | Name | `Building A` | `Building B` |
| `type` | Type | `	building` | `	building` |
| `latitude` | Server attribute | `40.75912` | `40.75901` |
| `longitude` | Server attribute | `-73.97600` | `-73.96997` |
| `address` | Server attribute | `645 5th Ave, New York` | `641 Lexington Ave, New York` |
| `email` | Server attribute | `contact@building-a.com` | `contact@building-b.com` |
| `phone` | Server attribute | `+121 244 55 66` | `+121 555 66 77` |
| `contactPerson` | Server attribute | `Thomas Johnson` | `Jack Williams` |

<Aside type="note">
  The `latitude` and `longitude` attributes are required by the OpenStreet Map widget to place entity markers on the map.
</Aside>

## Step 1.2 Import Devices

<DocLink bold={false} product={props.product} path='user-guide/devices/' target="_blank">Devices</DocLink> are IoT entities that collect data from their environment, send it to ThingsBoard, and respond to RPC commands.

Create four devices—an indoor air quality sensor, an energy meter, a water flow meter, and another indoor air quality sensor—using the <DocLink bold={false} product={props.product} path='user-guide/provisioning/#bulk-provisioning' target="_blank">bulk provisioning</DocLink> feature as well.
Each sensor has its own type and location coordinates, which will be used later to display them on the office map.

Save the following content as `devices.csv`:

```csv
name,type,label,xPos,yPos
SD-001,air-sensor,Indoor Air Quality Sensor,0.51,0.68
EM-002,energy-sensor,Energy Meter,0.14,0.95
WM-003,water-sensor,Water Flow Meter,0.14,0.32
AM-307,air-sensor,IAQ Sensor,0.49,0.38
```

Or <a href="/resources/docs/tutorial/advanced-dashboard-guide-1/devices.csv" download="devices.csv">download a pre-made one</a>.

<Steps>
1. Go to **Entities &#8702; Devices**.
2. Click **+ Add device &#8702; Import device**.
3. Drag the `devices.csv` file into the import window and click **Continue**.
4. Verify the delimiter, then click **Continue**.
5. Map columns: **name &#8702; Name**, **type &#8702; Device Profile**, **label &#8702; Label**.
6. Click **Continue**, then **OK**.
</Steps>

## Step 1.3 Add Relations

Feature <DocLink bold={false} product={props.product} path='user-guide/digital-twins/relations' target="_blank">Relations</DocLink> define directed connections between entities.

Link each building to its offices, and each office to its devices.

```
Buildings
├── Building A
│   ├── Office A
│   │   ├── SD-001
│   │   ├── EM-002
│   │   └── WM-003
│   └── Office B
│       └── AM-307
└── Building B
```

**Link offices to Building A:**

<Steps>
1. Open **Building A** details and go to the **Relations** tab.
2. Set direction to **From**, then click **+**.
3. Set relation type to **Contains** and entity type to **Asset**.
4. Select **Office A** and **Office B**, then click **Add**.
</Steps>

6. Save the dashboard by clicking the **Save** button in the upper-right corner of the screen.

### Add the Widget

With images uploaded and attributes set, add the OpenStreet Map widget to the dashboard.

<Steps>
1. Navigate to **Dashboards &#8702; Buildings**.
2. Click **+ Add widget**.
3. Open the **Buildings** dashboard, enter edit mode, and click **Add widget**.
4. Select **Maps &#8702; Map**.
5. Open the widget configuration and switch to the <strong>Advanced</strong> tab. 
6. Select <strong>Positron</strong> layer for <strong>OpenStreet</strong> map type.
7. Scroll to the <strong>Overlays</strong> section and click <strong>Add marker</strong> on the <strong>Markers</strong> tab.
8. Set <strong>Source</strong> to <strong>Entity</strong>, enter <strong>building</strong> as entity alias name, and click <strong>Create a new one!</strong>.  
9. Create alias:
    - Set filter type to **Asset type**
    - Specify asset type to **building**
    - Click **Add**
</Steps>

## Step 1.7 Customize Dashboard Appearance

Apply a background image and global CSS styles to give the dashboard a polished, professional look.

<Steps>
1. Click **Settings** on the dashboard toolbar.
2. In **Advanced settings &#8702; Dashboard CSS**, paste:
   ```css
   .tb-widget-container > .tb-widget {
     border-radius: 8px;
     box-shadow: 0px 4px 10px rgba(23, 33, 90, 0.5);
   }
   .tb-dashboard-page .tb-widget-container > .tb-widget .leaflet-popup a.tb-custom-action {
     font-family: 'Roboto';
     font-weight: 500;
     font-size: 14px;
     line-height: 20px;
     letter-spacing: 0.25px;
     border-bottom: none;
     color: #00695C;
   }
   ```

   <ImageGallery images={[
     { src: '/src/assets/images/user-guide/advanced-guides-for-working-with-dashboard/lesson-1/dashboard-settings-background-5.png', alt: 'Dashboard toolbar — Settings button highlighted', caption: 'Click the "Settings" button on the dashboard toolbar' },
     { src: '/src/assets/images/user-guide/advanced-guides-for-working-with-dashboard/lesson-1/dashboard-settings-css-2.png', alt: 'Dashboard settings dialog — Advanced settings with Dashboard CSS field', caption: 'Open "Advanced settings" and paste the CSS into the "Dashboard CSS" field' },
   ]} />

   This CSS snippet applies styles to elements with the `.tb-widget` class that are direct children of the `.tb-widget-container`.
   It specifies `border-radius: 8px`, which gives the element soft rounded corners, and `box-shadow: 0px 4px 10px rgba(23, 33, 90, 0.5)`, which creates a semi-transparent shadow with a subtle blur, offset by 4 pixels downward, adding visual depth, and specifies `opacity: 0.9` to make the element slightly transparent, allowing for some background visibility while maintaining the content&#39;s visibility.
