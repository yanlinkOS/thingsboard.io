Send a temperature alarm email to a customer using the Rule Engine. The email recipient and device address are read dynamically from entity attributes — no hardcoded addresses required.

## Use case

**Thermostat Home** sends temperature telemetry. The device is assigned to **Customer A**, who has an `email` server attribute.

When the temperature goes outside the acceptable range (below −40 °C or above 80 °C):

- ThingsBoard fetches the customer's email from their attributes
- Fetches the device's physical address from the device's server attributes
- Creates an alarm
- Sends an email to the customer with the device name, temperature value, and address

## Prerequisites

- Configure your <DocLink product={props.product} path="user-guide/ui/mail-settings">mail server settings</DocLink> in ThingsBoard before proceeding.
- Review <DocLink product={props.product} path="user-guide/notifications">Notification Center</DocLink> if you prefer a no-code alternative to rule-chain-based email.

## Step 2. Create rule chain

You can import a ready-made rule chain or build it manually.

**Option 1. Import rule chain**

- Download the **Send Alarm Email to Customer** rule chain as a <a href="/resources/docs/recipes/send-alarm-email-to-customer/send_alarm_email_to_customer_rule_chain.json" download="send_alarm_email_to_customer_rule_chain.json"><strong>JSON file</strong></a> and <DocLink product={props.product} path="user-guide/rule-engine/#import-a-rule-chain" target="_blank">import</DocLink> it into your instance.

**Option 2. Create manually**

- <DocLink product={props.product} path="user-guide/rule-engine/#create-a-rule-chain" target="_blank">Create a new rule chain</DocLink> named `Send Alarm Email to Customer`, open it, and add the following nodes:

### Step 2.1. Filter Script node

Add a <DocLink product={props.product} path="reference/rule-engine/nodes/filter/script" target="_blank">Filter Script</DocLink> node connected to the **Input** node.

Passes the message only when the temperature is outside the acceptable range.

- **Name**: `Check Temperature Range`
- **Add the following script**:

    <Tabs>
      <TabItem label="TBEL">
        ```javascript
        return msg.temperature < -15 || msg.temperature > 30;
        ```
      </TabItem>
      <TabItem label="JavaScript">
        ```javascript
        return msg.temperature < -15 || msg.temperature > 30;
        ```
      </TabItem>
    </Tabs>

### Step 2.4. Transformation Script node

Add the <DocLink product={props.product} path="reference/rule-engine/nodes/transformation/script/" target="_blank">Transformation Script</DocLink> node and place it after the **Get Device Address** node with a relation type **Success**.

This node will use for saving current temperature from Message Data to the Message Metadata using the following script:

- **Name**: `Add Temperature to Metadata`
- **Add the following script**:

    <Tabs>
      <TabItem label="TBEL">
        ```javascript
        metadata.temperature = msg.temperature;
        return {msg: msg, metadata: metadata, msgType: msgType};
        ```
      </TabItem>
      <TabItem label="JavaScript">
        ```javascript
        metadata.temperature = msg.temperature;
        return {msg: msg, metadata: metadata, msgType: msgType};
        ```
      </TabItem>
    </Tabs>

### Step 2.5. Create Alarm node

Add a <DocLink product={props.product} path="reference/rule-engine/nodes/action/create-alarm" target="_blank">Create Alarm</DocLink> node connected to the **Add Temperature to Metadata** node with relation type **Success**.

Creates an alarm when the temperature is out of range.

- **Name**: `Create Temperature Alarm`
- **Alarm type**: `High Temperature`
- **Alarm severity**: `Critical`

<Aside type="note">
  The email is sent only when the alarm is **created**. If an uncleared alarm of the same type already exists, the node updates it — and the email node downstream will not re-send.
</Aside>

- **Recipients**:
  - **To**: `${customerEmail}`
- **Message subject and content**:

    ```
    Temperature alert on device ${deviceName}
    ```
  
  - **Mail body type**: Plain text
  - **Body** template: 
    ```

    Device address: ${ss_address}
    ```

- **Input &#8702; Check Temperature Range**
- **Check Temperature Range &#8702;** `True` **&#8702; Get Customer Email**

- **Get Device Address &#8702;** `Success` **&#8702; Add Temperature to Metadata**
- **Add Temperature to Metadata &#8702;** `Success` **&#8702; Create Temperature Alarm**
- **Create Temperature Alarm &#8702;** `Created` **&#8702; Build Email**
- **Build Email &#8702;** `Success` **&#8702; Send Email**
