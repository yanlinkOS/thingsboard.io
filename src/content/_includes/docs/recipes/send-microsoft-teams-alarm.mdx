Configure ThingsBoard to automatically send notifications to Microsoft Teams when an alarm is created. This guide extends the <DocLink product={props.product} path="recipes/create-clear-alarms" bold={false} target="_blank">Create and Clear Alarms</DocLink> recipe by using the <DocLink product={props.product} path="user-guide/notifications" bold={false} target="_blank">Notification Center</DocLink> to deliver alarm-based messages to a Teams channel via a webhook.

## Use case

Assume your refrigeration equipment has a temperature sensor that periodically publishes telemetry to ThingsBoard. The normal operating range is **2 °C** – **5 °C**. When a reading goes outside this range:

- An alarm is created by the alarm rules.
- A message is sent to the configured Microsoft Teams channel.

## Prerequisites

Before proceeding:

- Complete the <DocLink product={props.product} path="recipes/create-clear-alarms" bold={false} target="_blank">Create and Clear Alarms</DocLink> recipe — the alarm rules configured there are required for this guide.
- Create a webhook workflow in Microsoft Teams and obtain the webhook URL — see <DocLink product={props.product} path="user-guide/ui/microsoft-teams-settings" bold={false} target="_blank">Microsoft Teams settings</DocLink>.

Review the following documentation for background:

- <DocLink product={props.product} path="user-guide/notifications" bold={false} target="_blank">Notification Center</DocLink> — how recipients, templates, and rules work together.
- <DocLink product={props.product} path="user-guide/alarm-rules" bold={false} target="_blank">Alarm rules</DocLink> — how alarms are created and cleared.

## Configure Notification Center

The Notification Center delivers notifications to users. It consists of three components:

- **Recipients** — define who receives notifications.
- **Templates** — define the notification content and delivery method.
- **Rules** — define the conditions under which notifications are sent.

### Step 1. Configure notification recipients

Create a recipient that posts to the Microsoft Teams channel via the webhook URL.

<Steps>
1. Navigate to **Notification Center &#8702; Recipients**.
2. Click **+ Add recipients** in the top-right corner.
3. Configure the recipient:
   - **Name**: `Microsoft Teams`
   - **Type**: Microsoft Teams
   - **Webhook URL**: paste the webhook URL copied from Teams 
   - **Channel type**: specify Microsoft Teams channel
4. Click **Add**.
</Steps>

### Step 2. Create notification template

Create a template that defines the Teams message subject and body using alarm variables.

<Steps>
1. Navigate to **Notification Center &#8702; Templates**.
2. Click **+ Add template** in the top-right corner.
3. **Setup** — configure the template:
   - **Name**: `Alarm notifications to Microsoft Teams`
   - **Type**: Alarm
   - **Delivery methods**: toggle **Microsoft Teams** on
4. Click **Next**.
5. **Compose** — expand the **Microsoft Teams** section and set:
   - **Subject**: 
      ```
      ${alarmType} on ${alarmOriginatorName} device
      ```
   - **Message**: 
      ```text
      A ${alarmType} has been detected on the ${alarmOriginatorName} device: ${details.data} °C.
      ```
6. Click **Add**.
</Steps>

Template variables used:

- `${alarmType}` — the alarm type name (e.g. *High temperature*).
- `${alarmOriginatorName}` — the name of the device that triggered the alarm.
- `${details.data}` — the reported telemetry value, populated by the alarm rule.

### Step 3. Create notification rule

Create a rule that links the alarm trigger to the template and recipient.

<Steps>
1. Navigate to **Notification Center &#8702; Rules**.
2. Click **+ Add rule** in the top-right corner.
3. **Basic settings** — configure the rule:
   - **Rule name**: `New alarm to Microsoft Teams`
   - **Trigger**: Alarm
   - **Template**: <DocLink product={props.product} path="recipes/send-microsoft-teams-alarm#step-2-create-notification-template" bold={false}>Alarm notifications to Microsoft Teams</DocLink>
   - **Escalation chain &#8702; First recipient**: <DocLink product={props.product} path="recipes/send-microsoft-teams-alarm#step-1-configure-notification-recipients" bold={false}>Microsoft Teams</DocLink>
4. Click **Next**.
5. **Alarm trigger settings** — configure the filter:
   - **Alarm type list**: Any type
   - **Alarm severity list**: Any severity
   - **Notify on**: Alarm created
6. Click **Add**.
</Steps>

<Aside type="note">
  Teams messages are sent only when an alarm is **created**, not when an existing alarm is updated.
</Aside>

## Verification

Publish a temperature value that triggers the alarm (above 5 °C).  
Use the <DocLink product={props.product} path="user-guide/devices/#check-connectivity" bold={false}>Check Connectivity</DocLink> feature or the command below:

<Aside type="note">
  {(props.product === Products.PAAS || props.product === Products.PAAS_EU) ? (
    <>Replace <code>$ACCESS_TOKEN</code> with the device access token.</>
  ) : (
    <>Replace <code>$THINGSBOARD_HOST_NAME</code> with your ThingsBoard host (e.g. <code>localhost:8080</code>) and <code>$ACCESS_TOKEN</code> with the device access token.</>
  )}
</Aside>

<Code code={`curl -v -X POST \\
  --header "Content-Type: application/json" \\
  --data '{"temperature": 25}' \\
  http://${tbHost(props.product)}/api/v1/$ACCESS_TOKEN/telemetry`} lang="bash" />

A new alarm is created and a message is posted to the configured Teams channel. The subject and body are populated from the template variables — for example: *"A High temperature has been detected on the Thermostat device: 25 °C."*
