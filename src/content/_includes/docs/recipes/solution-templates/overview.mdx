ThingsBoard provides IoT solution templates to significantly reduce the time-to-market for your IoT products.
Each solution template represents a ready-to-use PoC or MVP and includes:

- Pre-configured interactive <DocLink product={props.product} path='user-guide/dashboards'>dashboards</DocLink>
- <DocLink product={props.product} path='concepts/data-processing'>Data processing</DocLink> and business logic
- Sample <DocLink product={props.product} path='user-guide/devices'>devices</DocLink> with emulated telemetry
- All required <DocLink product={props.product} path='user-guide/digital-twins/entities'>entities</DocLink> and relations

Solution templates allow you to quickly evaluate a use case, demonstrate functionality, or use them as a baseline for further customization.

<Aside type="note">
Only Tenant Administrators can manage solution templates.
</Aside>

Browse available solution templates by navigating to **Solution templates** in the main menu.

## Managing solution templates

For each solution template, the following actions are available:

### View details

Open the template description by clicking **Details**. The details view provides an overview of the solution, including what dashboards, devices, and business logic are included. Use this to understand what the template offers before installing it.

<ImageGallery images={[
  { src: '/src/assets/images/recipes/solution-templates/overview/managing-solution-templates-1.png', alt: 'Details button on a solution template card' },
  { src: '/src/assets/images/recipes/solution-templates/overview/managing-solution-templates-2.png', alt: 'Solution template details view' },
]} />

### Install solution template

Install the template by clicking **Install**.

<Steps>
1. Go to **Solution templates** in the main menu.
2. Find the template you want to try and click **Install**.
3. Once the solution is installed, you are automatically forwarded to the main dashboard of the corresponding template, and the instructions dialog appears.
4. Browse the instructions and click **Close** to start using the solution.
</Steps>

The instructions are auto-generated for each installed template and typically include:
- Dashboard overview and navigation guide
- List of pre-provisioned devices with their credentials
- Example commands for sending telemetry data
- Configured <DocLink product={props.product} path='user-guide/alarm-rules'>alarm rules</DocLink> and thresholds
- Data processing logic, including calculated fields and rule chain descriptions
- <DocLink product={props.product === Products.CE ? Products.EDGE : Products.EDGE_PE} path='why-thingsboard-edge'>Edge</DocLink> computing setup for supported templates

### View instructions

After installing a template, you can reopen the auto-generated instructions at any time by clicking **Instructions**.

<ImageGallery images={[
  { src: '/src/assets/images/recipes/solution-templates/overview/managing-solution-templates-7.png', alt: 'Instructions button on an installed solution template' },
  { src: '/src/assets/images/recipes/solution-templates/overview/managing-solution-templates-8.png', alt: 'Auto-generated solution instructions dialog' },
]} />

### Delete a solution

Remove the installed solution and all <DocLink product={props.product} path='user-guide/digital-twins/entities'>entities</DocLink> created during the installation by clicking **Delete**.

<Aside type="caution">
Entities that you create through the solution dashboard (users, devices, etc.) are not deleted automatically.
</Aside>

## Connecting real devices

Each solution template includes detailed instructions describing:

- Expected telemetry and attribute payload formats
- Required keys and data structure
- Example commands for sending data to the platform

The provided examples use auto-generated devices and credentials created during template installation.
Try the examples first to understand how the solution works, then refer to the <DocLink product={props.product} path='user-guide/connectivity-guide'>How to connect your device</DocLink> guide to connect real devices using production credentials.
