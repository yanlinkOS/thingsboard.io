This guide shows how to use a <DocLink product={props.product} path="user-guide/calculated-fields/related-entities-aggregation">Related Entities Aggregation</DocLink> calculated field to calculate telemetry from multiple related devices, aggregate the values, and store the result on the parent asset.

## Use case

A building contains multiple water meters. Each meter sends telemetry: `waterConsumption`.

Goal: `totalWaterConsumption` calculated across all meters in the building.

ThingsBoard:
- finds all devices related to **Building A** via **Contains** relations
- sums up `waterConsumption` from all meters
- stores the result as `totalWaterConsumption` on **Building A**

## Step 1. Create devices and asset

Create an asset to represent the building:
- **Name**: `Building A`
- **Asset profile**: `building`

Create two devices with the **Water Meter** device profile:
- **Name**: `Water Meter A1`
- **Name**: `Water Meter A2`

## Step 2. Create relations

Link both water meters to the building using outbound **Contains** relations:

<Steps>
1. Open **Building A** details.
2. Go to the **Relations** tab.
3. Add an outbound **Contains** relation to **Water Meter A1**.
4. Add an outbound **Contains** relation to **Water Meter A2**.
</Steps>

<Aside type="note">
  The relation direction and type must match the aggregation path configured in the calculated field.
</Aside>

## Step 3. Create calculated field

You can import a ready-made calculated field or build it manually.

**Option 1. Import calculated field**

- Download the **Building Total Water Consumption** calculated field as <a href="/resources/docs/recipes/aggregate-related-entities/building_total_water_consumption_cf.json" download="building_total_water_consumption_cf.json"><strong>JSON file</strong></a> and <DocLink product={props.product} path="user-guide/calculated-fields/#export--import" target="_blank">import</DocLink> it into your instance.

    <Aside type="note">
      During import, set the <strong>Building A</strong> asset as the target entity.
    </Aside>

**Option 2. Create manually**

- Navigate to **Calculated fields** and click **+ Add calculated field**.

### General settings

- **Title**: `Building Total Water Consumption`
- **Entity type**: `Asset`
- **Entity**: `Building A`
- **Type**: `Related entities aggregation`

### Aggregation path to related entities

- **Relation direction**: `Down to child`
- **Relation type**: `Contains`

]} />

### Argument

Click **Add argument** and configure it to read `waterConsumption` from each related device:

- **Argument type**: `Latest telemetry`
- **Time series key**: `waterConsumption`
- **Argument name**: `waterConsumption`
- **Default value**: `0`

<MultiProductImageGallery product={props.product} images={[
  {
    src: '/src/assets/images/recipes/aggregate-related-entities/create-cf-2.png',

  },
]} />

### Metric

Click **Add metric** and configure the aggregation:

- **Metric name**: `totalWaterConsumption`
- **Aggregation**: `Sum`
- **Value source**: `Key`
- **Argument name**: `waterConsumption`

- **Output type**: `Time series`
- **Strategy**: `Process right away`

## Step 4. Verify the result

Send `waterConsumption` telemetry to both meters. Replace `$ACCESS_TOKEN` with the device access token.

**Water Meter A1:**

<HostCode product={props.product} lang="bash" code={`curl -v -X POST -d '{"waterConsumption": 2.75}' https://{HTTP_HOST}/api/v1/$ACCESS_TOKEN/telemetry --header "Content-Type:application/json"`} />

**Water Meter A2:**

<HostCode product={props.product} lang="bash" code={`curl -v -X POST -d '{"waterConsumption": 6.25}' https://{HTTP_HOST}/api/v1/$ACCESS_TOKEN/telemetry --header "Content-Type:application/json"`} />
