Configure ThingsBoard to automatically send a <a href="/docs/mobile/" bold={false} target="_blank">mobile app</a> push notification when an alarm is created. This guide extends the <DocLink product={props.product} path="recipes/create-clear-alarms" bold={false} target="_blank">Create and Clear Alarms</DocLink> recipe by using the Notification Center to deliver alarm-based push notifications to the ThingsBoard Mobile App.

## Use case

Assume your refrigeration equipment has a temperature sensor that periodically publishes telemetry to ThingsBoard. The normal operating range is **2 °C – 5 °C**. When a reading goes outside this range:

- An alarm is created by the alarm rules.
- A push notification is sent to the responsible users via the ThingsBoard Mobile App.

## Prerequisites

Before proceeding:

- Complete the <DocLink product={props.product} path="recipes/create-clear-alarms" bold={false} target="_blank">Create and Clear Alarms</DocLink> recipe — the alarm rules configured there are required for this guide.
- Install the <DocLink product={Products.CE} path={props.product === Products.CE ? 'mobile' : 'mobile/pe'} bold={false} target="_blank">ThingsBoard Mobile App</DocLink> on the target device and log in as a Tenant Administrator.
- Enable push notifications for the app in your device settings.

Review the following documentation for background:

- <DocLink product={props.product} path="user-guide/notifications" bold={false} target="_blank">Notification Center</DocLink> — how recipients, templates, and rules work together.
- <DocLink product={props.product} path="user-guide/alarm-rules" bold={false} target="_blank">Alarm rules</DocLink> — how alarms are created and cleared.

## Configure Notification Center

The Notification Center delivers notifications to users. It consists of three components:

- **Recipients** — define who receives notifications.
- **Templates** — define the notification content and delivery method.
- **Rules** — define the conditions under which notifications are sent.

### Step 1. Configure notification recipients

Mobile app push notifications are delivered to Platform users who have the <DocLink product={Products.CE} path={props.product === Products.CE ? 'mobile' : 'mobile/pe'} bold={false} target="_blank">ThingsBoard Mobile App</DocLink> installed and logged in. The built-in **Tenant administrators** recipient group targets all tenant-level administrators and requires no additional configuration.

<Steps>
1. Navigate to **Notification Center &#8702; Recipients**. For this example, use the built-in **Tenant administrators** recipient group, which includes all tenant-level administrators and requires no additional configuration.
2. Confirm the **Tenant administrators** group exists. If you need a different audience, click **+ Add recipients** and create a new Platform users group with the appropriate user filter.
</Steps>

### Step 2. Create notification template

Create a template that defines the push notification subject and message using alarm variables.

<Steps>
1. Navigate to **Notification Center &#8702; Templates**.
2. Click **+ Add template** in the top-right corner.
3. **Setup** — configure the template:
   - **Name**: `Alarm notification to mobile app`
   - **Type**: Alarm
   - **Delivery methods**: toggle **Mobile app** on
4. Click **Next**.
5. **Compose** — expand the **Mobile app** section and set:
   - **Subject**: 
      ```
      ${alarmType} on ${alarmOriginatorName} device
      ```
   - **Message**: 
      ```
      A ${alarmType} has been detected on the ${alarmOriginatorName} device: ${details.data} °C.
      ```
6. Click **Add**.
</Steps>

Template variables used:

- `${alarmType}` — the alarm type name (e.g. *High temperature*).
- `${alarmOriginatorName}` — the name of the device that triggered the alarm.
- `${details.data}` — the reported telemetry value, populated by the alarm rule.

### Step 3. Create notification rule

Create a rule that links the alarm trigger to the template and recipients.

<Steps>
1. Navigate to **Notification Center &#8702; Rules**.
2. Click **+ Add rule** in the top-right corner.
3. **Basic settings** — configure the rule:
   - **Rule name**: `New alarm to mobile app`
   - **Trigger**: Alarm
   - **Template**: <DocLink product={props.product} path="recipes/send-mobile-app-alarm/#step-2-create-notification-template" bold={false}>Alarm notification to mobile app</DocLink>
   - **Escalation chain &#8702; First recipient**: <DocLink product={props.product} path="recipes/send-mobile-app-alarm/#step-1-configure-notification-recipients" bold={false}>Tenant administrators</DocLink>
4. Click **Next**.
5. **Alarm trigger settings** — configure the filter:
   - **Alarm type list**: Any type
   - **Alarm severity list**: Any severity
   - **Notify on**: Alarm created
6. Click **Add**.
</Steps>

<Aside type="note">
  Push notifications are sent only when an alarm is **created**, not when an existing alarm is updated.
</Aside>

## Verification

Publish a temperature value that triggers the alarm (above 5 °C).  
Use the <DocLink product={props.product} path="user-guide/devices/#check-connectivity"  bold={false} target="_blank">Check Connectivity</DocLink> feature or the command below:

<Aside type="note">
  {(props.product === Products.PAAS || props.product === Products.PAAS_EU) ? (
    <>Replace <code>$ACCESS_TOKEN</code> with the device access token.</>
  ) : (
    <>Replace <code>$THINGSBOARD_HOST_NAME</code> with your ThingsBoard host (e.g. <code>localhost:8080</code>) and <code>$ACCESS_TOKEN</code> with the device access token.</>
  )}
</Aside>

<Code code={`curl -v -X POST \\
  --header "Content-Type: application/json" \\
  --data '{"temperature": 25}' \\
  http://${tbHost(props.product)}/api/v1/$ACCESS_TOKEN/telemetry`} lang="bash" />

A new alarm is created and the push notification is sent immediately to all Tenant administrators with the mobile app installed. The subject and message are populated from the template variables.
