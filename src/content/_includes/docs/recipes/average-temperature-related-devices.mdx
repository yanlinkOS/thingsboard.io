Use a <DocLink product={props.product} path="user-guide/calculated-fields/related-entities-aggregation">Related Entities Aggregation</DocLink> calculated field to collect temperature readings from multiple sensors and store the average on a parent asset.

## Use case

**Warehouse A** contains two thermometer sensors. Each reports a `temperature` telemetry key.

ThingsBoard:
- finds all devices related to **Warehouse A** via **Contains** relations
- averages the latest `temperature` reading from each sensor
- stores the result as `avgTemperature` on **Warehouse A**

## Step 1. Create devices and asset

Create an asset to represent the warehouse:
- **Name**: `Warehouse A`
- **Asset profile**: `warehouse`

Create two devices with the **thermometer** device profile:
- **Name**: `Sensor A1`
- **Name**: `Sensor A2`

## Step 2. Create relations

Link both sensors to the warehouse using outbound **Contains** relations:

<Steps>
1. Open **Warehouse A** details.
2. Go to the **Relations** tab.
3. Add an outbound **Contains** relation to **Sensor A1**.
4. Add an outbound **Contains** relation to **Sensor A2**.
</Steps>

<Aside type="note">
  The relation direction and type must match the aggregation path configured in the calculated field.
</Aside>

## Step 3. Create calculated field

You can import a ready-made calculated field or build it manually.

**Option 1. Import calculated field**

- Download the **Warehouse Average Temperature** calculated field as <a href="/resources/docs/recipes/average-temperature-related-devices/warehouse_average_temperature_cf.json" download="warehouse_average_temperature_cf.json"><strong>JSON file</strong></a> and <DocLink product={props.product} path="user-guide/calculated-fields/#export--import" target="_blank">import</DocLink> it into your instance.

<Aside type="note">
  During import, set the <strong>Warehouse A</strong> asset as the target entity.
</Aside>

**Option 2. Create manually**

- Navigate to **Calculated fields** and click **+ Add calculated field**.

### General settings

- **Title**: `Warehouse Average Temperature`
- **Entity type**: `Asset`
- **Entity**: `Warehouse A`
- **Type**: `Related entities aggregation`
- **Aggregation path**:
  - **Relation direction**: `Down to child`
  - **Relation type**: `Contains`

## Step 4. Verify the result

Post `temperature` telemetry to each device. Replace `$ACCESS_TOKEN` with the device access token.

**Sensor A1:**

<HostCode product={props.product} lang="bash" code={`curl -v -X POST -d '{"temperature": 21.06}' https://{HTTP_HOST}/api/v1/$ACCESS_TOKEN/telemetry --header "Content-Type:application/json"`} />

**Sensor A2:**

<HostCode product={props.product} lang="bash" code={`curl -v -X POST -d '{"temperature": 23.84}' https://{HTTP_HOST}/api/v1/$ACCESS_TOKEN/telemetry --header "Content-Type:application/json"`} />

The computed field reads the latest value from each related device, calculates the average, and stores it as the totalWaterConsumption time-series key for **Warehouse A**.

**Expected result:** Open **Warehouse A &#8702; Latest telemetry** tab. 
The `avgTemperature` value is `22.5` (average of 21.06 and 23.84, rounded to 1 decimal), updated automatically whenever new data arrives from either sensor.
