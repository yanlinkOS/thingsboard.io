This recipe shows how to handle a client-side RPC call from a device by fetching telemetry from a related device and returning it as the RPC response.

## Use case

A **Controller** sends a client-side RPC call with method `getCurrentTemperature`.

ThingsBoard:
- routes the request to a dedicated rule chain
- fetches the latest `temperature` from a related **Thermostat** device
- returns the temperature value as the RPC response

## Architecture

**Devices**
- **Thermostat A** with type **thermostat**
- **Controller A** with type **controller**

**Relation**
- **Controller A** -- **Thermostat** --> **Thermostat A**

**Rule chains**
- modified **Root Rule Chain**
- new **Related thermostat temperature** rule chain

## Step 1. Create devices

Create two devices:

**Device 1**
- **Name**: `Thermostat A`
- **Type**: `thermostat`

**Device 2**
- **Name**: `Controller A`
- **Type**: `controller`

## Step 3. Create rule chain

You can import a ready-made rule chain or build it manually.

**Option 1. Import rule chain**

- Download the **Related Thermostat Temperature** rule chain as a <a href="/resources/docs/recipes/rpc-reply-with-related-telemetry/related_thermostat_temperature_rule_chain.json" download="related_thermostat_temperature_rule_chain.json"><strong>JSON file</strong></a> and <DocLink product={props.product} path="user-guide/rule-engine/#import-a-rule-chain" target="_blank">import</DocLink> it into your instance.

**Option 2. Create manually**

- <DocLink product={props.product} path="user-guide/rule-engine/#create-a-rule-chain" target="_blank">Create a new rule chain</DocLink> named `Related Thermostat Temperature`, open it, and add the following nodes:

### Step 3.1 Related Entity Data node

Add the <DocLink product={props.product} path="reference/rule-engine/nodes/enrichment/related-entity-data" target="_blank">Related Entity Data</DocLink> node and connect it to the **Input** node.

This node fetches the latest `temperature` from **Thermostat A** and saves it in message metadata as `roomTemperature`.

Fill in the fields:
- **Name**: `Get Temperature from Related Thermostat`
- **Direction**: `From originator`
- **Max relation level**: `1`
- **Relation filters**:
  - **Relation type**: `Uses`
  - **Entity type**: `Device`
- **Data to fetch**: `Latest telemetry`
- **Latest telemetry mapping**:
  - **Source attribute key**: `temperature`
  - **Target key**: `roomTemperature`

### Step 3.2 Script Transformation node

Add the <DocLink product={props.product} path="reference/rule-engine/nodes/transformation/script" target="_blank">Transformation Script</DocLink> node and connect it to the **Related Entity Data** node with relation type **Success**.

This node builds the RPC reply payload. The **RPC call reply** node sends the message payload as the response, so the payload must contain the temperature value.

Fill in the fields:
- **Name**: `Build Response`

Script:

<Tabs>
  <TabItem label="TBEL">
    ```javascript
    return {msg: {temperature: metadata.roomTemperature}, metadata: metadata, msgType: msgType};
    ```
  </TabItem>
  <TabItem label="JavaScript">

  </TabItem>
</Tabs>

### Step 3.3 RPC call reply node

Add the <DocLink product={props.product} path="reference/rule-engine/nodes/action/rpc-call-reply" target="_blank">RPC call reply</DocLink> node and connect it to the **Transformation Script** node with relation type **Success**.

This node reads `requestId` from message metadata and sends the message payload as the RPC response to the originator.

Fill in the fields:
- **Name**: `Send Response`
- **Request ID**: `requestId`

## Step 4. Modify the Root Rule Chain

Open the **Root Rule Chain**. By default, the <DocLink product={props.product} path="reference/rule-engine/nodes/filter/message-type-switch" target="_blank">Message Type Switch</DocLink> node routes **RPC Request from Device** messages to a <DocLink product={props.product} path="reference/rule-engine/nodes/action/log" target="_blank">Log</DocLink> node named **Log RPC from Device**.

Add the following nodes after **Message Type Switch**:

### Step 4.1 Filter Script node

Add the <DocLink product={props.product} path="reference/rule-engine/nodes/filter/script" target="_blank">Filter Script</DocLink> node and connect it to the **Message Type Switch** node with relation type **RPC Request from Device**, replacing the existing connection to **Log RPC from Device**.

This node passes only requests with method `getCurrentTemperature`.

Fill in the fields:
- **Name**: `Filter getCurrentTemperature RPC`

Script:

<Tabs>
  <TabItem label="TBEL">
    ```javascript
    return msg.method == 'getCurrentTemperature';
    ```
  </TabItem>
  <TabItem label="JavaScript">
    ```javascript
    return msg.method == 'getCurrentTemperature';
    ```
  </TabItem>
</Tabs>

<MultiProductImageGallery product={props.product} images={[

### Step 4.2 Rule Chain node

Add the <DocLink product={props.product} path="reference/rule-engine/nodes/flow/rule-chain" target="_blank">Rule Chain</DocLink> node and connect it to the **Filter Script** node with relation type **True**.

This node forwards matched requests to the **Related thermostat temperature** rule chain.

Fill in the fields:
- **Name**: `To Related Thermostat Temperature`
- **Rule Chain**: `Related Thermostat Temperature`

### Step 4.3 Log node

Connect the existing **Log RPC from Device** node to the **Filter Script** node with relation type **False**.

Requests with an unrecognized method are logged instead of dropped.

<Aside type="note">
  Do not delete the existing <strong>Log RPC from Device</strong> node — reconnect it from the <strong>False</strong> output of the filter node.
</Aside>
