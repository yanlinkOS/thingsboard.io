The MQTT Attributes API covers three operations: publishing client-side attributes to ThingsBoard, requesting current attribute values from the server, and subscribing to server-pushed attribute updates. See <DocLink product={props.product} path='TODO'>Attributes</DocLink> for the difference between client-side, server-side, and shared attributes, and <DocLink product={props.product} path='reference/mqtt-api/getting-connected'>Getting Connected</DocLink> for connection parameters and credential types.

## Publish Client-Side Attributes

Report device state (serial number, firmware version, hardware config) to ThingsBoard.

**Topic:** `v1/devices/me/attributes`

```json
{"firmwareVersion": "2.1.0", "serialNumber": "SN-4A21F", "hardwareRevision": "B"}
```

```bash
mosquitto_pub -d -q 1 -h "$THINGSBOARD_HOST" -p 1883 -t "v1/devices/me/attributes" -u "$ACCESS_TOKEN" -m '{"firmwareVersion": "2.1.0", "serialNumber": "SN-4A21F"}'
```

## Request Attribute Values from Server

Ask ThingsBoard for the current values of client-side or shared attributes. Useful on device boot to read back configuration stored in ThingsBoard.

**Subscribe to responses:** `v1/devices/me/attributes/response/+`

**Publish request to:** `v1/devices/me/attributes/request/$request_id`

Request payload — specify which keys you want:

```json
{"clientKeys": "firmwareVersion,serialNumber", "sharedKeys": "targetTemperature,enabled"}
```

Response:

```json
{
  "client": {"firmwareVersion": "2.1.0", "serialNumber": "SN-4A21F"},
  "shared": {"targetTemperature": 24, "enabled": true}
}
```

`$request_id` is any integer you choose; ThingsBoard echoes it in the response topic so you can match replies to requests.

## Subscribe to Shared Attribute Updates

Receive pushed notifications whenever a shared attribute is changed from the ThingsBoard side (e.g. an operator changes a setpoint in the dashboard).

**Subscribe to:** `v1/devices/me/attributes`

ThingsBoard publishes only the changed keys:

```json
{"targetTemperature": 26}
```

```bash
mosquitto_sub -d -q 1 -h "$THINGSBOARD_HOST" -p 1883 -t "v1/devices/me/attributes" -u "$ACCESS_TOKEN"
```

<Aside type="tip">
  Subscribe to this topic at startup so the device immediately picks up any configuration changes made while it was offline.
</Aside>
