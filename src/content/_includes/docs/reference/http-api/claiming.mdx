Device Claiming lets an end user securely associate a deployed device with their account. For a full explanation of the claiming workflow, see <DocLink product={props.product} path='user-guide/claiming'>Claiming Devices</DocLink>.

See <DocLink product={props.product} path='reference/http-api/getting-connected'>Getting Connected</DocLink> for authentication and connection details.

## Claiming Request

The device initiates claiming by sending a `POST` request:

<HostCode product={props.product} lang="" code={`POST {HTTP_URL_PREFIX}/api/v1/$ACCESS_TOKEN/claim`} />

```json
{"secretKey": "mySecret", "durationMs": 60000}
```

<HostCode product={props.product} lang="bash" code={`curl -s -X POST -H "Content-Type: application/json" -d '{"secretKey":"mySecret","durationMs":60000}' "http://{HTTP_HOST}/api/v1/$ACCESS_TOKEN/claim"`} />

| Field | Required | Description |
|-------|----------|-------------|
| `secretKey` | No | Secret used to authorize the claim. Defaults to an empty string if omitted. |
| `durationMs` | No | Time window in milliseconds during which the device can be claimed. Defaults to the system value of `SECURITY_CLAIM_DURATION`. |

<Aside type="note">
  The device must be connected and able to send HTTP requests before initiating a claim.
</Aside>
