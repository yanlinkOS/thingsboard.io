The HTTP Attributes API lets devices upload client-side attributes, request attribute values, and poll for shared attribute updates. For an explanation of attribute types, see <DocLink product={props.product} path='user-guide/digital-twins/attributes'>Attributes</DocLink>.

See <DocLink product={props.product} path='reference/http-api/getting-connected'>Getting Connected</DocLink> for authentication and connection details.

## Publish Client-Side Attributes

Upload client-side attributes with a `POST` request:

<HostCode product={props.product} lang="" code={`POST {HTTP_URL_PREFIX}/api/v1/$ACCESS_TOKEN/attributes`} />

```json
{"attribute1": "value1", "attribute2": true, "attribute3": 42.0}
```

<HostCode product={props.product} lang="bash" code={`curl -s -X POST -H "Content-Type: application/json" -d '{"attribute1":"value1","attribute2":true,"attribute3":42.0}' "http://{HTTP_HOST}/api/v1/$ACCESS_TOKEN/attributes"`} />

## Request Attribute Values

Request client-side and shared attribute values with a `GET` request:

<HostCode product={props.product} lang="" code={`GET {HTTP_URL_PREFIX}/api/v1/$ACCESS_TOKEN/attributes?clientKeys=attr1,attr2&sharedKeys=shared1,shared2`} />

| Query parameter | Description |
|-----------------|-------------|
| `clientKeys` | Comma-separated list of client-side attribute keys to return |
| `sharedKeys` | Comma-separated list of shared attribute keys to return |

<HostCode product={props.product} lang="bash" code={`curl -s "http://{HTTP_HOST}/api/v1/$ACCESS_TOKEN/attributes?clientKeys=attribute1,attribute2&sharedKeys=shared1,shared2"`} />

Response:

```json
{"client": {"attribute1": "value1", "attribute2": true}, "shared": {"shared1": "value2", "shared2": false}}
```

<Aside type="note">
  Avoid using the same key names for client-side and shared attributes — overlapping keys lead to ambiguous responses.
</Aside>

## Subscribe to Shared Attribute Updates

Poll for shared attribute changes with a long-lived `GET` request:

<HostCode product={props.product} lang="" code={`GET {HTTP_URL_PREFIX}/api/v1/$ACCESS_TOKEN/attributes/updates?timeout=20000`} />

<HostCode product={props.product} lang="bash" code={`curl -s "http://{HTTP_HOST}/api/v1/$ACCESS_TOKEN/attributes/updates?timeout=20000"`} />

The request blocks until a shared attribute changes or the timeout elapses. The `timeout` parameter is in milliseconds. When an update occurs, ThingsBoard returns the changed attributes:

```json
{"shared": {"firmware_version": "2.1.0"}}
```
