The **Radial gauge** is a general-purpose circular dial widget with no preset units, title, or scale colors. It renders a full-arc gauge face with a sweeping needle, configurable tick range, and a digital value readout at the bottom — ready to represent any numeric telemetry key out of the box.

Use it when none of the purpose-built gauges fit your measurement — for example, displaying pressure in bar, humidity in percent, battery voltage, tank fill level, or any other sensor reading where you want the familiar, instantly-readable look of an analogue dial without being locked into a specific visual theme.

</div>

## Adding the widget

<Steps>
1. Open the dashboard in edit mode. Click **Add widget** in the top toolbar, or click the **Add new widget** icon in the center of an empty dashboard.
2. In the widget bundle selection dialog, find and click **Analogue gauges**.
3. Select the **Radial gauge** widget.
4. Configure the datasource and appearance, then click **Add**.
</Steps>

## Configuration

All settings in this section use **Basic** mode — a quick-start configuration covering the essentials. For finer widget controls like title font, card style, and tooltip, switch to the **Advanced** tab.

### Datasource

Select the source of the value to display:

- **Device** — a specific device; the gauge displays the latest telemetry value from this device.
- **Entity alias** — a set of entities matched by an <DocLink product={props.product} path="user-guide/aliases" bold={false} target="_blank">entity alias</DocLink>; use this when the target device is selected dynamically at runtime.

Add exactly one data key — the <DocLink product={props.product} path="user-guide/digital-twins/time-series-data/" bold={false} target="_blank">telemetry key</DocLink> whose value the gauge will display.

### Gauge appearance

Controls the visual elements of the dial face.

| Setting | Description |
|---------|-------------|
| **Units title** | Toggle to show or hide a label displayed in the center of the dial. No default text is set — enter any label that describes the measurement (for example, `Pressure` or `Humidity`). Set color and font. |
| **Units** | The unit string displayed on the dial. No default is set — enter the appropriate unit for your data (for example, `bar`, `%`, or `V`). Set color and font. Click the settings icon to configure unit conversion — see <DocLink product={props.product} path="user-guide/units/" bold={false} target="_blank">Unit conversion</DocLink>. |
| **Value** | Toggle to show or hide the digital readout at the bottom of the dial. Set the number of digits and the text color and font. |
| **Ticks** | The minimum and maximum values displayed on the scale (default: `−100` to `100`). Adjust to match the expected range of your telemetry. Set tick label color and font. |
| **Arrow and scale default color** | The color of the needle and the scale arc. |
| **Plate color** | Background fill color of the gauge face. |
| **Scale colors** | Toggle to enable color zones on the scale arc. Disabled by default — when enabled, click the color swatch to open the **Color settings** dialog and define value ranges with individual colors. |

### Card appearance

| Setting | Description |
|---------|-------------|
| **Background color** | Solid color background for the widget card. |
| **Show card buttons** | Toggle **Data export** and **Fullscreen** buttons in the widget header. |
| **Card border radius** | Corner rounding in pixels. |

### Actions

The Radial gauge supports the <DocLink product={props.product} path="user-guide/actions/#widget-header-button" bold={false} target="_blank">Widget header button</DocLink> action — a button placed in the widget header that triggers a configured action when clicked.

## Troubleshooting

<p style="font-size: 1.5em; font-weight: bold;">Gauge shows no data</p>

| Cause | Solution |
|-------|----------|
| Wrong data key name | Confirm the key name matches the telemetry key sent by the device. Key names are case-sensitive. |
| Device not sending data | Open **Devices → Latest telemetry** and check for recent values. If empty, verify the device transport configuration and access token. |
| Entity alias resolves to no entities | Open **Entity aliases** in the dashboard and verify the alias returns results. |

<p style="font-size: 1.5em; font-weight: bold;">Needle is pegged at the minimum or maximum</p>

| Cause | Solution |
|-------|----------|
| Ticks range too narrow | Adjust the **Ticks** min and max values to cover the actual range reported by the device. |
| Wrong telemetry key selected | Confirm the data key matches the key your device actually reports. |

<p style="font-size: 1.5em; font-weight: bold;">Value shows as N/A or stops updating</p>

| Cause | Solution |
|-------|----------|
| Device has stopped sending telemetry | Check device connectivity and the last activity timestamp in **Devices → Latest telemetry**. |
| Wrong datasource entity selected | Verify the correct device or alias is selected in the datasource configuration. |
