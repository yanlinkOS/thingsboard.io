Use this node when you need to route a message to one or more named connections based on custom logic ÔÇö for example, routing to `High`, `Normal`, or `Low` based on a sensor reading, or fanning a single message out to multiple parallel paths simultaneously. The script returns a connection name or an array of names.

## Configuration

- **Script language** ÔÇö required. Choose `TBEL` or `JavaScript`.
- **Script body** ÔÇö required. A function that receives three arguments and must return either a string or an array of strings (each string is a connection label):
  - `msg` ÔÇö the message data (object or array).
  - `metadata` ÔÇö the message metadata. In JavaScript, an object where all values are strings; in TBEL, a `java.util.Map<String, String>`.
  - `msgType` ÔÇö the message type as a string.

<Aside type="note">
  Scripts run in a sandboxed environment. Importing or calling external libraries is not supported ÔÇö all logic must be self-contained within the script body.
</Aside>

## Message processing algorithm

1. Select the script based on the configured **Script language**.
2. Execute the script function.
3. Interpret the return value:
   - A string Ôćĺ treated as a one-element list; route the message via that connection.
   - An array of strings Ôćĺ route the message via each listed connection.
   - Any other type, or an exception Ôćĺ route via `Failure`.

## Output connections

| Connection | Condition |
|------------|-----------|
| *(any name returned by the script)* | The message is routed via every connection name the script returns. |
| `Failure` | The script returned an invalid type (e.g., number, object) or threw an exception. |

## Examples

### Example 1 ÔÇö Single string return Ôćĺ `A`

```json maxLines=10 collapsible
{
  "scriptLang": "TBEL",
  "tbelScript": "return 'A';"
}
```

**Result**: routed via **`A`** ÔÇö a string is treated as a one-element list.

---

### Example 2 ÔÇö One-element array Ôćĺ `A`

```json maxLines=10 collapsible
{
  "scriptLang": "TBEL",
  "tbelScript": "return ['A'];"
}
```

**Result**: routed via **`A`**.

---

### Example 3 ÔÇö Multiple connections Ôćĺ `A`, `B`, `C`

```json maxLines=10 collapsible
{
  "scriptLang": "TBEL",
  "tbelScript": "return ['A', 'B', 'C'];"
}
```

**Result**: the message is forwarded to all three connections simultaneously.

---

### Example 4 ÔÇö Dynamic routing by temperature Ôćĺ `High`

**Incoming message**

Data:
```json maxLines=10 collapsible
{
  "temperature": 85
}
```

**Node configuration**

```json maxLines=15 collapsible
{
  "scriptLang": "TBEL",
  "tbelScript": "var t = msg.temperature;\nif (t > 80) return 'High';\nif (t > 60) return 'Normal';\nreturn 'Low';"
}
```

**Result**: routed via **`High`** ÔÇö temperature (85) exceeds 80.
