Fetches attributes and/or latest telemetry from the message originator and adds them to the message data or metadata.

---

## Configuration

| Field | Required | Description |
|-------|----------|-------------|
| **Client attributes** | No | List of client-side attribute keys to fetch |
| **Shared attributes** | No | List of shared attribute keys to fetch |
| **Server attributes** | No | List of server-side attribute keys to fetch |
| **Latest telemetry** | No | List of telemetry keys to fetch (latest value only) |
| **Fetch to** | Yes | Where to place the fetched values: **Metadata** or **Data** |
| **Get latest value with timestamp** | No | When enabled, telemetry values are stored as `{"ts": <ms>, "value": <v>}` JSON strings instead of plain values |
| **Tell Failure if absent** | No | When enabled, routes via `Failure` if any requested key is missing from the originator |

<Aside type="note">
  At least one key list (client, shared, server, or latest telemetry) must be non-empty.
</Aside>

---

## Key naming in metadata

When **Fetch to** is set to **Metadata**, fetched values are added with prefixed key names:

| Attribute type | Metadata key prefix | Example |
|---------------|---------------------|---------|
| Client-side | `cs_` | `cs_temperature` |
| Shared | `shared_` | `shared_threshold` |
| Server-side | `ss_` | `ss_region` |
| Latest telemetry | *(no prefix)* | `temperature` |

When **Fetch to** is set to **Data**, all keys are added without any prefix.

---

## Output connections

| Connection | Condition |
|------------|-----------|
| `Success` | All requested keys were fetched and added to the message |
| `Failure` | A key was missing and **Tell Failure if absent** is enabled, or an unexpected error occurred |

---

## Examples

### Example 1 — Fetch server attribute to metadata

**Configuration:** Server attributes = `region`; Fetch to = Metadata.

**Originator:** device with server-side attribute `region = "EU-West"`.

**Result:** `metadata.ss_region = "EU-West"` is added to the message.

---

### Example 2 — Fetch latest telemetry with timestamp to data

**Configuration:** Latest telemetry = `temperature`; Fetch to = Data; Get latest value with timestamp = enabled.

**Originator:** device with latest telemetry `temperature = 22.5` at timestamp `1700000000000`.

**Result:** `data.temperature = '{"ts":1700000000000,"value":22.5}'` is added to the message.
