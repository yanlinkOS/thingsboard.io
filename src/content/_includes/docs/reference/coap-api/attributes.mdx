The CoAP Attributes API lets devices upload client-side attributes, request attribute values, and subscribe to shared attribute updates using the CoAP Observe option. For an explanation of attribute types, see <DocLink product={props.product} path='user-guide/digital-twins/attributes'>Attributes</DocLink>.

See <DocLink product={props.product} path='reference/coap-api/getting-connected'>Getting Connected</DocLink> for credential types and connection details.

## Publish Client-Side Attributes

Send a `POST` request to upload client-side attributes:

| Credential type | URL |
|-----------------|-----|
| Access Token | <HostInline product={props.product} code="coap(s)://{COAP_HOST_LABEL}:5683/api/v1/$ACCESS_TOKEN/attributes" /> |
| X.509 Certificate | <HostInline product={props.product} code="coaps://{COAP_HOST_LABEL}/api/v1/attributes" /> |

```json
{"attribute1": "value1", "attribute2": true, "attribute3": 42.0}
```

**Access Token:**

<HostCode product={props.product} lang="bash" code={`coap-client -v 6 -m POST -t "application/json" -e '{"attribute1":"value1","attribute2":true,"attribute3":42.0}' "coap://{COAP_HOST}:5683/api/v1/$ACCESS_TOKEN/attributes"`} />

**X.509 Certificate:**

<HostCode product={props.product} lang="bash" code={`coap-client-openssl -v 6 -c cert.pem -j key.pem -m POST -t "application/json" -e '{"attribute1":"value1","attribute2":true,"attribute3":42.0}' "coaps://{COAP_HOST}/api/v1/attributes"`} />

## Request Attribute Values

Send a `GET` request with `clientKeys` and `sharedKeys` query parameters:

| Credential type | URL |
|-----------------|-----|
| Access Token | <HostInline product={props.product} code="coap(s)://{COAP_HOST_LABEL}:5683/api/v1/$ACCESS_TOKEN/attributes?clientKeys=attr1,attr2&sharedKeys=shared1,shared2" /> |
| X.509 Certificate | <HostInline product={props.product} code="coaps://{COAP_HOST_LABEL}/api/v1/attributes?clientKeys=attr1,attr2&sharedKeys=shared1,shared2" /> |

<Aside type="note">
  Use `coap-client` (not `coap-cli`) — the `coap-cli` npm tool does not support query parameters.
</Aside>

**Access Token:**

<HostCode product={props.product} lang="bash" code={`coap-client -v 6 -m GET "coap://{COAP_HOST}:5683/api/v1/$ACCESS_TOKEN/attributes?clientKeys=attribute1,attribute2&sharedKeys=shared1,shared2"`} />

Response:

```json
{"client": {"attribute1": "value1", "attribute2": true}, "shared": {"shared1": "value2", "shared2": false}}
```

When the device profile payload type is set to Protobuf, the request and response use fixed schemas. See <DocLink product={props.product} path='reference/mqtt-api/attributes#request-attribute-values-from-server'>MQTT Attributes</DocLink> for the Protobuf schemas.

<Aside type="note">
  Avoid using the same key names for client-side and shared attributes — overlapping keys lead to ambiguous responses.
</Aside>

## Subscribe to Shared Attribute Updates

Use the CoAP **Observe** option to receive push notifications when shared attributes change. The connection stays open and ThingsBoard sends a response each time a shared attribute is updated.

| Credential type | URL |
|-----------------|-----|
| Access Token | <HostInline product={props.product} code="coap(s)://{COAP_HOST_LABEL}:5683/api/v1/$ACCESS_TOKEN/attributes/updates" /> |
| X.509 Certificate | <HostInline product={props.product} code="coaps://{COAP_HOST_LABEL}/api/v1/attributes/updates" /> |

**Access Token (observe for 60 seconds):**

<HostCode product={props.product} lang="bash" code={`coap-client -v 6 -m GET -s 60 "coap://{COAP_HOST}:5683/api/v1/$ACCESS_TOKEN/attributes/updates"`} />

**X.509 Certificate:**

<HostCode product={props.product} lang="bash" code={`coap-client-openssl -v 6 -c cert.pem -j key.pem -m GET -s 60 "coaps://{COAP_HOST}/api/v1/attributes/updates"`} />

When a shared attribute changes, ThingsBoard pushes the updated values:

```json
{"shared": {"firmware_version": "2.1.0"}}
```

When the device profile payload type is set to Protobuf, notifications use a fixed schema. See <DocLink product={props.product} path='reference/mqtt-api/attributes#subscribe-to-shared-attribute-updates'>MQTT Attributes</DocLink> for the Protobuf schema.
