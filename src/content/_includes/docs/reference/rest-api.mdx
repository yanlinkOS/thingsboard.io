ThingsBoard provides a REST API for managing platform entities, integrations, device provisioning, and building custom applications.
The API is available on every ThingsBoard instance and documented interactively via Swagger UI.

## Swagger UI

Swagger UI lets you browse all available REST API endpoints, inspect request and response schemas, and test API calls directly in your browser.

Access it at:

```
http://THINGSBOARD_HOST:PORT/swagger-ui.html
```

If you are already logged into the ThingsBoard UI, Swagger UI authenticates automatically using your session.

## Authentication

### API key (recommended)

<Aside type="tip">
Available since ThingsBoard 4.3. See the <DocLink product={props.product} path='user-guide/security/api-keys'>API Keys</DocLink> guide to create and manage keys.
</Aside>

Pass the API key in the `X-Authorization` header:

```
X-Authorization: ApiKey YOUR_API_KEY_VALUE
```

Example request:

```bash
curl -X GET --header 'Accept: application/json' \
            --header 'X-Authorization: ApiKey YOUR_API_KEY_VALUE' \
            'http://THINGSBOARD_URL/api/auth/user'
```

API keys are long-lived, require no login, and can be revoked through the ThingsBoard UI.

### JWT token (deprecated)

Obtain an access token by sending credentials to the login endpoint:

```bash
curl -X POST --header 'Content-Type: application/json' \
             --header 'Accept: application/json' \
             -d '{"username":"[email protected]", "password":"tenant"}' \
             'http://THINGSBOARD_URL/api/auth/login'
```

Response:

```json
{"token":"YOUR_JWT_TOKEN", "refreshToken":"YOUR_JWT_REFRESH_TOKEN"}
```

Use the token in subsequent requests:

```
X-Authorization: Bearer YOUR_JWT_TOKEN
```

Access tokens are valid for 2.5 hours; refresh tokens for 1 week.

<Aside type="note">
If both API key and JWT are provided, the JWT token takes priority.
</Aside>
