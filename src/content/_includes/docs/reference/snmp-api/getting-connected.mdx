Simple Network Management Protocol (SNMP) is a standard protocol for collecting and modifying management data on network devices. ThingsBoard acts as an **SNMP manager** that periodically polls SNMP agents running on your devices, reading OID values via GET requests and writing values via SET requests.

ThingsBoard supports SNMP v1, v2c, and v3. Messages are transported over UDP (default) or TCP, configured via the `SNMP_UNDERLYING_PROTOCOL` environment variable. The default SNMP bind port is **1620** (`SNMP_BIND_PORT`).

```plaintext
  ┌──────────────────────┐         GET / SET         ┌──────────────────────┐
  │   ThingsBoard        │ ◀─────────────────────▶   │   SNMP Agent         │
  │   (SNMP Manager)     │        SNMP v1/v2c/v3     │   (Network Device)   │
  │                      │                            │                      │
  │  Polls OIDs at a     │                            │  Exposes MIB with    │
  │  configured interval │                            │  OID → value pairs   │
  └──────────────────────┘                            └──────────────────────┘
```

Each OID (Object Identifier) addresses a specific variable in the device's Management Information Base (MIB). ThingsBoard maps OID values to telemetry keys or attributes based on the SNMP Device Profile configuration.

## Device profile configuration

The SNMP Device Profile defines how ThingsBoard communicates with SNMP devices: request timeout, retry count, and communication configs (telemetry polling, attribute polling, shared attribute updates, RPC mappings).

<Steps>
1. Go to **Device profiles** and create or edit a profile. Set the transport type to **SNMP**.
2. Configure **Request timeout** (milliseconds before a request is resent or timed out) and **Retries** (number of retry attempts before timeout).
</Steps>

## Device configuration

Each SNMP device requires a **host** and **port** regardless of the SNMP version.

### SNMP v1 and v2c

For v1 and v2c, set a **community string** (a plaintext password sent with each request).

### SNMP v3

SNMP v3 uses the User-based Security Model (USM) with `authPriv` security level, which enables both authentication and encryption. Configure the following properties:

| Property | Description |
|----------|-------------|
| **Username** | USM user name |
| **Security name** | Security identity for authentication |
| **Context name** | Context scope for access control |
| **Authentication protocol** | Hash function for the authentication passphrase: SHA-1, SHA-224, SHA-256, SHA-384, SHA-512, or MD5 |
| **Authentication passphrase** | Secret used to verify message integrity |
| **Privacy protocol** | Encryption algorithm: DES (CBC mode) or AES-128, AES-192, AES-256 |
| **Privacy passphrase** | Secret used for data encryption |
| **Engine ID** | Unique identifier of the SNMP agent engine |

## Transport configuration

The SNMP transport service is configured via environment variables. Key parameters:

| Variable | Default | Description |
|----------|---------|-------------|
| `SNMP_ENABLED` | `true` | Enable or disable the SNMP transport |
| `SNMP_BIND_PORT` | `1620` | SNMP bind port |
| `SNMP_UNDERLYING_PROTOCOL` | `udp` | Transport protocol: `udp` or `tcp` |
| `SNMP_MAX_REQUEST_OIDS` | `100` | Maximum OID mappings per PDU. Requests exceeding this limit are split into multiple PDUs. |
| `SNMP_REQUEST_CHUNK_DELAY_MS` | `100` | Delay in milliseconds between sending split PDU chunks |
| `SNMP_RESPONSE_IGNORE_TYPE_CAST_ERRORS` | `false` | When `false`, throws an error if a response value does not match the configured data type. Set to `true` to silently ignore mismatched values. |
| `SNMP_RESPONSE_PROCESSING_PARALLELISM_LEVEL` | `4` | Thread pool size for processing SNMP responses |
| `SNMP_SCHEDULER_THREAD_POOL_SIZE` | `4` | Thread pool size for the scheduler that executes polling tasks |
