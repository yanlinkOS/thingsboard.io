The IoT Gateway supports [device provisioning](/docs/user-guide/provisioning/),
letting it **automatically register itself** on the ThingsBoard platform and obtain credentials on first
startup — no manual per-device credential setup required.

## How It Works

```plaintext
  ┌──────────────────────────────────────────────┐
  │  Gateway starts                              │
  └─────────────────────┬────────────────────────┘
                        │
                        ▼
  ┌──────────────────────────────────────────────┐
  │  Check: credentials stored locally?          │
  └──────────┬───────────────────────┬───────────┘
             │ Yes                   │ No
             ▼                       ▼
  ┌──────────────────┐   ┌────────────────────────┐
  │  Connect using   │   │  Send provision        │
  │  stored creds    │   │  request (key+secret)  │
  └──────────────────┘   └───────────┬────────────┘
                                     │
                                     ▼
                         ┌────────────────────────┐
                         │  Platform creates or   │
                         │  reuses device entity, │
                         │  generates credentials │
                         └───────────┬────────────┘
                                     │
                                     ▼
                         ┌────────────────────────┐
                         │  Credentials saved     │
                         │  to config dir         │
                         └───────────┬────────────┘
                                     │
                                     ▼
                         ┌────────────────────────┐
                         │  Connect to platform   │
                         └────────────────────────┘
```

If credentials need to be rotated or the authentication type changed, you can remove the stored
credentials and the gateway will re-provision on the next start.

## Environment Variables

Provisioning is configured via environment variables — typically set in `docker-compose.yml` or
your system environment.

| Variable | Required | Description |
|----------|----------|-------------|
| `TB_GW_PROVISIONING_DEVICE_KEY` | Yes | Provisioning device key (from the device profile) |
| `TB_GW_PROVISIONING_DEVICE_SECRET` | Yes | Provisioning device secret (from the device profile) |
| `TB_GW_PROVISIONING_DEVICE_NAME` | No | Name assigned to the device in ThingsBoard. A random name is generated if not set |
| `TB_GW_PROVISIONING_DEVICE_ACCESS_TOKEN` | No | If set, the platform assigns this token and uses **Access Token** authentication |
| `TB_GW_PROVISIONING_DEVICE_USERNAME` | No | MQTT Basic authentication username |
| `TB_GW_PROVISIONING_DEVICE_PASSWORD` | No | MQTT Basic authentication password |
| `TB_GW_PROVISIONING_DEVICE_CLIENT_ID` | No | MQTT Basic authentication client ID |
| `TB_GW_PROVISIONING_DEVICE_CA_CERT_FILENAME` | No | CA certificate filename in the `config/` folder. If set, the gateway generates a client certificate and uses **X.509 Certificate** authentication |

The credentials type is determined by which optional variables you provide:

| Variables set | Credentials type |
|---------------|-----------------|
| `ACCESS_TOKEN` | Access Token |
| `USERNAME` / `PASSWORD` / `CLIENT_ID` | MQTT Basic |
| `CA_CERT_FILENAME` | X.509 Certificate |
| *(none)* | Platform default |

## Resetting Provisioned Credentials

To force re-provisioning (e.g. to rotate credentials or switch authentication type), use the
[`gateway_remove_provisioned_credentials`](/docs/iot-gateway/features/service-rpc-methods/#gateway_remove_provisioned_credentials)
service RPC method. The gateway deletes its stored credentials and re-provisions on the next restart.

## Example: Setting Up Device Provisioning

<Steps>
1. **Create a Device Profile** — go to **Device profiles**, click **+** and select **Create new device profile**.

   <ImageGallery images={[
     {
       src: '/src/assets/images/gateway/device-provisioning/gateway-device-provisioning-1-ce.png',
       alt: 'Device profiles page with the Create new device profile option',
       caption: 'Open Device profiles and create a new profile.',
     },
   ]} />

2. Enter a profile name and open the **Device provisioning** tab.

   <ImageGallery images={[
     {
       src: '/src/assets/images/gateway/device-provisioning/gateway-device-provisioning-2-ce.png',
       alt: 'New device profile form with Device provisioning tab open',
       caption: 'Open the Device provisioning tab in the new profile form.',
     },
   ]} />

3. Select **Allow to create new devices**, then copy and save the **Provisioning device key** and
   **Provisioning device secret**. Click **Add**.

   <ImageGallery images={[
     {
       src: '/src/assets/images/gateway/device-provisioning/gateway-device-provisioning-3-ce.png',
       alt: 'Device provisioning tab with Allow to create new devices selected and key/secret shown',
       caption: 'Enable provisioning and copy the key and secret.',
     },
     {
       src: '/src/assets/images/gateway/device-provisioning/gateway-device-provisioning-4-ce.png',
       alt: 'Device profile created successfully',
       caption: 'Device profile created.',
     },
   ]} />

4. **Download the Gateway `docker-compose.yml`** and open it in a text editor.

   <ImageGallery images={[
     {
       src: '/src/assets/images/gateway/device-provisioning/gateway-device-provisioning-5-ce.png',
       alt: 'docker-compose.yml file opened in a text editor',
       caption: 'Open the downloaded docker-compose.yml.',
     },
   ]} />

5. Set the provisioning environment variables, replacing the placeholders with your values:

   ```yaml
   environment:
     - host=THINGSBOARD_HOST
     - port=THINGSBOARD_PORT
     - TB_GW_PROVISIONING_DEVICE_KEY=YOUR_PROVISION_KEY
     - TB_GW_PROVISIONING_DEVICE_SECRET=YOUR_PROVISION_SECRET
     - TB_GW_PROVISIONING_DEVICE_NAME=Factory-Gateway-01
   ```

   Replace `THINGSBOARD_HOST` and `THINGSBOARD_PORT` with the host and port of your ThingsBoard
   instance, and `YOUR_PROVISION_KEY` / `YOUR_PROVISION_SECRET` with the values from Step 3.

   <ImageGallery images={[
     {
       src: '/src/assets/images/gateway/device-provisioning/gateway-device-provisioning-6-ce.png',
       alt: 'docker-compose.yml with provisioning variables filled in',
       caption: 'Fill in the provisioning variables in docker-compose.yml.',
     },
   ]} />

6. **Start the Gateway:**

   ```bash
   docker compose up
   ```

   <ImageGallery images={[
     {
       src: '/src/assets/images/gateway/device-provisioning/gateway-device-provisioning-7-ce.png',
       alt: 'Terminal showing docker compose up output with gateway starting',
       caption: 'Start the gateway with docker compose up.',
     },
   ]} />

7. The gateway auto-registers, receives credentials, and connects to the platform.

   <ImageGallery images={[
     {
       src: '/src/assets/images/gateway/device-provisioning/gateway-device-provisioning-8-ce.png',
       alt: 'Gateway logs showing successful provisioning and connection',
       caption: 'Gateway provisioned and connected successfully.',
     },
   ]} />

8. In the ThingsBoard UI, verify the gateway appears as active on the **Gateways** page or in the
   device details.

   <ImageGallery images={[
     {
       src: '/src/assets/images/gateway/device-provisioning/gateway-device-provisioning-9-ce.png',
       alt: 'ThingsBoard Gateways page showing the new gateway as active',
       caption: 'Gateway visible and active in the ThingsBoard UI.',
     },
   ]} />
</Steps>
