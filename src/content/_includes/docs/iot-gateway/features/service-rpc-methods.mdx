Service RPC methods are built-in gateway commands prefixed with `gateway_` that let you check gateway
health, list connected devices, schedule restarts, and manage provisioning - all from the ThingsBoard
web UI without any connector configuration.

## Prerequisites

- A running ThingsBoard server <DocLink product="CE" path="installation/" bold={false}>installation guide</DocLink>.
- An installed and configured ThingsBoard IoT Gateway
  ([Installation guide](/docs/iot-gateway/installation/), [Configuration guide](/docs/iot-gateway/config/general/)).

## Step 2. Service RPC Methods

Every command prefixed with `gateway_` is interpreted as a service command directed at the gateway
itself rather than at a connected device or connector.

### gateway_ping

Checks the connection to the gateway and confirms that RPC processing is active.

```bash
gateway_ping
```

Response:

```json
{
  "code": 200,
  "resp": "pong"
}
```

<ImageGallery images={[
  {
    src: '/src/assets/images/gateway/gateway-rpc-ping.png',

    caption: 'gateway_ping — connection confirmed.',
  },
]} />

### gateway_devices

Lists all devices currently connected through the gateway. The response is a key-value map where
each key is the device name and the value identifies the connector it is connected through.

```bash
gateway_devices
```

Response:

```json
{
  "code": 200,
  "resp": {
    "Device Number One": "OPC-UA Connector"
  }
}
```

<ImageGallery images={[
  {

    caption: 'gateway_devices — list of connected devices and their connectors.',
  },
]} />

### gateway_restart

Schedules a restart of the gateway service after a specified delay in seconds.
The response is returned immediately after the task is added to the scheduler — before the restart
actually occurs.

```bash
gateway_restart 60
```

Response:

```json
{"success": true}
```

<ImageGallery images={[

]} />

### gateway_reboot

Schedules a full hardware reboot of the gateway machine after a specified delay in seconds.

<Aside type="note">
  This method is available only when the gateway is started as a Python module (not as a system
  daemon), and only when the user running the gateway has reboot permissions.
</Aside>

```bash
gateway_reboot 60
```

Response (returned as soon as the task is scheduled, before the reboot):

```json
{"success": true}
```

<ImageGallery images={[
  {
    src: '/src/assets/images/gateway/gateway-rpc-reboot.png',

### gateway_remove_provisioned_credentials

Removes the device credentials from the gateway's config folder. If
[device provisioning](/docs/iot-gateway/features/provisioning/) is enabled, the gateway
will automatically re-provision itself on the next startup.

```bash
gateway_remove_provisioned_credentials
```

Response:

```json
{"code": 200, "resp": "Provisioned credentials file removed successfully."}
```
