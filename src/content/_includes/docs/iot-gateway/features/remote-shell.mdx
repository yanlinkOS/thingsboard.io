Remote Shell lets you control the operating system of your IoT Gateway machine directly from the ThingsBoard
web interface - execute commands, check system health, troubleshoot issues, and perform maintenance tasks
without needing physical access to the device.

<Aside type="caution">
  This feature may introduce security vulnerabilities. Use SSL encryption when enabling it, and keep it
  disabled unless actively required.
</Aside>

## Prerequisites

- A running ThingsBoard server. The quickest option is [ThingsBoard Cloud](/installations/).
  For self-hosted setups follow the [Installation Guide](/docs/installation/).
- An installed and configured ThingsBoard IoT Gateway
  ([Installation guide](/docs/iot-gateway/installation/), [Getting Started](/docs/iot-gateway/getting-started/)).

## Step 1. Enable Remote Shell

<Steps>
1. Go to **Entities** > **Gateways** and click on the desired gateway.

   <ImageGallery images={[
     {
       src: '/src/assets/images/gateway/remote-shell/remote-shell-1.png',
       alt: 'Gateway list with the target gateway selected',
       caption: 'Open the gateway from the Gateways list.',
     },
   ]} />

2. Click the **General configuration** button in the right panel.

   <ImageGallery images={[
     {
       src: '/src/assets/images/gateway/remote-shell/remote-shell-2.png',
       alt: 'Gateway page with General configuration button highlighted',
       caption: 'Click General configuration in the navigation panel.',
     },
   ]} />

3. On the **General** tab, enable the **Remote shell** toggle and click **Save**.

   <ImageGallery images={[
     {
       src: '/src/assets/images/gateway/remote-shell/remote-shell-3.png',
       alt: 'General configuration tab with Remote shell toggle enabled',
       caption: 'Enable Remote shell and save the configuration.',
     },
   ]} />
</Steps>

## Step 2. Use the Remote Shell

<Steps>
1. On the gateway detail page, click the **Remote shell** button in the right panel.

   <ImageGallery images={[
     {
       src: '/src/assets/images/gateway/remote-shell/remote-shell-4.png',
       alt: 'Gateway navigation panel with Remote shell button',
       caption: 'Open the Remote shell page from the navigation panel.',
     },
   ]} />

2. The shell terminal opens and connects automatically.

   <ImageGallery images={[
     {
       src: '/src/assets/images/gateway/remote-shell/remote-shell-5.png',
       alt: 'Remote shell terminal interface ready for input',
       caption: 'The terminal connects automatically — no extra setup needed.',
     },
   ]} />

3. Run any OS command. For example, `ls` lists the files and directories in the current directory.

   <ImageGallery images={[
     {
       src: '/src/assets/images/gateway/remote-shell/remote-shell-6.png',
       alt: 'Remote shell showing output of the ls command',
       caption: 'Example: ls output showing gateway directory contents.',
     },
   ]} />
</Steps>
