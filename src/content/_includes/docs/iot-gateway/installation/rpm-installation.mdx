Install ThingsBoard IoT Gateway as an `.rpm` package on AlmaLinux or RHEL.
The package registers the Gateway as a `systemd` service that starts automatically on boot.

## Prerequisites

- Python 3.10 or higher is installed.

## Installation

<Steps>
  <ol>
    <li>
      **Download the `.rpm` package.**

      ```bash
      wget https://github.com/thingsboard/thingsboard-gateway/releases/latest/download/python3-thingsboard-gateway.rpm
      ```
    </li>
    <li>
      **Install the package.**

      ```bash
      sudo dnf install -y ./python3-thingsboard-gateway.rpm
      ```
    </li>
    <li>
      **Check the Gateway status.**

      ```bash
      systemctl status thingsboard-gateway
      ```

      <Aside type="note">
        Connection errors in the output are expected at this stage — the Gateway is not yet
        configured to connect to ThingsBoard:

        ```text
        ... python3[7563]: '2019-12-26 09:31:15' - ERROR - mqtt_connector - Default Broker connection FAIL with error 5 not authorised!
        ... python3[7563]: '2019-12-26 09:31:16' - DEBUG - tb_client - connecting to ThingsBoard
        ```
      </Aside>
    </li>
    <li>
      **Configure the Gateway.** Follow the <DocLink product={props.product} path="config/general" bold={false}>configuration guide</DocLink>
      to connect the Gateway to your ThingsBoard instance. After saving the configuration, restart the service to apply the changes.

      ```bash
      systemctl restart thingsboard-gateway
      ```
    </li>
  </ol>
</Steps>

## Offline Build

The Offline Build is a variant of the Gateway package for environments without internet access
(air-gapped, firewalled, or private datacenter deployments). It bundles all dependencies in advance,
including a prebuilt Python virtual environment, all required Python packages, and the application wheel (`.whl`),
so no runtime downloads are needed and all connectors are immediately available.

### Prerequisites

The Offline Build requires the same system prerequisites as the standard package, plus **Python 3.11** exactly.
Python 3.11 is supported on **AlmaLinux 9.x** (9.6 or earlier recommended). AlmaLinux 10 and newer do not provide
Python 3.11 in official repositories, and alternative installation methods are not guaranteed to work.
If an incompatible Python version is detected, the installer exits with an error.

To install Python 3.11 on AlmaLinux 9.x:

```bash
sudo dnf install -y epel-release
sudo dnf install -y python3.11
```

### Installation

<Steps>
  <ol>
    <li>
      **Download the offline `.rpm` package.**

      ```bash
      wget https://github.com/thingsboard/thingsboard-gateway/releases/latest/download/python3.11-thingsboard-gateway-offline.rpm
      ```
    </li>
    <li>
      **Install the package.**

      ```bash
      sudo dnf install -y ./python3.11-thingsboard-gateway-offline.rpm
      ```
    </li>
    <li>
      **Check the Gateway status.**

      ```bash
      systemctl status thingsboard-gateway
      ```
    </li>
  </ol>
</Steps>

### Troubleshooting

**Wrong Python version** — the installer exits with an error indicating Python 3.11 is missing.
Once Python 3.11 is installed, remove the previously installed package before retrying:

```bash
sudo rpm -e --noscripts thingsboard-gateway
```

Then reinstall the offline package as described above.
