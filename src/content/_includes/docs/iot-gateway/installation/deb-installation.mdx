Install ThingsBoard IoT Gateway as a `.deb` package on Ubuntu 22.04 LTS / 24.04 LTS or Raspberry Pi.
The package registers the Gateway as a `systemd` service that starts automatically on boot.

## Prerequisites

- System meets [Ubuntu minimum hardware requirements](https://help.ubuntu.com/lts/serverguide/preparing-to-install.html#system-requirements).
- Python 3.10 or higher is installed.

## Installation

<Steps>
  <ol>
    <li>
      **Download the `.deb` package.**

      ```bash
      wget https://github.com/thingsboard/thingsboard-gateway/releases/latest/download/python3-thingsboard-gateway.deb
      ```
    </li>
    <li>
      **Install the package.**

      ```bash
      sudo apt install ./python3-thingsboard-gateway.deb -y
      ```

      The package automatically installs all required dependencies:

      - System libraries: `libffi-dev`, `libglib2.0-dev`, `libxml2-dev`, `libxslt-dev`, `libssl-dev`, `zlib1g-dev`, `python3-dev`, `python3-pip`.
      - Python modules: `importlib`, `importlib-metadata`, `jsonschema`, `pymodbus`, `lxml`, `jsonpath-rw`, `paho-mqtt`, `pyserial`, `PyYAML`, `simplejson`, `pyrsistent`.
    </li>
    <li>
      **Check the Gateway status.**

      ```bash
      systemctl status thingsboard-gateway
      ```

      <Aside type="note">
        Connection errors in the output are expected at this stage — the Gateway is not yet
        configured to connect to ThingsBoard:

        ```text
        ... python3[7563]: '2019-12-26 09:31:15' - ERROR - mqtt_connector - Default Broker connection FAIL with error 5 not authorised!
        ... python3[7563]: '2019-12-26 09:31:16' - DEBUG - tb_client - connecting to ThingsBoard
        ```
      </Aside>
    </li>
    <li>
      **Configure the Gateway.** Follow the <DocLink product={props.product} path="config/general" bold={false}>configuration guide</DocLink>
      to connect the Gateway to your ThingsBoard instance. After saving the configuration, restart the service to apply the changes.

      ```bash
      systemctl restart thingsboard-gateway
      ```
    </li>
  </ol>
</Steps>

## Offline Build

The Offline Build is a variant of the Gateway package for environments without internet access
(air-gapped, firewalled, or private datacenter deployments). It bundles all dependencies in advance,
including a prebuilt Python virtual environment, all required Python packages, and the application wheel (`.whl`),
so no runtime downloads are needed and all connectors are immediately available.

### Prerequisites

The Offline Build requires the same system prerequisites as the standard package, plus **Python 3.11** exactly.
If an incompatible Python version is detected, the installer exits with instructions for installing the correct version.

To install Python 3.11:

```bash
sudo add-apt-repository ppa:deadsnakes/ppa -y
sudo apt-get update
sudo apt-get install -y python3.11 python3.11-venv
```

### Installation

<Steps>
  <ol>
    <li>
      **Download the offline `.deb` package.**

      ```bash
      wget https://github.com/thingsboard/thingsboard-gateway/releases/latest/download/python3.11-thingsboard-gateway-offline.deb
      ```
    </li>
    <li>
      **Install the package.**

      ```bash
      sudo apt install ./python3.11-thingsboard-gateway-offline.deb -y
      ```
    </li>
    <li>
      **Check the Gateway status.**

      ```bash
      systemctl status thingsboard-gateway
      ```
    </li>
  </ol>
</Steps>

### Troubleshooting

**Wrong Python version** — the installer displays a guide for installing the required version.
Once Python 3.11 is installed, remove the previously installed package before retrying:

```bash
sudo dpkg --purge python3-thingsboard-gateway
```

Then reinstall the offline package as described above.
