Upgrade ThingsBoard IoT Gateway to the latest version. Configuration files,
connector settings, and custom extensions are preserved during the upgrade. For a first-time
installation, see <DocLink product={props.product} path="installation" bold={false}>Installation options</DocLink>.

<Tabs defaultTab="deb" urlParam="upgradeMethod">
  <TabItem label="Ubuntu / Raspberry Pi" id="deb" icon="tabler:brand-ubuntu">
    <Aside type="caution">
      If upgrading from a Gateway version **lower than 3.7.6**, manually back up your configuration
      and extension files before proceeding.
    </Aside>

    <Steps>
      <ol>
        <li>
          **Download the `.deb` package.**

          ```bash
          wget https://github.com/thingsboard/thingsboard-gateway/releases/latest/download/python3-thingsboard-gateway.deb
          ```
        </li>
        <li>
          **Install the new package.**

          ```bash
          sudo apt install ./python3-thingsboard-gateway.deb -y
          ```

          <Aside type="note">
            This automatically upgrades your existing Gateway installation without removing configuration
            or extension files **(from Gateway version 3.7.6)**.
          </Aside>
        </li>
        <li>
          **Restart the Gateway.**

          ```bash
          sudo systemctl restart thingsboard-gateway
          ```
        </li>
        <li>
          **Verify the upgrade.** Confirm the service started successfully.

          ```bash
          sudo systemctl status thingsboard-gateway
          ```
        </li>
      </ol>
    </Steps>

    ## Configuration File Preservation

    During the upgrade, **all user configuration files are preserved**, including connector
    configuration files, custom user extensions, and `tb_gateway.json`. An automatic backup of
    your configuration directory is also created:

    | Backup | Path |
    |--------|------|
    | Connector configurations | `/etc/thingsboard-gateway/configs_backup.tar.gz` |
    | User extensions | `/var/lib/thingsboard_gateway/extensions_backup.tar.gz` |
  </TabItem>

  <TabItem label="Docker" id="docker" icon="tabler:brand-docker">
    <Aside type="caution">
      Running `docker compose down` disconnects all downstream devices from ThingsBoard immediately.
      Plan the upgrade during a maintenance window to minimise disruption.
    </Aside>

    <Steps>
      <ol>
        <li>
          **Stop and remove existing containers.** This shuts down the Gateway gracefully without
          deleting volumes or configuration data.

          ```bash
          docker compose down
          ```
        </li>
        <li>
          **Pull the latest Gateway image.**

          ```bash
          docker pull thingsboard/tb-gateway
          ```
        </li>
        <li>
          **Start the updated Gateway.** The `-d` flag runs containers in detached mode.

          ```bash
          docker compose up -d
          ```
        </li>
        <li>
          **Verify the upgrade.** Watch the logs to confirm the Gateway is running correctly.

          ```bash
          docker compose logs -f
          ```
        </li>
      </ol>
    </Steps>

    <Aside type="note">
      This method works with any Gateway version as long as your `docker-compose.yml` and volume
      mappings are correctly configured. Consider version pinning in your Compose file to control
      future upgrades.
    </Aside>
  </TabItem>

  <TabItem label="Python (pip)" id="pip" icon="tabler:brand-python">
    <Steps>
      <ol>
        <li>
          **Stop the Gateway.**

          If running as a systemd service:

          ```bash
          sudo systemctl stop thingsboard-gateway
          ```

          If running manually, stop it with **Ctrl + C**.
        </li>
        <li>
          **Upgrade the package.**

          ```bash
          pip install --upgrade thingsboard-gateway
          ```

          <Aside type="tip">
            Depending on your environment, use `pip3` or `python3 -m pip` instead of `pip`. Add
            `--user` for a user-level installation.
          </Aside>
        </li>
        <li>
          **Restart the Gateway.**

          If running as a systemd service:

          ```bash
          sudo systemctl start thingsboard-gateway
          ```

          If running manually:

          ```bash
          thingsboard-gateway
          ```
        </li>
      </ol>
    </Steps>

    <Aside type="note">
      If running in a virtual environment, ensure both the `pip install` command and the service
      control commands are executed within that environment. If installed via a systemd service,
      verify the service file points to the correct environment path.
    </Aside>
  </TabItem>
</Tabs>
