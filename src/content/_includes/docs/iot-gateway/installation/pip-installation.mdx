Install ThingsBoard IoT Gateway as a Python package using `pip` on Ubuntu or Debian-based systems.

## Prerequisites

- Python 3.10 or higher is installed.
- `sudo` access on the target machine.

## Installation

<Steps>
  <ol>
    <li>
      **Install system libraries.**

      ```bash
      sudo apt install python3-dev python3-pip libglib2.0-dev
      ```
    </li>
    <li>
      **Install the ThingsBoard Gateway package.**

      ```bash
      sudo pip3 install thingsboard-gateway
      ```
    </li>
    <li>
      **Download and unpack the example configuration.**

      ```bash
      wget https://github.com/thingsboard/thingsboard-gateway/releases/latest/download/configs.tar.gz
      sudo mkdir /etc/thingsboard-gateway
      sudo mkdir /var/log/thingsboard-gateway
      sudo tar -xvzf configs.tar.gz -C /etc/thingsboard-gateway
      ```
    </li>
    <li>
      **Set folder permissions.** Replace `YOUR_USER` with the user that will run the Gateway.

      ```bash
      sudo chown YOUR_USER:YOUR_USER -R /var/log/thingsboard-gateway
      sudo chown YOUR_USER:YOUR_USER -R /etc/thingsboard-gateway
      sudo chown YOUR_USER:YOUR_USER /tmp
      ```
    </li>
    <li>
      **Create a Gateway device in ThingsBoard.**

      Go to **Entities** > **Gateways** and click the **+** icon in the upper-right corner.
      Enter a name (e.g., `My Gateway`), select the **default** device profile, and click **Create**.

      <ImageGallery images={[
        {
          src: '/src/assets/images/gateway/dashboard/gateway-getting-started-1-ce.png',
          alt: 'Gateways page with the add button highlighted',
          caption: 'Go to Entities > Gateways and click the + icon to add a new gateway.',
        },
        {
          src: '/src/assets/images/gateway/dashboard/gateway-getting-started-2-ce.png',
          alt: 'New gateway dialog with name and device profile fields',
          caption: 'Enter a gateway name, select the default device profile, and click Create.',
        },
      ]} />
    </li>
    <li>
      **Configure the Gateway credentials.**

      Click the **General configuration** button on the Gateway device page and copy the **Access token**
      from the **General** tab. Then open the Gateway configuration file and paste the token, replacing
      the host and port with your ThingsBoard instance address:

      ```bash
      nano /etc/thingsboard-gateway/config/tb_gateway.json
      ```

      <ImageGallery images={[
        {
          src: '/src/assets/images/gateway/install/configure-gateway-1-ce.png',
          alt: 'Gateway device page with General configuration button',
          caption: 'Click the General configuration button.',
        },
        {
          src: '/src/assets/images/gateway/install/configure-gateway-2-ce.png',
          alt: 'General tab showing the Access token field',
          caption: 'Copy the Access token from the General tab.',
        },
        {
          src: '/src/assets/images/gateway/install/configure-gateway-3-ce.png',
          alt: 'tb_gateway.json open in nano with host, port, and token fields',
          caption: 'Paste the access token and update the host and port. Save with Ctrl+O, exit with Ctrl+X.',
        },
      ]} />
    </li>
    <li>
      **Configure the log file path.**

      Click **General configuration** again, navigate to the **Logs** tab, and set the **File path**
      to `/var/log/thingsboard-gateway` for each entry in the **Local logging** section. Click **Save**.

      <ImageGallery images={[
        {
          src: '/src/assets/images/gateway/install/configure-gateway-4-ce.png',
          alt: 'Gateway device page with General configuration button',
          caption: 'Click the General configuration button.',
        },
        {
          src: '/src/assets/images/gateway/install/configure-gateway-5-ce.png',
          alt: 'Logs tab with the file path field updated',
          caption: 'Set the file path to /var/log/thingsboard-gateway for all Local logging entries, then click Save.',
        },
      ]} />
    </li>
    <li>
      **Start the Gateway and verify.**

      ```bash
      thingsboard-gateway
      ```

      If everything is configured correctly, the Gateway device status changes to **Active** in ThingsBoard.

      <ImageGallery images={[
        {
          src: '/src/assets/images/gateway/install/configure-gateway-6-ce.png',
          alt: 'ThingsBoard Gateways page showing the gateway status as Active',
          caption: 'The gateway status changes to Active once the connection is established.',
        },
      ]} />
    </li>
  </ol>
</Steps>
