## Prerequisites

- Active <a href="https://cloud.digitalocean.com/" target="_blank">DigitalOcean</a> account

## Step 1. Create a DigitalOcean Droplet

### 1.1 Create Droplet

Log in to your DigitalOcean account. Click the **Droplets** menu item and then the **Create Droplet** button.

<ImageGallery images={[
  { src: '/src/assets/images/docs/installation/digital-ocean/create-droplet.png', alt: 'Create Droplet button', caption: 'Click "Create Droplet" from the Droplets page.' },
]} />

### 1.2 Select your plan

Select **Ubuntu 24.04 LTS** as your image and choose your plan.

<Aside type="note">
For beginners, we recommend **2 CPUs and 4 GB of RAM**. Advanced users may choose the plan that suits their workload best.
</Aside>

<ImageGallery images={[
  { src: '/src/assets/images/docs/installation/digital-ocean/choose-plan.png', alt: 'Choose plan', caption: 'Select Ubuntu and your plan size.' },
]} />

### 1.3 Choose region

Select a datacenter region from the available list.

- **Optional**: Enable backups. Although optional, we highly recommend this.
- **Optional**: Add block storage.

<ImageGallery images={[
  { src: '/src/assets/images/docs/installation/digital-ocean/choose-region.png', alt: 'Choose region', caption: 'Select a datacenter region and optionally enable backups.' },
]} />

### 1.4 Authentication

- Use an existing or create a new SSH key. Make sure you have access to the SSH key — you will need it later to connect via SSH.
- Enter a meaningful hostname.
- Click **Create Droplet**.

<ImageGallery images={[
  { src: '/src/assets/images/docs/installation/digital-ocean/create-final.png', alt: 'Create Droplet final', caption: 'Configure SSH key, hostname, and click "Create Droplet".' },
]} />

Once the droplet is created, copy the IP address to a safe place. You will use it later.

<ImageGallery images={[
  { src: '/src/assets/images/docs/installation/digital-ocean/droplet-created.png', alt: 'Droplet created', caption: 'Copy the IP address of your new droplet.' },
]} />

## Step 2. Configure firewall rules

Configure firewall rules to allow MQTT, CoAP, and HTTP traffic.

<Steps>
1. Navigate to **Networking → Firewalls** and click **Create Firewall**.

   <ImageGallery images={[
     { src: '/src/assets/images/docs/installation/digital-ocean/create-firewall.png', alt: 'Create Firewall', caption: 'Navigate to Firewalls and click "Create Firewall".' },
   ]} />

2. Give your firewall a meaningful name and configure inbound rules:

   | Port | Protocol | Description |
   |---|---|---|
   | 22 | TCP | SSH access |
   | 80 | TCP | HTTP access |
   | 443 | TCP | HTTPS access |
   | 1883 | TCP | MQTT protocol |
   | 5683 | UDP | CoAP protocol |

   <ImageGallery images={[
     { src: '/src/assets/images/docs/installation/digital-ocean/firewall-config.png', alt: 'Firewall configuration', caption: 'Configure inbound rules for HTTP, HTTPS, MQTT, and CoAP.' },
   ]} />

3. Leave outbound rules as-is. Select your droplet or tag to assign this firewall, then click **Create Firewall**.

   <ImageGallery images={[
     { src: '/src/assets/images/docs/installation/digital-ocean/firewall-final.png', alt: 'Firewall final', caption: 'Assign the firewall to your droplet and click "Create Firewall".' },
   ]} />
</Steps>

## Step 3. Connect to your instance using SSH

Use the <a href="https://www.digitalocean.com/docs/droplets/how-to-connect-with-ssh/" target="_blank">official guide</a> and the SSH key you created during [Step 1.4](#14-authentication).

## Step 4. Install ThingsBoard

Follow the <DocLink product={props.product} path="installation/docker">Docker installation guide</DocLink> to install and configure ThingsBoard.

<Aside type="note">
Use your droplet IP address instead of "localhost" to access the web UI.
</Aside>

## Post-installation steps

### Configure HTTPS

You may want to configure HTTPS access using HAProxy. This is possible if you have a valid DNS name assigned to your instance. Follow the <DocLink product={props.product} path="TODO">HAProxy HTTPS guide</DocLink> to install HAProxy and generate a valid SSL certificate using Let's Encrypt.

### Upgrading

To maintain database integrity and system stability, perform upgrades sequentially (e.g., v4.0.2 → v4.1.0 → v4.2.0). See the <DocLink product={props.product} path="installation/upgrade-instructions/">Upgrade Instructions</DocLink> for detailed steps.

## Troubleshooting

ThingsBoard logs are stored in `/var/log/thingsboard`. To check for errors:

```bash
cat /var/log/thingsboard/thingsboard.log | grep ERROR
```
