## Introduction

This guide covers the basics of <a href="https://thingsboard.io/products/mqtt-broker/" target="_blank" rel="noopener noreferrer">TBMQ</a>:

- Connecting MQTT clients to the broker.
- Publishing and subscribing to messages.
- Configuring client authentication and authorization.

For details on the TBMQ architecture, see <DocLink product={props.product} path="architecture">Architecture</DocLink>.

## Try live demo

The fastest way to get started with TBMQ is to use the free public MQTT broker at <a href="https://demo.tbmq.io" target="_blank" rel="noopener noreferrer">demo.tbmq.io</a>.
This sandbox environment allows you to explore TBMQ features without any installation.

**MQTT connection details:**

| Parameter | Value           |
|-----------|-----------------|
| Host      | demo.tbmq.io    |
| TCP Port  | 1883            |
| TLS Port  | 8883            |
| Username  | demo            |
| Password  | *(leave empty)* |

You can start publishing and subscribing to topics immediately using these credentials.
<a href="https://demo.tbmq.io/signup" target="_blank" rel="noopener noreferrer">Sign up</a> for a free account to access the TBMQ UI and explore sessions, subscriptions, and other features.

<Aside type="note">
  The demo instance provides read-only access to the UI. You can observe all sessions, subscriptions, and broker statistics, but cannot modify configurations.
  If you prefer to run TBMQ locally with full administrative access, proceed to the [Install TBMQ locally](#install-tbmq-locally) section.
</Aside>

## Configure client authentication and authorization

<Aside type="note">
  If you are using the <a href="https://demo.tbmq.io" target="_blank" rel="noopener noreferrer">demo instance</a>, skip this section. Demo credentials are pre-configured and ready to use.
</Aside>

To secure the connection to the broker, enable <DocLink product={props.product} path="security/authentication/basic">Basic authentication</DocLink>.

After you enable Basic authentication, create MQTT Client Credentials of type `Basic` to authenticate connecting clients:

- Go to **Authentication** → **Credentials** and click **Add** (+).
- Enter a credentials name, for example, "Getting Started Credentials".
- Enter "username" and "password" values of your choice, for example, `username` and `password`.
- Click **Add**.

By default, authorization rules allow publishing and subscribing to any topic.

## Publishing and subscribing to topics

Publish messages and subscribe to topics to observe the flow of messages.
This tutorial uses <a href="https://mosquitto.org/" target="_blank" rel="noopener noreferrer">Mosquitto</a> clients.
For details, see the <a href="https://mosquitto.org/man/mosquitto_pub-1.html" target="_blank" rel="noopener noreferrer">mosquitto_pub</a> and
<a href="https://mosquitto.org/man/mosquitto_sub-1.html" target="_blank" rel="noopener noreferrer">mosquitto_sub</a> documentation.

<Tabs>
<TabItem label="Demo Instance">

##### Subscribe to a topic

To subscribe to the **sensors/temperature** topic and start receiving messages, use the following command:

```bash
mosquitto_sub -d -h demo.tbmq.io -p 1883 -t sensors/temperature -q 1 -u demo
```

After connecting, go to the **Sessions** page to view the session details.

If you signed up for a demo account, log in to <a href="https://demo.tbmq.io" target="_blank" rel="noopener noreferrer">demo.tbmq.io</a> and go to the **Sessions** page to see all active sessions, including your new session.

##### Publish a message

To publish a message to the **sensors/temperature** topic, use the following command:

```bash
mosquitto_pub -d -h demo.tbmq.io -p 1883 -t sensors/temperature -m 32 -q 1 -u demo
```

</TabItem>
<TabItem label="Local Deployment">

##### Subscribe to a topic

To subscribe to the **sensors/temperature** topic and start receiving messages, use the following command:

```bash
mosquitto_sub -d -h localhost -p 1883 -t sensors/temperature -q 1 -u username -P password
```

<Aside type="note">
  Replace `username` and `password` with the credentials you created in the [authentication section](#configure-client-authentication-and-authorization).
</Aside>

After connecting, go to the **Sessions** page to view the session details.
