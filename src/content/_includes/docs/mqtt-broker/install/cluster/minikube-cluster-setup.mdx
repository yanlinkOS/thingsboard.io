This guide covers setting up TBMQ in cluster mode using Minikube.

## Prerequisites

- A running Kubernetes cluster with `kubectl` configured to communicate with it. If you don't have Minikube installed, follow [these instructions](https://kubernetes.io/docs/setup/learning-environment/minikube/).
- [Helm](https://helm.sh/docs/intro/install/) installed.

## Running

Deploy TBMQ:

```bash
./k8s-deploy-tbmq.sh
```

Once all resources have started, open `http://{your-cluster-ip}:30001` in your browser (e.g. **http://192.168.49.2:30001**).
Check your cluster IP with:

```bash
minikube ip
```

You should see the TBMQ login page. Use the default **System Administrator** credentials:

- **Username:** `sysadmin@thingsboard.org`
- **Password:** `sysadmin`

On first login you will be prompted to change the default password.

## Logs, delete statefulsets and services

To view TBMQ node logs:

1. List running TBMQ pods:

   ```bash
   kubectl get pods -l app=tbmq
   ```

2. Fetch logs for a specific pod:

   ```bash
   kubectl logs -f TBMQ_POD_NAME
   ```

   Where `TBMQ_POD_NAME` is the pod name from the list above.

To check the state of all pods, services, deployments, and statefulsets:

```bash
kubectl get pods
kubectl get services
kubectl get deployments
kubectl get statefulsets
```

See [kubectl Cheat Sheet](https://kubernetes.io/docs/reference/kubectl/cheatsheet/) for more details.

To delete TBMQ nodes:

```bash
./k8s-delete-tbmq.sh
```

To delete all resources including databases:

```bash
./k8s-delete-all.sh
```
