## MQTT listeners parameters

<div class="config-def-list">
  <div class="config-def-item">
    <p class="config-def-meta"><code class="config-def-env">MQTT&#95;PROXY&#95;PROTOCOL&#95;ENABLED</code> · <span class="config-def-label">Default</span> <code>false</code></p>
    <p class="config-def-desc">Enables proxy protocol support as a global setting for all listeners. If enabled, supports both v1 and v2.  Useful to get the real IP address of the client in the logs, for session details info and unauthorized clients feature.</p>
  </div>
  <div class="config-def-item">
    <p class="config-def-meta"><code class="config-def-env">NETTY&#95;LEAK&#95;DETECTOR&#95;LVL</code> · <span class="config-def-label">Default</span> <code>DISABLED</code></p>
    <p class="config-def-desc">Netty leak detector level. Accepted values: DISABLED, SIMPLE, ADVANCED, PARANOID. Applied globally for all listeners.</p>
  </div>
  <div class="config-def-item">
    <p class="config-def-meta"><code class="config-def-env">NETTY&#95;WRITE&#95;BUFFER&#95;HIGH&#95;WATER&#95;MARK</code> · <span class="config-def-label">Default</span> <code>1280</code></p>
    <p class="config-def-desc">Threshold (in KB) at which Netty considers the channel non-writable. When reached, TBMQ stops delivering data to the subscriber until the channel becomes writable again.  Non-persistent clients lose data in this case.</p>
  </div>
  <div class="config-def-item">
    <p class="config-def-meta"><code class="config-def-env">NETTY&#95;WRITE&#95;BUFFER&#95;LOW&#95;WATER&#95;MARK</code> · <span class="config-def-label">Default</span> <code>640</code></p>
    <p class="config-def-desc">Threshold (in KB) at which Netty considers the channel writable again. When reached, TBMQ resumes delivering data to the subscriber.</p>
  </div>
  <div class="config-def-item">
    <p class="config-def-meta"><code class="config-def-env">NETTY&#95;SO&#95;RECEIVE&#95;BUFFER</code> · <span class="config-def-label">Default</span> <code>0</code></p>
    <p class="config-def-desc">Netty socket receive buffer size (in KB). When the buffer limit is reached, TCP triggers backpressure and notifies the sender to slow down.  Defaults to 0, which uses the system's default buffer size.</p>
  </div>
  <div class="config-def-item">
    <p class="config-def-meta"><code class="config-def-env">LISTENER&#95;TCP&#95;ENABLED</code> · <span class="config-def-label">Default</span> <code>true</code></p>
    <p class="config-def-desc">Enables or disables MQTT TCP port listener.</p>
  </div>
  <div class="config-def-item">
    <p class="config-def-meta"><code class="config-def-env">LISTENER&#95;TCP&#95;BIND&#95;ADDRESS</code> · <span class="config-def-label">Default</span> <code>0.0.0.0</code></p>
    <p class="config-def-desc">MQTT TCP listener bind address.</p>

  <div class="config-def-item">
    <p class="config-def-meta"><code class="config-def-env">LISTENER&#95;TCP&#95;BIND&#95;PORT</code> · <span class="config-def-label">Default</span> <code>1883</code></p>
    <p class="config-def-desc">MQTT TCP listener bind port.</p>
  </div>
  <div class="config-def-item">
    <p class="config-def-meta"><code class="config-def-env">MQTT&#95;TCP&#95;PROXY&#95;PROTOCOL&#95;ENABLED</code></p>

If explicitly set, supports both v1 and v2 and takes precedence over the global setting.  Useful to get the real IP address of the client in the logs, for session details info and unauthorized clients feature.</p>

    <p class="config-def-desc">Number of Netty worker group threads.</p>
