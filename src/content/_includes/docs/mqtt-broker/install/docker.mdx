This guide describes how to install and start a standalone TBMQ instance using Docker on Linux or macOS.
For cluster installation, see the <DocLink product={props.product} path="install/cluster/docker-compose-setup">cluster setup page</DocLink>.

## Prerequisites

Running TBMQ on a single machine requires at least 2 GB of RAM.

- <a href="https://docs.docker.com/engine/install/" target="_blank" rel="noopener noreferrer">Install Docker</a>

</Aside>

## Logs, stop, and start commands

To examine service logs for errors, for example, to view TBMQ logs, run:

```bash
docker compose logs -f tbmq
```

To stop the containers:

```bash
docker compose stop
```

To start the containers:

```bash
docker compose start
```

## Enabling MQTTS (MQTT over SSL/TLS)

To enable **MQTT over SSL/TLS (MQTTS)** in TBMQ, provide valid SSL certificates and configure TBMQ to use them.

For details on supported formats and configuration options, see the <DocLink product={props.product} path="security/mqtts">MQTT over SSL</DocLink> guide.

**Prepare SSL certificates**

Obtain a valid SSL certificate and private key. For example:

- `mqttserver.pem` — public certificate (may include the full chain)
- `mqttserver_key.pem` — private key

<Aside>
  Self-signed certificates are supported for testing. For production environments, use certificates from a trusted Certificate Authority.
</Aside>

**Mount certificates into the container**

In your `docker-compose.yml`, mount the directory containing the certificates:

```yaml
volumes:
  - PATH_TO_CERTS:/config/certificates
```

Replace `PATH_TO_CERTS` with the path to your certificate files. Ensure TBMQ has read access to these files.

**Configure environment variables**

Add the following variables to the `tbmq` service environment in `docker-compose.yml`:

```yaml
LISTENER_SSL_ENABLED: "true"
LISTENER_SSL_PEM_CERT: "/config/certificates/mqttserver.pem"
LISTENER_SSL_PEM_KEY: "/config/certificates/mqttserver_key.pem"
LISTENER_SSL_PEM_KEY_PASSWORD: "server_key_password"
```

<Aside type="tip">
  Leave `LISTENER_SSL_PEM_KEY_PASSWORD` empty if your private key is not password-protected.
</Aside>

**Expose the MQTTS port**

In `docker-compose.yml`:

```yaml
ports:
  - "8883:8883"
```

**Restart TBMQ**

Apply the changes by restarting TBMQ:

```bash
./tbmq-install-and-run.sh
```

Once restarted, MQTT clients can securely connect to **port 8883** using TLS/SSL.
