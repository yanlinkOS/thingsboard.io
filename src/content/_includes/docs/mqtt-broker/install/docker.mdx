This guide covers installing and starting a standalone TBMQ instance using Docker on Linux or macOS.

## Prerequisites

- At least 2 GB of RAM.
- [Docker](https://docs.docker.com/engine/installation/) installed.

<Aside type="caution">
  Add your Linux user to the `docker` group before running Docker commands.
  See [Manage Docker as a non-root user](https://docs.docker.com/engine/install/linux-postinstall/#manage-docker-as-a-non-root-user).
</Aside>

## Logs, stop and start commands

To view TBMQ logs:

```bash
docker compose logs -f tbmq
```

To stop the containers:

```bash
docker compose stop
```

To start the containers:

```bash
docker compose start
```

## Enabling MQTTS (MQTT over SSL/TLS)

To enable MQTT over SSL/TLS in TBMQ, provide valid SSL certificates and configure TBMQ to use them.
For supported formats and configuration options, see the <DocLink product={props.product} path="security/mqtts">MQTT over SSL</DocLink> guide.

**Prepare SSL certificates**

Obtain a valid SSL certificate and private key, for example:

- `mqttserver.pem` — public certificate (may include the full chain)
- `mqttserver_key.pem` — private key

<Aside>
  Self-signed certificates are supported for testing. Use certificates from a trusted Certificate Authority in production.
</Aside>

**Mount certificates into the container**

In `docker-compose.yml`, add a volume mount:

```yaml
volumes:
  - PATH_TO_CERTS:/config/certificates
```

Replace `PATH_TO_CERTS` with the path to your certificate files and ensure TBMQ has read access.

**Configure environment variables**

Add the following to the `tbmq` service environment in `docker-compose.yml`:

```yaml
LISTENER_SSL_ENABLED: "true"
LISTENER_SSL_PEM_CERT: "/config/certificates/mqttserver.pem"
LISTENER_SSL_PEM_KEY: "/config/certificates/mqttserver_key.pem"
LISTENER_SSL_PEM_KEY_PASSWORD: "server_key_password"
```

Leave `LISTENER_SSL_PEM_KEY_PASSWORD` empty if your private key has no password.

**Expose the MQTTS port**

In `docker-compose.yml`:

```yaml
ports:
  - "8883:8883"
```

**Restart TBMQ**

```bash
./tbmq-install-and-run.sh
```

Once restarted, MQTT clients can connect securely on **port 8883**.
