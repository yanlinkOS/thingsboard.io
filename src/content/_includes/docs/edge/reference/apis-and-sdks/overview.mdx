## Built-in transport protocols

These protocols apply to devices that communicate over standard IP transports and can connect directly to ThingsBoard Edge.

- <DocLink product={props.product} path="reference/apis-and-sdks/mqtt-api" target="_blank">MQTT API</DocLink>
- <DocLink product={props.product} path="reference/apis-and-sdks/coap-api" target="_blank">CoAP API</DocLink>
- <DocLink product={props.product} path="reference/apis-and-sdks/http-api" target="_blank">HTTP API</DocLink>
- <DocLink product={props.product} path="reference/apis-and-sdks/lwm2m-api" target="_blank">LwM2M API</DocLink>
- <DocLink product={props.product} path="reference/apis-and-sdks/snmp-api" target="_blank">SNMP API</DocLink>

Most protocols support JSON, Protobuf, or a custom data format. This is the best option for new devices when you control the firmware.

## IoT Gateway

<a href="/docs/iot-gateway/" target="_blank" rel="noopener noreferrer">ThingsBoard IoT Gateway</a> connects devices in local networks without direct internet access, or using non-IP protocols — including MQTT, OPC-UA, Modbus, BLE, HTTP, CAN, BACnet, ODBC, and SNMP. The gateway converts device data to the ThingsBoard format and forwards it over MQTT to the platform.
