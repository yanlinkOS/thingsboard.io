  - ThingsBoard Edge version X.Y.Z works with ThingsBoard Server version X.Y.Z and the next **two** later versions.

## Manual installation and configuration

If you are unable to use the guided instructions above, follow the steps below to install and configure Edge manually.

### Step 1. Install Java 17 (OpenJDK)

ThingsBoard Edge requires Java 17. To install OpenJDK 17:

```bash
sudo apt update && sudo apt install openjdk-17-jdk-headless
```

Configure your operating system to use OpenJDK 17 by default:

```bash
sudo update-alternatives --config java
```

Verify the installation:

```bash
java -version
```

The expected output:

```text
openjdk version "17.x.xx"
OpenJDK Runtime Environment (...)
OpenJDK 64-Bit Server VM (...)
```

### Step 2. Configure the ThingsBoard Edge database

ThingsBoard Edge supports **SQL** and **hybrid** database approaches.

<Tabs>
<TabItem label="PostgreSQL">

**PostgreSQL** is recommended for development and production environments with moderate load (fewer than 5,000 messages/sec). Many cloud providers offer managed PostgreSQL services, making it a cost-effective option for most deployments.

To install the PostgreSQL database:

```bash
# Automated repository configuration:
sudo apt install -y postgresql-common
sudo /usr/share/postgresql-common/pgdg/apt.postgresql.org.sh

# Install and launch the PostgreSQL service:
sudo apt update
sudo apt -y install postgresql-16
sudo service postgresql start
```

Once PostgreSQL is installed, set the password for the main PostgreSQL user:

```bash
sudo -u postgres psql -c "\password"
```

Enter and confirm the password when prompted.

Create a new PostgreSQL database named **tb_edge**:

```bash
echo "CREATE DATABASE tb_edge;" | psql -U postgres -d postgres -h 127.0.0.1 -W
```

</TabItem>
<TabItem label="Hybrid (PostgreSQL + Cassandra)">

The **hybrid** approach is recommended when you plan to manage 1M+ devices in production or handle high data ingestion rates (more than 5,000 messages/sec). In this case, ThingsBoard Edge stores time-series data in Cassandra while using PostgreSQL for primary entities like devices, assets, dashboards, and customers.

#### PostgreSQL installation

To install the PostgreSQL database:

```bash
# Automated repository configuration:
sudo apt install -y postgresql-common
sudo /usr/share/postgresql-common/pgdg/apt.postgresql.org.sh

# Install and launch the PostgreSQL service:
sudo apt update
sudo apt -y install postgresql-16
sudo service postgresql start
```

Once PostgreSQL is installed, set the password for the main PostgreSQL user:

```bash
sudo -u postgres psql -c "\password"
```

Create a new PostgreSQL database named **tb_edge**:

```bash
echo "CREATE DATABASE tb_edge;" | psql -U postgres -d postgres -h 127.0.0.1 -W
```

#### Cassandra installation

To install Apache Cassandra:

```bash
# Add Cassandra repository
echo "deb [signed-by=/etc/apt/keyrings/apache-cassandra.asc] https://debian.cassandra.apache.org 50x main" | sudo tee /etc/apt/sources.list.d/cassandra.sources.list
sudo curl -o /etc/apt/keyrings/apache-cassandra.asc https://downloads.apache.org/cassandra/KEYS
sudo apt-get update
# Cassandra installation
sudo apt-get install -y cassandra cassandra-tools
```

Verify that Cassandra is running:

```bash
nodetool status
sudo systemctl status cassandra.service
```

The status column in the output should report `UN` (Up/Normal). You may need to wait a few moments for Cassandra to initialize.

#### ThingsBoard Edge configuration for hybrid mode

Open the ThingsBoard Edge configuration file:

```bash
sudo nano /etc/tb-edge/conf/tb-edge.conf
```

Add the following lines. Replace `PUT_YOUR_POSTGRESQL_PASSWORD_HERE` with your actual PostgreSQL user password:

```bash
# DB Configuration
export DATABASE_TS_TYPE=cassandra
export SPRING_DATASOURCE_URL=jdbc:postgresql://localhost:5432/tb_edge
export SPRING_DATASOURCE_USERNAME=postgres
export SPRING_DATASOURCE_PASSWORD=PUT_YOUR_POSTGRESQL_PASSWORD_HERE
```

Optionally, add the following parameters to connect to external Cassandra nodes:

```bash
export CASSANDRA_CLUSTER_NAME=Edge Cluster
export CASSANDRA_KEYSPACE_NAME=thingsboard
export CASSANDRA_URL=127.0.0.1:9042
export CASSANDRA_USE_CREDENTIALS=false
export CASSANDRA_USERNAME=
export CASSANDRA_PASSWORD=
```

</TabItem>
</Tabs>

### Step 3. Select the queue service

ThingsBoard Edge can use different messaging systems for storing messages and enabling communication between its services.

<Tabs>
<TabItem label="In Memory">

The **In Memory** queue is the built-in default implementation. It is suitable for development or proof-of-concept environments, but is **not recommended for production** or clustered deployments due to limited scalability.

No additional configuration is required.

</TabItem>
<TabItem label="Kafka (recommended)">

**Kafka** is recommended for production deployments and is used in most ThingsBoard production environments. It provides high throughput, durability, and horizontal scalability.

Install Docker for <a href="https://docs.docker.com/engine/install/ubuntu/" target="_blank" rel="noopener noreferrer">Ubuntu</a> if you have not done so already.

<Aside type="note">
  ThingsBoard supports **Docker Compose V2** (Docker Desktop or Compose plugin). Upgrade to Docker Compose V2 — Docker no longer supports the standalone `docker-compose` tool.
</Aside>

To install **Kafka** in a Docker container, run the following command. It creates a `docker-compose-kafka.yml` file and starts the container:

```bash maxLines=20 collapsible
cat > docker-compose-kafka.yml <<EOF && docker compose -f docker-compose-kafka.yml up -d
services:
  kafka:
    restart: always
    image: bitnamilegacy/kafka:4.0
    ports:
      - 9092:9092 #to localhost:9092 from host machine
      - 9093 #for Kraft
      - 9094 #to kafka:9094 from within Docker network
    environment:
      ALLOW_PLAINTEXT_LISTENER: "yes"
      KAFKA_CFG_LISTENERS: "OUTSIDE://:9092,CONTROLLER://:9093,INSIDE://:9094"
      KAFKA_CFG_ADVERTISED_LISTENERS: "OUTSIDE://localhost:9092,INSIDE://kafka:9094"
      KAFKA_CFG_LISTENER_SECURITY_PROTOCOL_MAP: "INSIDE:PLAINTEXT,OUTSIDE:PLAINTEXT,CONTROLLER:PLAINTEXT"
      KAFKA_CFG_INTER_BROKER_LISTENER_NAME: "INSIDE"
      KAFKA_CFG_AUTO_CREATE_TOPICS_ENABLE: "false"
      KAFKA_OFFSETS_TOPIC_REPLICATION_FACTOR: "1"
      KAFKA_TRANSACTION_STATE_LOG_MIN_ISR: "1"
      KAFKA_TRANSACTION_STATE_LOG_REPLICATION_FACTOR: "1"
      KAFKA_CFG_PROCESS_ROLES: "controller,broker" #KRaft
      KAFKA_CFG_NODE_ID: "0" #KRaft
      KAFKA_CFG_CONTROLLER_LISTENER_NAMES: "CONTROLLER" #KRaft
      KAFKA_CFG_CONTROLLER_QUORUM_VOTERS: "0@kafka:9093" #KRaft
    volumes:
      - kafka-data:/bitnami
volumes:
  kafka-data:
    driver: local
EOF
```

Configure ThingsBoard Edge to use Kafka:

```text
echo -e "export TB_QUEUE_TYPE=kafka\nexport TB_KAFKA_SERVERS=localhost:9092" | sudo tee -a /etc/tb-edge/conf/tb-edge.conf > /dev/null
```

Replace `localhost:9092` with your actual Kafka bootstrap server address if necessary.

</TabItem>
</Tabs>

## Troubleshooting

ThingsBoard Edge logs are stored in:

```bash
/var/log/tb-edge
```

To check for errors:

```bash
cat /var/log/tb-edge/tb-edge.log | grep ERROR
```

### Service management

To stop the ThingsBoard Edge service:

```bash
sudo service tb-edge stop
```

To restart the ThingsBoard Edge service:

```bash
sudo service tb-edge restart
```

To check the status of the ThingsBoard Edge service:

```bash
sudo service tb-edge status
```
