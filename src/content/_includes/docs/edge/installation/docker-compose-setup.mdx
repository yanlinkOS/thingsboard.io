ThingsBoard Edge clustering connects multiple Edge nodes to a shared database and syncs aggregated data to ThingsBoard Server or Cloud. It is designed for large-scale deployments — factories, smart cities, or any site with thousands of devices in one region.

<ImageGallery images={[
  {

    alt: 'Edge cluster architecture: three Edge nodes sharing a database and syncing to ThingsBoard Cloud',
    caption: 'Edge cluster architecture',
  },
]} />

**Key properties of an Edge cluster:**
- All Edge nodes connect to the same database.
- Devices connect to any node based on load balancing.
- Edge nodes share workload and maintain local failover.
- The cluster syncs aggregated data to ThingsBoard Server or Cloud.


For details about microservices architecture, see the <a href="/docs/reference/architecture/microservices/" target="_blank">microservices architecture</a> page.

## Prerequisites

ThingsBoard Edge microservices run in a Dockerized environment. Install <a href="https://docs.docker.com/engine/install/" target="_blank" rel="noopener noreferrer">Docker CE</a> and <a href="https://docs.docker.com/compose/install/" target="_blank" rel="noopener noreferrer">Docker Compose</a> before continuing.

<Aside type="note">
  ThingsBoard supports **Docker Compose V2** (Docker Desktop or Compose plugin). Upgrade to Docker Compose V2 — Docker no longer supports the standalone `docker-compose` tool.
</Aside>

You also need an active **ThingsBoard{props.product === Products.EDGE_PE ? ' Professional Edition' : ' Community Edition'}** account with an Edge instance created on the server.

## Step 4. Create host volumes

Create the log folders and set correct ownership:

```bash
./docker-create-log-folders.sh
```

Verify that all required volume folders exist with correct ownership:

```bash
./docker-check-log-folders.sh
```

<Aside type="note" title="macOS with Docker Desktop">
  If you use Docker Desktop on macOS with <a href="https://docs.docker.com/desktop/features/synchronized-file-sharing/" target="_blank" rel="noopener noreferrer">Synchronized file sharing</a> (enabled by default for <code>/Users</code> subdirectories), pass the <code>--skipChown</code> flag — ownership is managed automatically by the virtualization engine:

  ```bash
  ./docker-create-log-folders.sh --skipChown
  ./docker-check-log-folders.sh --skipChown
  ```
</Aside>

## Step 5. Install and start ThingsBoard Edge

Run the installation script:

```bash
./docker-install-tb.sh
```

Start all services:

```bash
./docker-start-services.sh
```

<Aside type="note">
  Services take a few minutes to start. Once running, open the ThingsBoard Edge UI at <code>http://&#123;your-host-ip&#125;</code> (for example, <a href="http://localhost" target="_blank" rel="noopener noreferrer">http://localhost</a>). Log in with your ThingsBoard account credentials.
</Aside>

To view Edge node logs:

```bash
docker compose logs -f tb-edge1 tb-edge2 tb-edge3
```

To check the status of all containers:

```bash
docker compose ps
```

To view logs for all running services:

```bash
docker compose logs -f
```

### Stop and remove containers

To stop all services:

```bash
./docker-stop-services.sh
```

To stop and remove all deployed containers:

```bash
./docker-remove-services.sh
```

## Upgrading

To update one or more services (pulls a newer Docker image and rebuilds the container):

```bash
./docker-update-service.sh [SERVICE...]
```

Omit `[SERVICE...]` to update all services.

To upgrade the database:

```bash
./docker-stop-services.sh
./docker-upgrade-tb.sh
./docker-start-services.sh
```

## HTTPS with a valid certificate

ThingsBoard Edge uses HAproxy for traffic proxying, with ports 80 and 443 for the web UI. To obtain a valid TLS certificate via Certbot:

```bash
docker exec haproxy-certbot certbot-certonly --domain your_domain --email your_email
docker exec haproxy-certbot haproxy-refresh
```

<Aside type="note">
  A valid certificate is used only when you access the UI by domain name. Accessing by IP address falls back to a self-signed certificate.
</Aside>
