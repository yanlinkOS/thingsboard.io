ThingsBoard Edge can run alongside the <DocLink product={Products.GW} path="" target="_blank">ThingsBoard IoT Gateway</DocLink> on the same device. The IoT Gateway collects data from local sensors and legacy protocols, translates it into ThingsBoard-compatible format, and sends it to the Edge. The Edge rule engine processes it in real time and synchronizes results with the central server.

<DocImage src="/src/assets/images/docs/edge/user-guide/iot-gateway/how-does-it-work.webp" alt="Architecture diagram — IoT Gateway sends data to Edge, Edge synchronizes with the ThingsBoard server" width="600px" />

<Aside type="note">
When deployed in a remote location with an unreliable internet connection, Edge continues operating until connectivity is restored.
</Aside>

## Prerequisites

- <a href="https://docs.docker.com/engine/install/" target="_blank" rel="noopener noreferrer">Docker</a> installed on the IoT Gateway device.
- A ThingsBoard server account (Cloud, Professional Edition, or Community Edition).
- A Linux-based IoT Gateway device with SSH access.

<Aside type="caution">
ThingsBoard Edge uses port **8080** for its web UI, and the ThingsBoard IoT Gateway also defaults to port **8080**. When both run on the same device, one of them must use a different port. Configure this in the respective `docker-compose.yml` file before starting the services.
</Aside>

## Step 2. Install Edge on the IoT Gateway device

Connect to the device over SSH:

```bash
ssh username@ip_address
```

Replace `username` and `ip_address` with your device credentials. If SSH runs on a non-default port, add `-p <port>` to the command.

<Steps>
1. Create a directory for the Edge service:

   ```bash
   mkdir tb_edge
   ```

2. Inside that directory, create a `docker-compose.yml` file:

   ```bash
   nano docker-compose.yml
   ```

3. On the server, go to **Edge management → Instances**, click the Edge instance, and click **Install & Connect Instructions**.

4. Select the **Docker** tab, copy the configuration lines, paste them into `docker-compose.yml`, and save the file.

   <ImageGallery images={[
     {
       src: '/src/assets/images/docs/edge/user-guide/iot-gateway/1.2-instrucrions-button.webp',
       alt: 'Edge details page with Install & Connect Instructions button',
       caption: 'Click Install & Connect Instructions on the Edge details page.',
     },
     {
       src: '/src/assets/images/docs/edge/user-guide/iot-gateway/1.3-docker.webp',
       alt: 'Install & Connect Instructions dialog with Docker tab selected',
       caption: 'Select the Docker tab and copy the configuration lines.',
     },
     {
       src: '/src/assets/images/docs/edge/user-guide/iot-gateway/paste-yml-terminal.webp',
       alt: 'Terminal with docker-compose.yml open and configuration pasted',
       caption: 'Step 4: Paste the configuration into docker-compose.yml and save. Press CTRL+X to close.',
     },
   ]} />

5. Start the Edge container:

   ```bash
   docker compose up -d
   docker compose logs -f mytbedge
   ```

   Run both commands from the `tb_edge` directory.
</Steps>

To verify Edge started successfully, log in to the Edge UI and go to **Edge → Status**. The **Status** field should show **Connected**.

To access the Edge UI from your local machine, open another terminal tab and set up SSH port forwarding:

```bash
ssh -N -L 8080:127.0.0.1:8080 username@ip_address
```

- **`8080` (first):** The port on your local machine.
- **`127.0.0.1`:** The target host accessible from the IoT Gateway device.
- **`8080` (second):** The port on the IoT Gateway device.
- **`username`:** The username with access to the device.
- **`ip_address`:** The IP address of the device.

Any connection to `localhost:8080` on your local machine is forwarded to `127.0.0.1:8080` on the IoT Gateway device. Open <a href="http://127.0.0.1:8080" target="_blank" rel="noopener noreferrer">http://127.0.0.1:8080</a> and log in with your tenant credentials.

## Step 3. Provision the IoT Gateway on Edge

<Steps>
1. Log in to your ThingsBoard server, go to **Edge management → Instances**, and click **Manage dashboards** next to the Edge instance.

2. Click **Add** (+), select the **ThingsBoard IoT Gateways** dashboard from the drop-down menu, and click **Assign**. The **ThingsBoard IoT Gateways** dashboard is a built-in dashboard available on every ThingsBoard installation.

   <Aside type="note">
   If the **ThingsBoard IoT Gateways** dashboard is not available in the list, <a href="/downloads/thingsboard_iot_gateways.json" download target="_blank" rel="noopener noreferrer">download the ThingsBoard IoT Gateways dashboard</a> and import it via **Dashboards → Import dashboard** before assigning it.
   </Aside>

3. Open the Edge instance at <a href="http://127.0.0.1:8080" target="_blank" rel="noopener noreferrer">http://127.0.0.1:8080</a>, log in with your tenant credentials, and go to **Dashboards**. Open the **ThingsBoard IoT Gateways** dashboard.

4. Click **Add** (+), enter the gateway name in the **Name** field, select the **default** device profile, and click **Create**.

5. Download the `docker-compose.yml` file for the IoT Gateway from the **Launch command** pop-up window.
</Steps>

## Step 4. Install the IoT Gateway software on the device

Connect to the device over SSH:

```bash
ssh username@ip_address
```

<Steps>
1. Create a directory for the IoT Gateway service:

   ```bash
   mkdir iot_gateway
   ```

2. Inside that directory, create a `docker-compose.yml` file:

   ```bash
   nano docker-compose.yml
   ```

3. Paste the contents of the downloaded `docker-compose.yml` into the new file and save it. Press **CTRL+X** to close.

   <ImageGallery images={[
     {
       src: '/src/assets/images/docs/edge/user-guide/iot-gateway/10-copy-paste-configs.webp',
       alt: 'Terminal with docker-compose.yml open and IoT Gateway configuration pasted',
       caption: 'Paste the IoT Gateway configuration and press CTRL+X to save and close.',
     },
   ]} />

4. Start the IoT Gateway:

   ```bash
   docker compose up -d
   docker compose logs -f
   ```
</Steps>

To verify the IoT Gateway connected successfully, open the **ThingsBoard IoT Gateways** dashboard on the Edge instance. The gateway device should appear in the list with an active status.
