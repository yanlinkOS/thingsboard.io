**White labeling** lets businesses customize the ThingsBoard Edge interface — logo, colors, and branding — to match their own corporate identity. Settings propagate to Edge instances through an ownership hierarchy.

<Aside type="note">
  White labeling is a Professional Edition feature. For full configuration instructions, see the <DocLink product={Products.PE} path="user-guide/white-labeling" target="_blank">ThingsBoard White Labeling</DocLink> documentation.
</Aside>

## Hierarchy

White-labeling settings follow an ownership hierarchy: system administrator settings can be overridden by the tenant, which can be overridden by the customer, and so on.

<ImageGallery images={[
  {
    src: '/src/assets/images/docs/edge/user-guide/white-labeling/hierarchy.svg',
    alt: 'White-labeling hierarchy diagram',
    caption: 'White-labeling settings are inherited and can be overridden at each level.',
  },
]} />

This hierarchy extends to Edge instances:

- **Tenant-owned Edge** — white-labeling settings are taken from the **tenant level**, merged with those from the system administrator.

<ImageGallery images={[
  {
    src: '/src/assets/images/docs/edge/user-guide/white-labeling/hierarchy-edge-tenant.svg',
    alt: 'Tenant-owned Edge white-labeling hierarchy',
    caption: 'Tenant-owned Edge: settings merged from system administrator and tenant.',
  },
]} />

- **Customer-owned Edge** — white-labeling settings are taken from the **customer**, merged with those from the tenant and system administrator.

<ImageGallery images={[
  {
    src: '/src/assets/images/docs/edge/user-guide/white-labeling/hierarchy-edge-customer.svg',
    alt: 'Customer-owned Edge white-labeling hierarchy',
    caption: 'Customer-owned Edge: settings merged from all three levels.',
  },
]} />

The displayed interface depends on the **account** used to log in. For example, if a user logs in to a customer-owned Edge with a **tenant account**, they will see the tenant's interface.

<ImageGallery images={[

    alt: 'White-labeled Edge interface example',
    caption: 'The interface shown is determined by the account used to log in, not the Edge ownership.',
  },
]} />

## Edge login page

On the **Edge**, domain-based branding is **not used**. The appearance of the Edge login page depends on **ownership only**.

For example, when a user navigates to a customer-owned Edge, the customer's white-labeling login configuration is applied.
