Edge devices are physical IoT entities — sensors, actuators, switches, and similar hardware — that collect and process data locally on the Edge instance. They work the same way as devices on the ThingsBoard server: they report telemetry, respond to RPC commands, and belong to device profiles.

For a general introduction to devices in ThingsBoard, see <DocLink product={props.product} path="user-guide/devices" target="_blank">Devices</DocLink>.

There are two ways to get a device onto an Edge instance:

- **Create on Edge:** Use this when the device is new and will operate primarily on Edge. ThingsBoard pushes it to the server automatically.
- **Assign from server:** Use this when the device already exists on the server and you want to extend its management to a specific Edge instance.

## Create a device on Edge

Devices created on the Edge instance are automatically pushed to the server once the connection is established. Any subsequent changes are synchronized to the server without additional action.

<Aside type="note">
Before creating a device, make sure the target <DocLink product={props.product} path="user-guide/device-profiles" target="_blank">device profile</DocLink> is available on the Edge instance. Device profiles are created on the server and must be assigned to the Edge instance before they appear in the Edge UI.
</Aside>

To create a device manually:

<Steps>
1. Go to **Entities → Devices** and click **Add new device** (+).

2. Enter the device **Name** and select a **Device profile**, then click **Add**.

3. Verify connectivity by sending a telemetry message from the device using the shell command shown in the on-screen instructions. This step is optional.

4. Log in to your ThingsBoard server and go to **Entities → Devices** to confirm the new device was pushed to the server automatically.

   If the Edge instance was offline when you created the device, the push happens automatically as soon as the connection to the server is restored.
</Steps>

## Assign a device from the server

Use this flow when the device already exists on the server and you want to make it available on a specific Edge instance. The device configuration and history remain on the server; Edge receives a synchronized copy.

{props.product === Products.EDGE && (
  <Steps>
    <ol>
      <li>Log in to your ThingsBoard server and go to <strong>Edge management → Instances</strong>.</li>
      <li>Open the Edge instance and click <strong>Manage devices</strong>.</li>
      <li>Click <strong>Add</strong> (+), select the device from the dropdown, and click <strong>Assign</strong>.</li>
      <li>Log in to your Edge instance and go to <strong>Entities → Devices</strong> to confirm the device was assigned.</li>
    </ol>
  </Steps>
)}

{props.product === Products.EDGE_PE && (
  <Steps>
    <ol>
      <li>Log in to your ThingsBoard server and go to <strong>Edge management → Instances</strong>.</li>
      <li>Open the Edge instance and click <strong>Manage edge device groups</strong>.</li>
      <li>Click <strong>Add</strong> (+), select the device group from the dropdown, and click <strong>Assign</strong>.</li>
      <li>Open the assigned group, click <strong>Add</strong> (+), fill in the device <strong>Name</strong> and <strong>Device profile</strong>, and click <strong>Add</strong>.</li>
      <li>Log in to your Edge instance and go to <strong>Entities → Devices</strong> to confirm the device and group were assigned.</li>
    </ol>
  </Steps>
)}

## Unassign a device from Edge

Unassigning removes the device from the Edge instance but keeps it on the server. All telemetry and attributes collected on Edge remain on the server.

<Steps>
1. Log in to your ThingsBoard server and go to **Edge management → Instances**.

2. Open the Edge instance and click **Manage devices**.

3. Find the device in the list and click **Unassign from edge** (×).

   The device is removed from the Edge instance immediately. It remains fully accessible on the server.
</Steps>
