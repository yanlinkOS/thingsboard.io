Dashboards on ThingsBoard Edge work the same way as on the ThingsBoard server. They display real-time IoT data from devices connected locally to the Edge, and continue working even when the Edge instance is offline — widgets show the most recently received data during an outage. When the connection is restored, any changes made on Edge are synchronized to the server automatically.

For a general introduction to dashboards and widgets, see <DocLink product={props.product} path="user-guide/dashboards" target="_blank">Dashboards</DocLink>.

## Assign a dashboard from the server

Dashboards created on the ThingsBoard server can be assigned to one or more Edge instances. This is the recommended approach when you want to deploy the same dashboard layout to multiple Edge instances simultaneously.

<Aside type="note">
The dashboard must already exist on the ThingsBoard server before you can assign it. For details on creating server-side dashboards, see <DocLink product={props.product} path="user-guide/dashboards" target="_blank">Dashboards</DocLink>.
</Aside>

{props.product === Products.EDGE && (
  <Steps>
    <ol>
      <li>Log in to the ThingsBoard server and go to <strong>Edge management → Instances</strong>.</li>
      <li>Click the Edge instance to open the context menu and select <strong>Manage dashboards</strong>.</li>
      <li>Click <strong>+</strong>, select the dashboard from the dropdown, and click <strong>Assign</strong>.<br />
        The dashboard is added to the Edge event queue and appears on Edge once the connection is established.
      </li>
    </ol>
  </Steps>
)}

{props.product === Products.EDGE_PE && (
  <Steps>
    <ol>
      <li>Log in to the ThingsBoard server and go to <strong>Edge management → Instances</strong>.</li>
      <li>Click the Edge instance to open the context menu and select <strong>Manage edge dashboard groups</strong>.</li>
      <li>Click <strong>+</strong> to assign an existing dashboard group, or open a group and click <strong>+</strong> to add individual dashboards to it.<br />
        The dashboards are added to the Edge event queue and appear on Edge once the connection is established.
      </li>
    </ol>
  </Steps>
)}

To verify the assignment, log in to the Edge instance and go to **Dashboards**. The assigned dashboards appear in the list.

## Create a dashboard on Edge

You can also create dashboards directly on the Edge instance. Dashboards created on Edge are automatically propagated to the ThingsBoard server when the connection is established.

<Steps>
1. Log in to the ThingsBoard Edge instance and go to **Dashboards**.

2. Click **Add** (+) and select **Create new dashboard**. To upload a previously exported dashboard JSON file instead, select **Import dashboard**.

3. Enter the dashboard **Title** and click **Add**.

4. Add widgets using **+ Add widget**, then click **Save**.

   The dashboard appears in the list and is pushed to the server automatically.
</Steps>

<Aside type="note">
If you have multiple Edge instances, deleting a dashboard on one Edge instance does not affect the others. To remove a dashboard from all Edge instances at once, delete it from the ThingsBoard server.
</Aside>

## Make a dashboard public

A public dashboard is accessible to anyone with the link, without logging in. On Edge, the link points to the Edge instance's local URL, so the dashboard remains available on the local network even if the internet connection is lost.

<Aside type="note">
For data to appear on a public dashboard, the devices it displays must also be made public.
</Aside>

For a general introduction to entity views, see <DocLink product={props.product} path="user-guide/provision-entity-view" target="_blank">Entity views</DocLink>.

## Remove a dashboard from Edge

Removing a dashboard unassigns it from the Edge instance only. The dashboard remains on the server and can be reassigned at any time.

{props.product === Products.EDGE && (
  <Steps>
    <ol>
      <li>Log in to the ThingsBoard server and go to <strong>Edge management → Instances</strong>.</li>
      <li>Click the Edge instance to open the context menu and select <strong>Manage dashboards</strong>.</li>
      <li>Find the dashboard in the list and click <strong>Unassign from edge</strong> (×).<br />
        The dashboard is removed from the Edge instance on the next synchronization.
      </li>
    </ol>
  </Steps>
)}

{props.product === Products.EDGE_PE && (
  <Steps>
    <ol>
      <li>Log in to the ThingsBoard server and go to <strong>Edge management → Instances</strong>.</li>
      <li>Click the Edge instance to open the context menu and select <strong>Manage edge dashboard groups</strong>.</li>
      <li>Find the dashboard group in the list and remove it, or open the group and remove individual dashboards.<br />
        The dashboards are removed from the Edge instance on the next synchronization.
      </li>
    </ol>
  </Steps>
)}
