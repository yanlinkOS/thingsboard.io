Defines how physical hardware is represented in the cloud.

- <DocLink product={props.product} path='user-guide/devices/'>Devices:</DocLink> Physical or virtual entity publishing telemetry or receiving commands, like **Sensors** or **Actuators**.
- <DocLink product={props.product} path='user-guide/TBD/'>Assets:</DocLink> Abstract entity used to group devices into logical structures: **Factory Floor**, **Building**, etc
- <DocLink product={props.product} path='user-guide/TBD/'>Relations:</DocLink> Connections between entities, allowing you to build parent-child hierarchies like **Building** (Asset) → **Floor** (Asset) → **Thermostat** (Device).
- <DocLink product={props.product} path='user-guide/TBD/'>Attributes:</DocLink> Entity metadata: **serial number**, **firmware version**, **location**, etc.
- <DocLink product={props.product} path='user-guide/TBD/'>Telemetry:</DocLink> Time-series data points: **temperature**, **humidity**, **voltage**, etc.
- <DocLink product={props.product} path='user-guide/TBD/'>Commands:</DocLink> Send commands from the platform back to the device and vice-versa.
