**Relations** connect two entities within the same <DocLink product={props.product} path='TODO'>Tenant</DocLink>, modeling real-world relationships such as hierarchy, ownership, or dependency.

Each relation has:
- A **direction**: _From_ (source) → _To_ (target)
- A **type**: a free-text string (e.g., _Contains_, _Manages_, or any custom value)

## Direction

A relation always flows from one entity to another. The **From** entity is the source; the **To** entity is the target.

**Example:** To express that _Region A_ contains _Field A_:
- **From**: Region A (Asset)
- **To**: Field A (Asset)
- **Type**: Contains

Direction matters: querying relations _from_ Region A lists its children; querying relations _to_ Field A reveals its parent.

## Relation Types

The type is a free-text string — use built-in conventions or define your own:

| Type | Meaning |
|------|---------|
| Contains | A hierarchically contains B |
| Manages | A manages or is responsible for B |
| Uses | A depends on or uses B |
| Supports | A provides abstract support to B |

Any string is valid: `Contains`, `Manages`, `AbsolutelyNewRelationType` — ThingsBoard places no restrictions on type names.

## Creating Relations

1. Open the entity details page.
2. Go to the **Relations** tab.
3. Select the relation **direction** (**From** or **To**).
4. Click **+** to open the **Add relation** dialog.
5. Enter the **Relation type** — type any string, for example `Contains` or a custom value like `AbsolutelyNewRelationType`.
6. Select the **target entity**.
7. Optionally fill in **Additional info (JSON)** for metadata.
8. Click **Add**.

## Hierarchy Example

The _Contains_ relation models parent–child structures. For example:

- Region A → **Contains** → Field A
- Region A → **Contains** → Field B
- Field A → **Contains** → Moisture Sensor
- Field B → **Contains** → Irrigation System

## REST API

Relations can be created, queried, and deleted via the <DocLink product={props.product} path='TODO'>REST API</DocLink>. A relation is represented as:

```json
{
  "from": { "id": "156c853f-89b1-321b-635j-y8febb2784h4", "entityType": "DEVICE" },
  "to":   { "id": "784f394c-42b6-435a-983c-b7beff2784f9", "entityType": "DEVICE" },
  "type": "Contains",
  "additionalInfo": {}
}
```

Fields:
- **from** / **to** — entity type and UUID of the source and target
- **type** — the relation type string
- **additionalInfo** — optional custom JSON metadata
