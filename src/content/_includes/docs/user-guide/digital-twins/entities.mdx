ThingsBoard models the physical world as a graph of **entities** connected by **relations**. Every entity supports attributes, time-series data, and alarm rules — giving you a unified way to represent, monitor, and control your IoT infrastructure.

<ImageGallery images={[{ src: '/src/assets/schemas/entities.svg', alt: 'ThingsBoard entity hierarchy: Tenant → Customers → Assets → Devices' }]} />

## Core Entities

| Entity | Purpose |
|--------|---------|
| <DocLink product={props.product} path='user-guide/multi-tenancy' bold={true} target="_blank">Tenant</DocLink> | Top-level business entity that owns all other entities |
| <DocLink product={props.product} path='user-guide/customers' bold={true} target="_blank">Customer</DocLink> | Organization or individual that uses tenant <DocLink product={props.product} path='user-guide/devices' bold={true} target="_blank">devices</DocLink> and <DocLink product={props.product} path='user-guide/assets' bold={true} target="_blank">assets</DocLink> |
| <DocLink product={props.product} path='user-guide/users' bold={true} target="_blank">User</DocLink> | Person who logs in to browse dashboards and manage entities |
| <DocLink product={props.product} path='user-guide/devices' bold={true} target="_blank">Device</DocLink> | Physical or virtual thing that publishes telemetry and handles RPC commands |
| <DocLink product={props.product} path='user-guide/assets' bold={true} target="_blank">Asset</DocLink> | Abstract object (building, field, vehicle) used to group and aggregate data |
| <DocLink product={props.product} path='user-guide/entity-views' bold={true} target="_blank">Entity View</DocLink> | Read-only projection of a device or asset that exposes selected telemetry and attributes |
| <DocLink product={props.product} path='user-guide/dashboards' bold={true} target="_blank">Dashboard</DocLink> | Visualization of IoT data with interactive widgets |

Some entity types support **profiles** that define shared configuration for a group of similar entities:

| Profile | Applies to | Configures |
|---------|-----------|------------|
| **<DocLink product={props.product} path='user-guide/device-profiles' bold={false} target="_blank">Device Profile</DocLink>** | Devices | Transport protocol, rule chain, queue, alarm rules, provisioning |
| **<DocLink product={props.product} path='user-guide/asset-profiles' bold={false} target="_blank">Asset Profile</DocLink>** | Assets | Rule chain, queue, alarm rules |

## Entity Data

Every entity supports three types of data:

| Data type | Description | Example |
|-----------|-------------|---------|
| **<DocLink product={props.product} path='user-guide/digital-twins/attributes' bold={false} target="_blank">Attributes</DocLink>** | Static or semi-static key-value pairs | Serial number, firmware version, location |
| **<DocLink product={props.product} path='user-guide/digital-twins/time-series-data' bold={false} target="_blank">Time-series</DocLink>** | Timestamped data points for storage, querying, and visualization | Temperature, humidity, battery level |
| **<DocLink product={props.product} path='user-guide/digital-twins/relations' bold={false} target="_blank">Relations</DocLink>** | Directed connections between entities | Contains, Manages, Produces |

Relations help model real-world hierarchies — for example, a region that contains fields, each containing multiple sensors. See <DocLink product={props.product} path='user-guide/digital-twins/relations' target="_blank">Relations</DocLink> for details on creating, querying, and using relations in rule chains and dashboards.
