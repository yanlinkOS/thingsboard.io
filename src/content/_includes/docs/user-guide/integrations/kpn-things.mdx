KPN Things is an IoT platform provided by KPN, a Dutch telecommunications company.
It supports a range of IoT connectivity types — including LoRa, LTE-M, and mobile data — and lets users connect, manage, and route device data to cloud destinations via a concept called **Flows**.
After integrating KPN Things with ThingsBoard, you can connect, communicate, process, and visualize data from your devices in the ThingsBoard IoT platform.

## Create the KPN Things integration in ThingsBoard

<Steps>
  <ol start="1">
    <li>Go to <strong>Integrations center &#8702; Integrations</strong> and click <strong>+ Add integration</strong>.</li>
    <li>
      <span><strong>Basic settings:</strong></span>
      <ul>
        <li>Set <strong>Integration type</strong> to <strong>KPN Things</strong>.</li>
        <li><strong>Enable integration</strong> and <strong>Allow create devices or assets</strong> are on by default.</li>
        <li>Click <strong>Next</strong>.</li>
      </ul>
    </li>
    <li>
      <span><strong>Uplink data converter:</strong></span>
      <ul>
        <li>Choose <strong>Create new</strong>. The default converter uses <code>decodeToJson(payload)</code> — KPN Things sends data pre-formatted in a structure ThingsBoard already understands, so no custom decoding is needed unless you want to transform the fields further.</li>
        <li>Click <strong>Next</strong>.</li>
      </ul>
    </li>
    <li>
      <span><strong>Downlink data converter:</strong></span>
      <ul>
        <li>Leave the <strong>Downlink data converter</strong> field empty for now.</li>
        <li>Click <strong>Skip</strong>.</li>
      </ul>
    </li>
    <li>
      <span><strong>Connection:</strong></span>
      <ul>
        <li>Specify your <strong>Base URL</strong>.</li>
        <li>Copy the <strong>HTTP endpoint URL</strong> — you will need it when configuring the KPN Things destination.</li>
      </ul>
      <span>See <a href="#connection-settings">Connection settings</a> for a full description of each parameter.</span>
    </li>
    <li>Click <strong>Add</strong>.</li>
  </ol>
</Steps>

### Connection settings

<h4>Base URL</h4>

The base address of the ThingsBoard server, used to construct the HTTP endpoint URL. For ThingsBoard Cloud, this is `https://thingsboard.cloud`.

<h4>Destination shared secret</h4>

Optional secret token used to authenticate KPN Things requests arriving at ThingsBoard. Click the generate icon to create a value, then configure KPN Things to include it in outgoing requests so ThingsBoard can verify the source.

<h4>HTTP endpoint URL</h4>

Auto-generated endpoint for this integration. Paste this URL into the KPN Things ThingsBoard destination so that KPN Things forwards device uplink messages to ThingsBoard.

<h4>Allow downlink</h4>

Enable to allow ThingsBoard to send commands back to devices via KPN Things. When toggled on, additional **Credentials** fields appear:

- **GRP tenant id** — the KPN Things tenant identifier
- **API id** — the API client ID used to authenticate ThingsBoard against the KPN Things API
- **API Key** — the corresponding API secret key

<h4>Enable security (Headers filter)</h4>

Enable to restrict incoming requests to those carrying a specific HTTP header with a matching value. Click **Add** to define one or more **Header** / **Value** pairs. Requests missing any listed header or carrying an incorrect value are rejected. Use this to validate the `X-Things-Secret` header generated by KPN Things — see [Update integration with security header](#update-integration-with-security-header).

<h4>Execute remotely</h4>

When enabled, ThingsBoard generates an **Integration key** and **Integration secret** that allow the integration to run as a separate process outside the ThingsBoard cluster — useful when the integration must reach services not accessible from the ThingsBoard server.

<h4>Advanced settings</h4>

- **Description** — optional text description for this integration
- **Metadata** — key-value pairs injected into every uplink message as `integrationMetadata` in converter scripts

## KPN Things setup

Open the <a href="https://portal.kpnthings.com/" target="_blank">KPN Things Portal</a> and log in to your account.

<Aside type="note">
The KPN Things Portal interface may change in the future.
</Aside>

### Add a new device

<Steps>
1. Go to the **Things Manager** tab and click **Add new Device**.
2. Choose **Device Simulator**, set a name for your device, and click **Add Device**.
3. Install the **KPN Things Device Simulator** app on your phone. Scan the QR code displayed on screen with the app. Note that the **Shared Secret** is only visible at this step — save it in a secure place. Click **Add Network Info**.
4. KPN Internet Network is created. Click **Finish**.
</Steps>

### Link the device to the flow

<Steps>
1. Go to **Flows** in the left menu and click the **My first flow** row.

4. The device is successfully linked to the flow.

### Enable data processing

<Steps>
1. Click the **My Device** row to open the Device & Connectivity detail page.
2. Go to the **Flows** tab and click the **My first flow** row.
3. Click **Data Processing** to open the Data Processing details page.
4. Enable the **Decoded ThingsML and raw SenML data** decoder toggle. Your change is saved automatically.
</Steps>

### Add a new destination

<Steps>
1. Go to **Destinations** in the left menu and click **Add new Destination**.
2. Find the **ThingsBoard** destination type and click **Choose ThingsBoard**.
3. Enter a name for your destination.
4. Paste the **HTTP endpoint URL** copied from the ThingsBoard integration into the **HTTP endpoint URL** field.
5. In the **Security Header** section, the **Header** name is pre-filled as `X-Things-Secret`. Click **Generate** to create the token value and save it — you will add it to ThingsBoard in the next step to authenticate incoming requests.
6. Click **Add ThingsBoard Destination**.
</Steps>

## Update integration with security header

To authenticate incoming requests from KPN Things, add the generated `X-Things-Secret` token as a security header in the ThingsBoard integration.

<Steps>
1. Go to **Integrations center &#8702; Integrations**, open the KPN Things integration, and click the edit icon.
2. Expand **Enable security (Headers filter)** and click **Add**.
3. Enter `X-Things-Secret` in the **Header** field and paste the generated token into the **Value** field. Click **Apply changes**.
</Steps>
