Azure IoT Hub Integration connects ThingsBoard to Azure IoT Hub over MQTT, receiving device messages and converting them into ThingsBoard telemetry and attributes.

<DocImage src="/src/assets/images/user-guide/integrations/azure-iot-hub/iot-hub-integration.svg" alt="Azure IoT Hub integration overview diagram" />

## Prerequisites

Before creating the integration in ThingsBoard, set up your Azure IoT Hub and register a device:

<Steps>
1. <a href="https://docs.microsoft.com/en-us/azure/iot-hub/iot-hub-create-through-portal#create-an-iot-hub" target="_blank" rel="noopener noreferrer">Create an IoT hub</a> in the Azure portal.
2. <a href="https://docs.microsoft.com/en-us/azure/iot-hub/iot-hub-create-through-portal#register-a-new-device-in-the-iot-hub" target="_blank" rel="noopener noreferrer">Register a new device</a> in the IoT hub. Note the device ID and SAS key — you will need them when configuring the integration.
</Steps>

## ThingsBoard integration setup

### Create an uplink converter

The uplink converter transforms incoming Azure IoT Hub messages into the ThingsBoard data format. Azure IoT Hub uses a <DocLink product={props.product} path="user-guide/integrations/uplink-data-converter">generic uplink converter</DocLink>. ThingsBoard provides a pre-built converter for Azure IoT Hub — import the JSON file and customize the decoder function to match your device payload structure.

<Aside type="tip" title="TBEL">
  We recommend using <DocLink product={props.product} path="user-guide/tbel">TBEL</DocLink> for converter scripts — it executes much more efficiently than JavaScript inside ThingsBoard.
</Aside>

**Sample payload:**
```json
{
  "devName": "T1",
  "msg": {
    "temp": 42,
    "humidity": 77
  }
}
```

The `devName` field maps to the device name. The nested `msg` object provides `temperature` and `humidity` telemetry.

<Steps>
  <ol>
    <li>
      <span>Download the uplink converter file:</span>
      <Tabs>
        <TabItem label="TBEL (Recommended)">
          <a href="/resources/docs/integrations/azure-iot-hub/azure_iot_hub_tbel_uplink_converter.json" download="azure_iot_hub_tbel_uplink_converter.json"><strong>azure_iot_hub_tbel_uplink_converter.json</strong></a>
        </TabItem>
        <TabItem label="JavaScript">
          <a href="/resources/docs/integrations/azure-iot-hub/azure_iot_hub_javascript_uplink_converter.json" download="azure_iot_hub_javascript_uplink_converter.json"><strong>azure_iot_hub_javascript_uplink_converter.json</strong></a>
        </TabItem>
      </Tabs>
    </li>
    <li>Go to **Integrations center &#8702; Data converters**.</li>
    <li>Click **+ Add data converter &#8702; Import converter**.</li>
    <li>Drag and drop the downloaded JSON file into the **Import converter** window and click **Import**.</li>
  </ol>
</Steps>

**The decoder function used in this tutorial:**

<Tabs>
<TabItem label="TBEL (Recommended)">
```js maxLines=15 collapsible
var data = decodeToJson(payload);
var deviceName = data.devName;
var deviceType = 'thermostat';

var result = {
    deviceName: deviceName,
    deviceType: deviceType,
    telemetry: {
        temperature: data.msg.temp,
        humidity: data.msg.humidity
    }
};

return result;
```
</TabItem>
<TabItem label="JavaScript">
```js maxLines=20 collapsible
var data = decodeToJson(payload);
var deviceName = data.devName;
var deviceType = 'thermostat';

var result = {
    deviceName: deviceName,
    deviceType: deviceType,
    telemetry: {
        temperature: data.msg.temp,
        humidity: data.msg.humidity
    }
};

function decodeToString(payload) {
    return String.fromCharCode.apply(String, payload);
}

function decodeToJson(payload) {
    var str = decodeToString(payload);
    var data = JSON.parse(str);
    return data;
}

return result;
```
</TabItem>
</Tabs>

**To adapt this converter to your device:**

- **Different device name** — change `data.devName` to the field in your payload that carries the device identifier.
- **Different payload structure** — adjust the field paths (e.g. `data.msg.temp`) to match your payload layout.
- **Additional telemetry** — add more keys to the `telemetry` object (e.g. `pressure: data.msg.pressure`).
- **Attributes** — add an `attributes` key alongside `telemetry` to store static device properties.

### Create the integration

<Steps>
  <ol start="1">
    <li>Go to <strong>Integrations center &#8702; Integrations</strong> and click <strong>+ Add integration</strong>.</li>
    <li>
      <span><strong>Basic settings:</strong></span>
      <ul>
        <li>Set <strong>Integration type</strong> to <strong>Azure IoT Hub</strong>.</li>
        <li><strong>Enable integration</strong> and <strong>Allow create devices or assets</strong> are on by default.</li>
        <li>Click <strong>Next</strong>.</li>
      </ul>
    </li>
    <li>
      <span><strong>Uplink data converter:</strong></span>
      <ul>
        <li>Click <strong>Select existing</strong> and choose the <a href="#create-an-uplink-converter">previously imported</a> <code>Azure IoT Hub Uplink Converter</code> from the list.</li>
        <li>Click <strong>Next</strong>.</li>
      </ul>
    </li>
    <li>
      <span><strong>Downlink data converter:</strong></span>
      <ul>
        <li>Leave empty and click <strong>Skip</strong> — the downlink converter can be added later if needed.</li>
      </ul>
    </li>
    <li>
      <span><strong>Connection settings:</strong></span>
      <ul>
        <li><strong>Hostname</strong> — your Azure IoT Hub hostname (e.g. <code>tbot.azure-devices.net</code>).</li>
        <li><strong>Device ID</strong> — the device ID registered in Azure IoT Hub (e.g. <code>T1</code>).</li>
        <li><strong>Credentials</strong> — select <strong>Shared Access Signature</strong> and enter the device SAS key. See <a href="#credentials">Credentials</a> for X.509 certificate options.</li>
        <li><strong>Topic filter</strong> — MQTT topic to subscribe to (e.g. <code>devices/T1/messages/devicebound/#</code>). See <a href="#topic-filter">Topic filter</a> for details.</li>
      </ul>
    </li>
    <li>Click <strong>Add</strong> to save the integration.</li>
  </ol>
</Steps>

#### Credentials

Two authentication types are supported:

**Shared Access Signature:**
- **SAS Key** — the device SAS key from Azure IoT Hub. See <a href="https://docs.microsoft.com/en-us/azure/iot-edge/how-to-authenticate-downstream-device#symmetric-key-authentication" target="_blank" rel="noopener noreferrer">symmetric key authentication</a>.
- **CA certificate file** — Azure IoT Hub uses the Baltimore CyberTrust Root certificate for TLS. Leave this field empty to use the built-in default; upload a custom bundle only if required by your environment.

**PEM (X.509):**
- **CA certificate file** — Baltimore CyberTrust Root certificate is used by default.
- **Certificate file** — your device certificate.
- **Private key file** — the private key for the certificate.
- **Private key password** — optional password for the private key.

For X.509 CA-signed authentication, see the <a href="https://docs.microsoft.com/en-us/azure/iot-edge/how-to-authenticate-downstream-device#x509-ca-signed-authentication" target="_blank" rel="noopener noreferrer">Microsoft documentation</a> and the <a href="https://github.com/Azure/azure-iot-sdk-c/tree/master/tools/CACertificates" target="_blank" rel="noopener noreferrer">CACertificates instructions</a>.

#### Connection settings

<h5>Hostname</h5>

Your Azure IoT Hub hostname in the format `{hub-name}.azure-devices.net`. Find it in the Azure portal under **IoT Hub &#8702; Overview**.

<h5>Device ID</h5>

The device ID registered in Azure IoT Hub (e.g. `T1`). ThingsBoard authenticates to Azure IoT Hub as this device over MQTT on port 8883.

<h5>Credentials</h5>

Authentication type for the MQTT connection:

- `Shared Access Signature` — authenticate using a device SAS key (symmetric key)
- `PEM` — authenticate using X.509 certificates

<h5 id="topic-filter">Topic filter</h5>

The MQTT topic ThingsBoard subscribes to for incoming messages. The default format is `devices/{deviceId}/messages/devicebound/#`, which receives all cloud-to-device messages addressed to the registered device. The `#` wildcard matches any suffix, including lock tokens appended by Azure.

| Topic | Description |
|-------|-------------|
| `devices/T1/messages/devicebound/#` | All cloud-to-device messages for device T1 |

See <a href="https://docs.microsoft.com/en-us/azure/iot-hub/iot-hub-mqtt-support#receiving-cloud-to-device-messages" target="_blank" rel="noopener noreferrer">IoT Hub MQTT support</a> for details.

<h5>Execute remotely</h5>

When enabled, ThingsBoard generates an **Integration key** and **Integration secret** that allow the integration to run as a separate process outside the ThingsBoard cluster — useful when the IoT Hub endpoint is only reachable from a restricted network.

<h5>Advanced settings</h5>

| Parameter | Default | Description |
|-----------|---------|-------------|
| **Protocol version** | `MQTT 3.1.1` | MQTT protocol version. Azure IoT Hub requires MQTT 3.1.1. |
| **Max bytes in message** | `32368` | Maximum message payload size in bytes. Messages exceeding this limit are dropped. |
| **Connection timeout (sec)** | `10` | Seconds ThingsBoard waits for a broker response before marking the connection as failed. |
| **Description** | — | Optional text description for the integration. |
| **Metadata** | — | Key-value pairs injected into every message as `integrationMetadata` in the converter script. |

## Test uplink

Use the Azure portal to send a message to device `T1` and confirm that ThingsBoard receives and decodes it correctly.

<Steps>
1. In the Azure portal, open your IoT hub and navigate to **IoT devices &#8702; T1**.
2. Click **Message to Device** and enter the test payload in the **Message Body** field:
    ```json
    {
      "devName": "T1",
      "msg": {
        "temp": 42,
        "humidity": 77
      }
    }
    ```
3. Click **Send Message**.
</Steps>
