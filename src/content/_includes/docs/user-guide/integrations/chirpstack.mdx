The <a href="https://www.chirpstack.io/" target="_blank">ChirpStack</a> open-source LoRaWAN Network Server stack provides components for building LoRaWAN networks.
After integrating ChirpStack with ThingsBoard, you can connect, process, and visualize data from LoRaWAN devices in the ThingsBoard platform.

## Prerequisites

You need a running instance of the ChirpStack Network Server stack. This guide uses a local instance installed via Docker Compose.
See the <a href="https://www.chirpstack.io/project/guides/docker-compose/" target="_blank">ChirpStack Docker Compose guide</a> for installation instructions.

You must also have a device connected to the network. See the <a href="https://www.chirpstack.io/project/guides/connect-device/" target="_blank">ChirpStack device connection guide</a>.

## Create ChirpStack integration in ThingsBoard

You need access to ThingsBoard Professional Edition — either via <a href="https://thingsboard.cloud/signup" target="_blank">ThingsBoard Cloud</a> or a <DocLink product={props.product} path="TODO">self-hosted installation</DocLink>.

### Step 1. Basic settings

<Steps>
1. Sign in to your ThingsBoard account.
2. Go to **Integration center** → **Integrations**, click **+** to add a new integration.
3. Select **ChirpStack** as the integration type.
4. Optionally enable <DocLink product={props.product} path="user-guide/integrations">Debug mode</DocLink> for troubleshooting.
5. Click **Next**.
</Steps>

### Step 2. Uplink data converter

The uplink converter transforms incoming ChirpStack messages into ThingsBoard format.

Starting from **ThingsBoard 4.0**, the ChirpStack integration supports simplified typed converters — you can map message fields to attributes or telemetry without writing a custom decoder script.

<Aside type="note">Converters created before ThingsBoard 4.0 continue to work without changes.</Aside>

<Steps>
1. Enter a unique name for the converter.
2. Optionally enable <DocLink product={props.product} path="user-guide/integrations">Debug mode</DocLink>.
3. In **Main decoding configuration**, select the entity type (**Device** or **Asset**), specify the entity name (use `$eui` to populate dynamically), and write or use the default decoder script.
4. In **Advanced decoding parameters**, optionally set Device profile, Device label, Customer name, Device group name (all support `$` patterns). Map keys to **Attributes** and **Telemetry** sections. Configure **Update only keys list** to skip unchanged values.
5. Click **Next**.
</Steps>

<Aside type="tip">
We recommend using <DocLink product={props.product} path="user-guide/tbel">TBEL</DocLink> for converter scripts — it executes much more efficiently than JavaScript inside ThingsBoard.
</Aside>

Below are example uplink converter scripts. The ThingsBoard 4.0+ typed converter does not require these scripts if you use the field mapping UI, but you can still use custom scripts for advanced parsing:

<Tabs>
<TabItem label="TBEL (Recommended)">
```js maxLines=20 collapsible
var data = decodeToJson(payload);
var deviceName = data.deviceInfo.deviceName;
var deviceType = data.deviceInfo.deviceProfileName;
var groupName = 'IAQ devices';

function decodePayload(input) {
    var output = { attributes: {}, telemetry: {} };
    output.telemetry.HEX_bytes = bytesToHex(input);
    if (input.length > 0) {
        for (var i = 0; i < input.length; ) {
            var channel_id = input[i++];
            if (i < input.length) {
                var channel_type = input[i++];
                if (channel_id === 0x01 && channel_type === 0x75) {
                    output.telemetry.battery = input[i]; i += 1;
                } else if (channel_id === 0x03 && channel_type === 0x00) {
                    output.telemetry.pir = input[i] === 0 ? "normal" : "trigger"; i += 1;
                } else if (channel_id === 0x04 && channel_type === 0x00) {
                    output.telemetry.daylight = input[i] === 0 ? "dark" : "light"; i += 1;
                }
            }
        }
    }
    return output;
}

var telemetry = {};
var attributes = {};

// Timestamp parsing
var dateString = data.time;
var timestamp = -1;
if (dateString != null) {
    timestamp = new Date(dateString).getTime();
}
if (timestamp == -1) {
    timestamp = Date.now();
}

attributes.deduplicationId = data.deduplicationId;

var excludeFromAttributesList = ["deviceName", "rxInfo", "confirmed", "data",
    "deduplicationId", "time", "adr", "dr", "fCnt"];
var excludeFromTelemetryList = ["data", "deviceInfo", "txInfo", "devAddr",
    "adr", "time", "fPort", "region_common_name", "region_config_id", "deduplicationId"];

var telemetryData = toFlatMap(data, excludeFromTelemetryList, false);
var attributesData = toFlatMap(data, excludeFromAttributesList, false);

var customDecoding = decodePayload(base64ToBytes(data.data));

if (customDecoding.?telemetry.size() > 0) { telemetry.putAll(customDecoding.telemetry); }
if (customDecoding.?attributes.size() > 0) { attributes.putAll(customDecoding.attributes); }

telemetry.putAll(telemetryData);
attributes.putAll(attributesData);

var result = {
    deviceName: deviceName,
    deviceType: deviceType,
    groupName: groupName,
    attributes: attributes,
    telemetry: { ts: timestamp, values: telemetry }
};

return result;
```
</TabItem>

```js maxLines=20 collapsible
var payloadStr = decodeToString(payload);
var deviceName = 'Device A';
var deviceType = 'thermostat';

        humidity: 80,

    }

function decodeToString(payload) {

}

return result;
```

To forward data from ChirpStack to ThingsBoard, add an HTTP integration in your ChirpStack application:

<Steps>

2. Name the application and click **Submit**.
3. Open the application and navigate to the **Integrations** tab.

5. Paste the **HTTP endpoint URL** copied from the ThingsBoard integration. Click **Submit**.

## Advanced usage: downlink

To send downlink messages from ThingsBoard to a device through ChirpStack, you need a downlink converter and a rule chain configuration.

### Add downlink converter

Example downlink converter that sends an attribute update message:

<Tabs>
<TabItem label="TBEL (Recommended)">
```js maxLines=15 collapsible
// Encode downlink data from incoming Rule Engine message
var result = {
    contentType: "TEXT",
    data: btoa(msg.downlink),
    metadata: {
        DevEUI: metadata.cs_devEui,
        fPort: metadata.cs_fPort
    }
};

return result;
```

### Modify Root Rule Chain

Import and configure a rule chain to process shared attribute updates as downlinks:

<Steps>
1. Download the <a href="/resources/downlink_to_chirpstack.json" download="downlink_to_chirpstack.json">downlink_to_chirpstack.json</a> file.
2. Go to **Rule Chains**, click **+** → **Import rule chain**, and import the downloaded file.
3. In the imported "Downlink to ChirpStack" rule chain, double-click the **integration downlink** node and select your ChirpStack integration. Save changes.
4. Save the rule chain.
</Steps>
