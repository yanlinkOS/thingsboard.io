HTTP Integration converts existing protocols and payload formats to ThingsBoard message format. Common deployment scenarios:

- Stream device or asset data from an external system, IoT platform, or connectivity provider back-end.
- Stream data from a custom cloud application.
- Connect an existing device with a custom HTTP-based protocol to ThingsBoard.

<DocImage src="/src/assets/images/user-guide/integrations/http-integration.svg" alt="HTTP integration architecture" />

## Create uplink converter

Before creating the integration, create an uplink converter in **Integration center** → **Data converters**. The uplink converter parses incoming HTTP payloads into ThingsBoard format.

<Steps>
1. Click **+** → **Create new converter**. Ensure the type is **Uplink**.
2. Enter a name. Optionally enable <DocLink product={props.product} path="user-guide/integrations">Debug mode</DocLink>.
3. Paste the decoder script and click **Add**.
</Steps>

<Aside type="tip">
We recommend using <DocLink product={props.product} path="user-guide/tbel">TBEL</DocLink> for converter scripts — it executes much more efficiently than JavaScript inside ThingsBoard.
</Aside>

<Tabs>
<TabItem label="TBEL (Recommended)">
```js maxLines=20 collapsible
// Decode an uplink message from a buffer
// payload - array of bytes
// metadata - key/value object

/** Decoder **/

// decode payload to JSON
var data = decodeToJson(payload);

var deviceName = data.deviceName;
var deviceType = data.deviceType;

// Result object with device attributes/telemetry data
var result = {
   deviceName: deviceName,
   deviceType: deviceType,
   attributes: {
       model: data.model,
       serialNumber: data.param2,
   },
   telemetry: {
       temperature: data.temperature
   }
};

/** Helper functions 'decodeToString' and 'decodeToJson' are already built-in **/

return result;
```
</TabItem>
<TabItem label="JavaScript">
```js maxLines=20 collapsible
// Decode an uplink message from a buffer
// payload - array of bytes
// metadata - key/value object

/** Decoder **/

// decode payload to JSON
var data = decodeToJson(payload);

var deviceName = data.deviceName;
var deviceType = data.deviceType;

// Result object with device attributes/telemetry data
var result = {
   deviceName: deviceName,
   deviceType: deviceType,
   attributes: {
       model: data.model,
       serialNumber: data.param2,
   },
   telemetry: {
       temperature: data.temperature
   }
};

/** Helper functions **/

function decodeToString(payload) {
   return String.fromCharCode.apply(String, payload);
}
function decodeToJson(payload) {
   var str = decodeToString(payload);
   var data = JSON.parse(str);
   return data;
}

return result;
```
</TabItem>
</Tabs>

## Create integration

<Steps>
1. Go to **Integration center** → **Integrations**, click **+** to create a new integration. Name it "HTTP Integration", select type **HTTP**. Click **Next**.
</Steps>

<DocImage src="/src/assets/images/user-guide/integrations/http/http-add-integration-1-pe.png" alt="Create HTTP integration" />

</Steps>

<DocImage src="/src/assets/images/user-guide/integrations/http/http-add-integration-3-pe.png" alt="Skip downlink converter" />

## Send uplink message

Use the **HTTP endpoint URL** copied from the integration to send a test message. Replace `$DEVICE_NAME`, `$DEVICE_TYPE`, and `$YOUR_HTTP_ENDPOINT_URL` with your values:

```bash
curl -v -X POST \
  -d '{"deviceName":"$DEVICE_NAME","deviceType":"$DEVICE_TYPE","temperature":33,"model":"test"}' \
  $YOUR_HTTP_ENDPOINT_URL \
  -H "Content-Type:application/json"
```

### Enable security option

You can require additional headers for incoming messages. To do this:

<Steps>
1. Open the integration settings and check the **Enable security** checkbox.
2. Click on **Headers filter** and specify an arbitrary header name and value. Save changes.
</Steps>

<DocImage src="/src/assets/images/user-guide/integrations/http/http-enable-security-1-pe.png" alt="Enable security — configure header filter" />

Once configured, include the header in your uplink message. Replace `$DEVICE_NAME`, `$DEVICE_TYPE`, `$YOUR_HTTP_ENDPOINT_URL`, and `$VALUE` with your values:

```bash
curl -v -X POST \
  -d '{"deviceName":"$DEVICE_NAME","deviceType":"$DEVICE_TYPE","temperature":33,"model":"test"}' \
  $YOUR_HTTP_ENDPOINT_URL \
  -H "Content-Type:application/json" \
  -H "$VALUE"
```

<DocImage src="/src/assets/images/user-guide/integrations/http/http-enable-security-2-pe.png" alt="Send message with security header" />

## Downlink converter

Create a downlink converter in **Integration center** → **Data converters**. Enable **Debug mode** to view events.

In this example, we send an attribute update message back to the device. Change the downlink encoder function to use `JSON.stringify(msg)` as the data field — this forwards the received message as-is:

<Tabs>
<TabItem label="TBEL (Recommended)">
```js maxLines=15 collapsible
// Encode downlink data from incoming Rule Engine message

// msg - JSON message payload downlink message json
// msgType - type of message, for ex. 'ATTRIBUTES_UPDATED', 'POST_TELEMETRY_REQUEST', etc.
// metadata - list of key-value pairs with additional data about the message
// integrationMetadata - list of key-value pairs with additional data defined in Integration executing this converter

var result = {

    // downlink data content type: JSON, TEXT or BINARY (base64 format)
    contentType: "JSON",

    // downlink data
    data: JSON.stringify(msg),

    // Optional metadata object presented in key/value format
    metadata: {
    }
};

return result;
```
</TabItem>
<TabItem label="JavaScript">
```js maxLines=15 collapsible
// Encode downlink data from incoming Rule Engine message

// msg - JSON message payload downlink message json
// msgType - type of message, for ex. 'ATTRIBUTES_UPDATED', 'POST_TELEMETRY_REQUEST', etc.
```

</TabItem>

<Steps>
1. Go to **Rule chains** and open the **Root Rule Chain**.
2. Create an **integration downlink** node and specify your HTTP integration in its settings.
</Steps>

<DocImage src="/src/assets/images/user-guide/integrations/http/http-downlink-rule-chain-1-pe.png" alt="Add integration downlink node" />

<Steps>
3. Connect the **message type switch** node to the **integration downlink** node with **Attributes Updated** and **Post attributes** links. Apply changes.
</Steps>

### Test downlink

When a shared attribute is created or updated, a downlink message is sent to the integration.

<Steps>
1. Go to **Devices**, select your device, and navigate to the **Attributes** tab.
2. Select **Shared attributes**, click **+** to add a new attribute.
3. Set the key (e.g. `firmware`) and value (e.g. `01052020.v1.1`), then save.
</Steps>

<DocImage src="/src/assets/images/user-guide/integrations/http/http-downlink-add-attribute-1-pe.png" alt="Add shared attribute" />
