Particle is an IoT cloud platform for connected devices. After integrating Particle with ThingsBoard, device events flow from the Particle Cloud to ThingsBoard via a webhook — and ThingsBoard can send RPC commands back to devices through the Particle Cloud function API.

## Prerequisites

- A <a href="https://console.particle.io/" target="_blank" rel="noopener noreferrer">Particle Console</a> account with at least one registered device.
- The <a href="https://docs.particle.io/getting-started/developer-tools/cli/" target="_blank" rel="noopener noreferrer">Particle CLI</a> installed if you want to send downlink commands.

## Connect the Particle Photon

Follow the <a href="https://docs.particle.io/quickstart/photon/" target="_blank" rel="noopener noreferrer">official Particle setup guide</a> to connect your Photon to Wi-Fi and register it with the Particle Cloud. The device must be online before events can be forwarded to ThingsBoard.

<Steps>
1. **Power on your device.** Plug the USB cable into a power source. The RGB LED blinks blue when the device is ready for setup. If it is not blinking blue, hold the **SETUP** button until it does.
2. **Connect to Wi-Fi.** Go to <a href="https://setup.particle.io" target="_blank" rel="noopener noreferrer">setup.particle.io</a>, click **Setup a Photon**, download `photonsetup.html`, and open it in your browser to complete Wi-Fi configuration.
</Steps>

Once connected, the device appears in the <a href="https://console.particle.io/devices" target="_blank" rel="noopener noreferrer">Particle Console</a> under **Devices** with **Status: Online**. Note the **device ID** (the `coreid` column) — the uplink converter uses it as the ThingsBoard device name.

## ThingsBoard integration setup

### Create an uplink converter

The uplink converter parses incoming Particle event payloads and maps them to ThingsBoard telemetry and attributes. Particle uses a <DocLink product={props.product} path="user-guide/integrations/uplink-data-converter">generic uplink converter</DocLink>. Each Particle event contains a `coreid` field (the device ID, used as the ThingsBoard device name), a `data` field (the event payload), and metadata such as `published_at`. The decoder stores `online`/`offline` events as a `status` attribute and all other event data as raw `rawData` telemetry.

<Aside type="tip" title="TBEL">
  We recommend using <DocLink product={props.product} path="user-guide/tbel">TBEL</DocLink> for converter scripts — it executes much more efficiently than JavaScript inside ThingsBoard.
</Aside>

<Steps>
  1. Go to **Integrations center &#8702; Data converters**
  2. Click **+ Add data converter &#8702; Create new converter**.
  3. Ensure the type is **Uplink**.
  4. Enter a converter name (e.g. `Particle Uplink Converter`).
  5. Paste the decoder script and click **Add**.
</Steps>

**The decoder function used in this tutorial:**

<Tabs>
<TabItem label="TBEL (Recommended)">
```js maxLines=15 collapsible
/** Decoder **/

// decode payload to JSON
var data = decodeToJson(payload);
if (data.coreid != "api") {
    var deviceName = data.coreid;
    var deviceType = 'Photon';
    var groupName = 'Particle devices';

    var attributes = {
        integrationName: metadata['integrationName']
    };

    var telemetry = {};

    if (data.data == 'online' || data.data == 'offline') {
        attributes.status = data.data;
    } else {
        telemetry.rawData = data.data;
    }

    var result = {
        deviceName: deviceName,
        deviceType: deviceType,
        groupName: groupName,
        attributes: attributes,
        telemetry: telemetry
    };

    /** Helper functions 'decodeToString' and 'decodeToJson' are already built-in **/

    return result;
}
```
</TabItem>
<TabItem label="JavaScript">
```js maxLines=15 collapsible
/** Decoder **/

// decode payload to JSON
var data = decodeToJson(payload);
if (data.coreid != "api") {
    var deviceName = data.coreid;
    var deviceType = 'Photon';
    var groupName = 'Particle devices';

    var attributes = {
        integrationName: metadata['integrationName']
    };

    var telemetry = {};

    if (data.data == 'online' || data.data == 'offline') {
        attributes.status = data.data;
    } else {
        telemetry.rawData = data.data;
    }

    var result = {
        deviceName: deviceName,
        deviceType: deviceType,
        groupName: groupName,
        attributes: attributes,
        telemetry: telemetry
    };

    return result;
}

/** Helper functions **/

function decodeToString(payload) {
    return String.fromCharCode.apply(String, payload);
}

function decodeToJson(payload) {
    var str = decodeToString(payload);
    var data = JSON.parse(str);
    return data;
}
```
</TabItem>
</Tabs>

**To adapt this converter to your device:**

- **Structured payload** — if your device publishes JSON (e.g. `{"temp": 23.5, "humidity": 60}`), parse it with `decodeToJson(data.data)` and map individual fields to telemetry keys instead of storing `rawData`.
- **Different device name** — `coreid` is the Particle device ID (a hex string). To use a human-readable name, include a name field in the event data and extract it from `data.data` after parsing.
- **Additional attributes** — add more keys to the `attributes` object (e.g. firmware version, hardware revision).
- **Event filtering** — the `data.coreid != "api"` check discards Particle Cloud API events that have no associated physical device.

### Create a downlink converter

The downlink converter encodes outgoing ThingsBoard RPC messages into commands sent to your Particle device. For the full encoder reference, see <DocLink product={props.product} path="user-guide/integrations/downlink-data-converter">Downlink data converter</DocLink>. via the <a href="https://docs.particle.io/reference/cloud-apis/api/#call-a-function" target="_blank" rel="noopener noreferrer">Particle Cloud function API</a>. It reads the RPC `method` and `params` from the incoming ThingsBoard message and forwards them to the registered Particle function on the device.

<Steps>
  1. Go to **Integrations center &#8702; Data converters**, click **+ Add data converter &#8702; Create new converter**.
  2. Switch the type to **Downlink**.
  3. Enter a converter name (e.g. `Particle Downlink Converter`).
  4. Paste the encoder script and click **Add**.
</Steps>

**The encoder used in this tutorial:**

<Tabs>
<TabItem label="TBEL (Recommended)">
```js maxLines=15 collapsible
/** Encoder **/

var command = {};
command["method"] = msg.method;
if (msg.params == "false" || msg.params == "true") {
    command["params"] = Boolean.valueOf(msg.params);
} else {
    command["params"] = msg.params;
}

var result = {
    contentType: "JSON",
    data: JSON.stringify(command),
    metadata: {
        deviceId: metadata.deviceName
    }
};

return result;
```
</TabItem>
<TabItem label="JavaScript">
```js maxLines=15 collapsible

```
</TabItem>
</Tabs>

The `metadata.deviceId` is set to the ThingsBoard device name (which equals the Particle `coreid`) and is used by the integration to identify the target device when calling the Particle Cloud API.

### Generate a Particle access token

To allow ThingsBoard to call Particle Cloud device functions for downlink, generate an access token using the <a href="https://docs.particle.io/getting-started/developer-tools/cli/" target="_blank">Particle CLI</a>.

Install the CLI:

```bash
bash <( curl -sL https://particle.io/install-cli )
```

Then, according to <a href="https://docs.particle.io/reference/developer-tools/cli/#particle-login" target="_blank">official documentation</a>, you will need to log in to your Particle account:

```bash
particle login
```

  { src: '/src/assets/images/user-guide/integrations/particle/cli-login.png', alt: 'Terminal — particle login command: prompts for email and password, then prints Successfully completed login', caption: 'Particle CLI — particle login prompts for credentials and confirms successful authentication' },
]} />

<a href="https://docs.particle.io/reference/developer-tools/cli/#particle-token-create" target="_blank">Create an access token</a>. Use the never-expires option for a long-lived integration token:

```bash
# Never-expiring token (recommended for integrations)
particle token create --never-expires

# Token with default 90-day expiration
particle token create
```

<ImageGallery images={[

Save the token — you will need it in the next step.

### Create the integration

<Steps>
1. Go to **Integrations center &#8702; Integrations** and click **+ Add integration**.
2. **Basic settings:**
    - Set **Integration type** to **Particle**.
    - **Enable integration** and **Allow create devices or assets** are on by default.
    - Click **Next**.
3. **Uplink data converter:**
    - Click **Select existing**, choose the <a href="#create-an-uplink-converter">previously created</a> `Particle Uplink Converter`.
    - Click **Next**.
4. **Downlink data converter:**
    - Click **Select existing**, choose the <a href="#create-a-downlink-converter">previously created</a> `Particle Downlink Converter`.
    - Click **Next**.
5. **Connection settings:**
    - Note the **HTTP endpoint URL** displayed — you will paste it into the Particle Console webhook in the next section.
    - Enable **Allow downlink** and paste the <a href="#generate-a-particle-access-token">Particle access token</a> you generated into the **Token** field.
6. Click **Add** to save the integration.
</Steps>

## Configure webhook on Particle Console

To push device events from Particle Cloud to ThingsBoard, create a webhook in the Particle Console that forwards matching events to the ThingsBoard integration endpoint.

<Steps>
1. In the <a href="https://console.particle.io/" target="_blank" rel="noopener noreferrer">Particle Console</a>, open your product, go to **Integrations**, and click **+ Add New Integration**.
2. Select **Webhook** from the integration type list.
3. Fill in the webhook form:
   - **Name** — e.g. `ThingsBoard`
   - **Event Name** — the Particle event name your device publishes (e.g. `spark`). Only events matching this name trigger the webhook.
   - **URL** — paste the **HTTP endpoint URL** copied from the ThingsBoard integration Connection step.
   - **Request Type** — `POST`
   - **Request Format** — `JSON`
   - Leave **Status** as `Enabled`.
4. Click **Create Webhook**.
</Steps>

## Test uplink

To trigger a test event, publish a Particle event from the CLI using the same event name you configured in the webhook (e.g. `spark`):

```bash
particle publish spark "test data" --private
```

Alternatively, open the webhook in the <a href="https://console.particle.io/" target="_blank" rel="noopener noreferrer">Particle Console</a>, go to **Integrations**, select the ThingsBoard webhook, and click **Test** to send a sample payload.

Once the webhook forwards an event to ThingsBoard, go to **Entities &#8702; Devices** — the device is automatically provisioned using the Particle `coreid` as the device name, assigned the `Photon` profile, and placed in the `Particle devices` group.

## Test RPC control with a dashboard

### Configure downlink via rule chain

To route RPC commands from ThingsBoard dashboards to your Particle device, connect the **Integration Downlink** node to the Root Rule Chain.

<Steps>
1. Go to **Rule chains** and open the **Root Rule Chain**.
2. Find the **Integration Downlink** node in the node panel, drag it onto the canvas, enter a name (e.g. `To Particle integration`), select the Particle integration, and click **Add**.
3. Connect the **Message Type Switch** node to the **Integration Downlink** node using the **RPC Request to Device** relation type.
4. Click **Apply changes** to save the rule chain.
</Steps>

<Steps>
1. Open the <a href="https://build.particle.io/" target="_blank" rel="noopener noreferrer">Particle Web IDE</a> and click **Web-Connected LED** in the example apps list.

### Configure the dashboard switch widget

<Steps>
  <ol>
    <li>Open an existing ThingsBoard dashboard or create a new one and enter <strong>Edit</strong> mode.</li>
    <li>Click <strong>Add widget &#8702; Control widgets &#8702; Switch Control</strong> to add the widget.</li>
    <li>On the <strong>Data</strong> tab, set <strong>Target device</strong> to your Particle device (create a new alias pointing to the provisioned device if needed).</li>
    <li>Open the <strong>Appearance</strong> tab.</li>
    <li>Set <strong>Retrieve value using method</strong> to <strong>Don't retrieve</strong>.</li>
    <li>Set <strong>RPC set value method</strong> to <code>led</code>.</li>
    <li>
      <span>Paste the following into the <strong>Convert value function</strong> field:</span>
      ```js
      return value ? "on" : "off";
      ```
    </li>
    <li>Click <strong>Add</strong> to save the widget, then click <strong>Save</strong> to save the dashboard.</li>
  </ol>
</Steps>
