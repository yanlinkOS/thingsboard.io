The AWS IoT integration streams data from <a href="https://aws.amazon.com/iot/" target="_blank">AWS IoT</a> to ThingsBoard. AWS IoT acts as an MQTT broker — it receives messages from devices, filters them, and forwards them to subscribers. ThingsBoard subscribes to the relevant topics and processes the incoming data through uplink converters.

<DocImage src="/src/assets/images/user-guide/integrations/aws-iot-integration.svg" alt="AWS IoT integration architecture" />

## AWS IoT setup

You need an <a href="https://aws.amazon.com/iot/" target="_blank">AWS account</a> with IoT Core enabled. The setup involves creating a policy, a thing (device), and certificates for secure MQTT communication.

### Create a policy

A policy defines permissions for IoT operations (publish, subscribe, connect).

<Steps>
1. In the AWS IoT console, go to **Security** → **Policies** and click **Create Policy**.
2. Enter a **Policy Name** and switch to **JSON** editor.
3. Paste the policy document below, replacing `YOUR_REGION` and `YOUR_AWS_ID` with your values.
4. Click **Create**.
</Steps>

<Aside type="caution">
Replace <code>YOUR_REGION</code> and <code>YOUR_AWS_ID</code> with your actual AWS region and account ID. For example: <code>eu-west-1:111197721064</code>. Your region is in the URL when signed in: <code>https://<b>eu-west-1</b>.console.aws.amazon.com</code>.
</Aside>

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": ["iot:Publish", "iot:Receive"],
      "Resource": ["arn:aws:iot:YOUR_REGION:YOUR_AWS_ID:topic/*"]
    },
    {
      "Effect": "Allow",
      "Action": ["iot:Subscribe"],
      "Resource": ["arn:aws:iot:YOUR_REGION:YOUR_AWS_ID:topicfilter/*"]
    },
    {
      "Effect": "Allow",
      "Action": ["iot:Connect"],
      "Resource": ["arn:aws:iot:YOUR_REGION:YOUR_AWS_ID:client/*"]
    }
  ]
}
```

### Create a thing and certificates

A "thing" is a digital representation of a physical device in AWS IoT.

<Steps>
1. Go to **All devices** → **Things** and click **Create thing**.
2. Select **Create single thing** and click **Next**.
3. Enter a **Name** for the device and click **Next**.
4. Select **Auto-generate a new certificate** and click **Next**.
5. Attach the policy you created earlier, then click **Create thing**.
6. Download the required certificate files:
   - **Device certificate** (`*.pem.crt`)
   - **Private key** (`*-private.pem.key`)
   - **Root CA certificate** (`*.pem`)
   - Also save the **Public key** file for your records.
7. Click **Done**.
</Steps>

## ThingsBoard setup

### Create uplink converter

Before creating the integration, create an uplink data converter to parse incoming AWS IoT messages.

<Steps>
1. Go to **Integration center** → **Data converters**, click **+** → **Create new converter**.
2. Ensure the type is **Uplink**. Enter a name.
3. Optionally enable <DocLink product={props.product} path="user-guide/integrations">Debug mode</DocLink>.
4. Paste the decoder script and click **Add**.
</Steps>

<Aside type="tip">
We recommend using <DocLink product={props.product} path="user-guide/tbel">TBEL</DocLink> for converter scripts — it executes much more efficiently than JavaScript inside ThingsBoard.
</Aside>

<Tabs>
<TabItem label="TBEL (Recommended)">
```js
var data = decodeToJson(payload);
var topicParts = metadata.topic.split("/");
var deviceType = topicParts[0];
var deviceName = topicParts[1];

var result = {
    deviceName: deviceName,
    deviceType: deviceType,
    attributes: {
        state: data.val0,
    },
    telemetry: {
        temperature: data.val1,
        fan_ins: data.val2,
        fan_out: data.val3,
    }
};

return result;
```
</TabItem>
<TabItem label="JavaScript">
```js
var payloadStr = decodeToString(payload);
var data = JSON.parse(payloadStr);
var topicParts = metadata.topic.split("/");
var deviceType = topicParts[0];
var deviceName = topicParts[1];

var result = {
    deviceName: deviceName,
    deviceType: deviceType,
    attributes: {
        state: data.val0,
    },
    telemetry: {
        temperature: data.val1,
        fan_ins: data.val2,
        fan_out: data.val3,
    }
};

function decodeToString(payload) {
    return String.fromCharCode.apply(String, payload);
}

return result;
```
</TabItem>
</Tabs>

### Send uplink message

To send a test message, use the AWS IoT **MQTT test client**:

<Steps>
1. In the AWS IoT console, go to **MQTT test client** → **Publish to a topic**.
2. Enter the topic (e.g. `tb/aws/iot/sensors/freezer-432`).
3. Enter the test payload and click **Publish**.
</Steps>

Example payload:

```json
{
    "val0": "loaded",
    "val1": -18,
    "val2": 1785,
    "val3": 548
}
```

## Advanced usage: downlink

To send messages from ThingsBoard to devices via AWS IoT, create a downlink converter and configure a rule chain.

### Create downlink converter

<Tabs>
<TabItem label="TBEL (Recommended)">
```js
var data = {};

data.v0 = msg.state;
data.m0 = "att_upd_success";
data.devSerialNumber = metadata['ss_serialNumber'];

var result = {
    contentType: "JSON",
    data: JSON.stringify(data),
    metadata: {
        type: "sensors/device/upload"
    }
};

return result;
```
</TabItem>
