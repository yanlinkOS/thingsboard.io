ThingsBoard can store binary content (files) in the database. This is currently used to store report files generated by the <DocLink product={props.product} path="reference/rule-engine/nodes/action/generate-dashboard-report" target="_blank">Generate Dashboard Report</DocLink> rule node. Stored files can be attached to emails via the <DocLink product={props.product} path="reference/rule-engine/nodes/external/send-email" target="_blank">Send Email</DocLink> node, or sent to external systems using the <DocLink product={props.product} path="reference/rule-engine/nodes/external/rest-api-call" target="_blank">REST API Call node</DocLink>.

Users can access stored files through the **Files** or **Reports** widgets, which are part of the **Files** Widgets Bundle.

## Files Widget

ThingsBoard provides access to stored files via the **Files** widget, included in the **Files** Widgets Bundle.

<DocImage src="/src/assets/images/user-guide/file-storage/file-storage-files-widget.png" alt="Files widget showing stored report files with time range filter and download button" />

The widget supports filtering files by time range and searching by file name. Each file can be downloaded by clicking the **Download file** button.
