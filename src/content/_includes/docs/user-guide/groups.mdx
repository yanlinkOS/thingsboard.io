**Entity Groups** let you organize <DocLink product={props.product} path='user-guide/devices' target="_blank">devices</DocLink>, <DocLink product={props.product} path='user-guide/assets' target="_blank">assets</DocLink>, <DocLink product={props.product} path='user-guide/entity-views' target="_blank">entity views</DocLink>, <DocLink product={props.product} path='user-guide/customers' target="_blank">customers</DocLink>, <DocLink product={props.product} path='user-guide/users' target="_blank">users</DocLink>, <DocLink product={props.product} path='user-guide/dashboards' target="_blank">dashboards</DocLink>, and <DocLink product={props.product} path='user-guide/edge-computing' target="_blank">edge instances</DocLink> into custom collections. Each entity can belong to multiple groups simultaneously. A special **"All"** group always contains every entity of a given type within the tenant.

Groups serve three main purposes:

- **Organization** — categorize entities by location, function, project, or any custom criteria
- **Access control** — share groups with customers and configure permissions per group
- **Bulk operations** — change ownership, move, add, or remove entities in batches

## Create Entity Group

The process is the same for all entity types. The steps below use devices as an example:

<Steps>
1. Navigate to the entity page (e.g., **Entities &#8702; Devices**) and switch to the **Groups** tab.
2. Click **+ Add entity group**.
3. Enter the group **name**.
4. Optionally configure sharing with a customer (see **[share entity group](#share-entity-group)** below).
5. Click **Add**.
</Steps>

The new group appears in the Groups tab. Open it to view and manage its entities.

## Group Configuration

Open a group and click the **pencil** icon to open the group settings. The configuration dialog has several tabs.

### General

Edit the group name and, for device groups, assign firmware or software versions.

### Columns

Customize which columns appear in the group's entity table. You can:

- **Add a column** — click **+**, select the data source (entity field, attribute, or telemetry key), and set a title
- **Delete a column** — click the **trash** icon next to the column
- **Reorder columns** — drag columns to change their position
- **Style function** — apply conditional formatting (e.g., color temperature values based on thresholds):

```javascript
return value >= 45 ? {
    color: 'rgb(255, 106, 12)',
    fontWeight: 600
} : {
    color: 'rgb(0, 132, 214)',
    fontWeight: 600
}
```

- **Content function** — transform the displayed value (e.g., append a unit symbol):

```javascript
return value ? value + ' ℃' : '-';
```

### Settings

Control the group's display behavior:

| Setting | Description |
|---------|-------------|
| **Enable search** | Show or hide the search bar |
| **Enable add entity** | Allow adding new entities from the group view |
| **Enable delete entity** | Allow deleting entities from the group view |
| **Manage credentials** | Show credential management options (device groups only) |
| **Pagination** | Configure the number of entities per page |
| **Detail action** | Select which action opens entity details |

### Actions

Define quick actions available for each entity in the group. Actions can navigate to a specific dashboard or execute custom logic.

<Steps>
1. Go to the **Actions** tab in group configuration.
2. Click **+** to add an action.
3. Choose the action type — **Navigate to dashboard** or **Custom action**.
4. For dashboard navigation, select the target dashboard and optionally a specific state.
5. Click **Add**.
</Steps>

### Permissions

Define who can access the entities inside this group and what actions they can perform. A permission entry links a **group <DocLink product={props.product} path='user-guide/roles'>role</DocLink>** to a **user group**.

<Steps>
1. Open the group details → **Permissions** tab.
2. Click **+** to add a permission.
3. Select a **Role** from the dropdown.
4. Select the **User group owner** (Tenant or a specific <DocLink product={props.product} path='user-guide/customers'>Customer</DocLink>) and the **User group**.
5. Click **Add**.
</Steps>

You can edit or delete existing permissions at any time. Changes take effect immediately for all affected users.

<Aside type="tip">
Permissions supplement tenant-wide RBAC and do not override global restrictions. See <DocLink product={props.product} path='user-guide/roles'>Roles</DocLink> for details on how group roles interact with generic roles.
</Aside>

## Batch Operations

Select multiple entities within a group using checkboxes to perform bulk actions:

| Operation | Description |
|-----------|-------------|
| **Change owner** | Transfer selected entities to another Customer or back to the Tenant |
| **Add to group** | Add selected entities to another group (does not remove from current group) |
| **Move to group** | Move selected entities to another group (removes from current group) |
| **Remove from group** | Remove selected entities from the current group (does not delete the entities) |
| **Delete** | Permanently delete selected entities |

## Share Entity Group

Sharing a group grants a customer's user group access to all entities inside it by creating a single permission entry.

<Steps>
1. In the **Groups** tab, click the **Share** icon next to the target group.
2. Select the **Customer**.
3. Optionally enable **All users** to provide access to every user of that customer.
4. Choose the permission level — **Read**, **Write**, or a custom **Role**.
5. Click **Share**.
</Steps>

To update or revoke access later, open the group's [Permissions](#permissions) tab and edit or delete the corresponding entry.

## Make Group Public or Private

Public groups provide read-only access to anyone without sign-in. Use this only for non-sensitive data.

**Make public:**
<Steps>
1. In the **Groups** tab, click the **Make public** icon next to the target group.
2. Confirm in the dialog.
</Steps>

<Aside type="note">
Public dashboards that display device data require both the dashboard group and the related device group to be public.
</Aside>

**Make private:**
<Steps>
1. Click the **Make private** icon next to the group.
2. Confirm in the dialog. Public access is removed immediately.
</Steps>

For controlled access to specific customers, use [share entity group](#share-entity-group) or configure entries on the [Permissions](#permissions) tab instead.

## Delete a Group

Click the **trash** icon next to a group and confirm. You can also select multiple groups using checkboxes and delete them in bulk.

<Aside type="caution">
Deleting a group also deletes all entities inside it. If you only want to remove entities from the group without deleting them, use the **Remove from group** batch operation instead.
</Aside>
