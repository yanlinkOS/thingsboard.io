## Domain details

Click on a registered domain to open the details dialog. It displays:

- The **CNAME record** for the domain.
- The **SSL certificate** details, including the validity period (**Not before** and **Not after** dates).

<Aside type="note">
  The SSL certificate is valid for 90 days. ThingsBoard automatically renews it at least 30 days before it expires — no manual action is required.
</Aside>

## Set up a CNAME record

To point your domain to ThingsBoard, add a CNAME record at your DNS provider:

| Field | Value |
|---|---|
| **Type** | `CNAME` |
| **Name / Host** | Your subdomain (e.g. `mycompany`) |
| **Target / Value** | See note below |
| **TTL** | `3600` (or your provider's default) |

{props.product === Products.PAAS && (
<Aside type="note">
  Set the CNAME target to `eu.thingsboard.cloud`.
</Aside>
)}
{(props.product === Products.CE || props.product === Products.PE) && (
<Aside type="note">
  Set the CNAME target to the hostname or IP address of your ThingsBoard server.
</Aside>
)}

## Troubleshooting

If your domain is not resolving correctly, verify your CNAME record using [Google Admin Toolbox](https://toolbox.googleapps.com/apps/dig/) or the `dig` command on Linux:

```bash
dig $YOUR_DOMAIN_NAME any
```

Example:

```bash
dig mycompany.thingsboard.space any
```

A correctly configured CNAME record produces output similar to:

```
;; ANSWER SECTION:
mycompany.thingsboard.space. 3600 IN    CNAME   eu.thingsboard.cloud
```

If the answer section is missing or shows an `HINFO` record instead of `CNAME`, the DNS record is not configured correctly. Check your DNS provider settings and wait for propagation (up to 24–48 hours for some providers).
