<ConditionalHeading level={2} id="connect-trendz-to-thingsboard" showFor="ce,pe">Connect Trendz to ThingsBoard</ConditionalHeading>

The **Trendz settings** page lets a System Administrator configure the connection between ThingsBoard and the Trendz Analytics add-on.

<ConditionalHeading level={3} id="prerequisites" showFor="ce,pe">Prerequisites</ConditionalHeading>

- Trendz must be <a href="/docs/trendz/install/installation-options/" target="_blank">installed</a> and network-accessible from ThingsBoard

<ConditionalHeading level={3} id="access-the-settings" showFor="ce,pe">Access the settings</ConditionalHeading>

Log in to ThingsBoard as a **System Administrator** and click **Trendz settings** in the left menu.

<ConditionalHeading level={3} id="configuration" showFor="ce,pe">Configuration</ConditionalHeading>

Set the two internal URLs that allow ThingsBoard and Trendz to communicate with each other:

| Setting | Description |
|---------|-------------|
| **Trendz Internal URL** | URL that ThingsBoard uses to reach Trendz |
| **ThingsBoard Internal URL** | URL that Trendz uses to reach ThingsBoard |

The URL values depend on your deployment type:

| Deployment | Trendz Internal URL | ThingsBoard Internal URL |
|---|---|---|
| **Same machine** (Ubuntu, Windows, CentOS) | `http://localhost:8888` | `http://localhost:8080` |
| **Docker** (same compose file) | `http://trendz:8888` | `http://thingsboard-pe:8080` |
| **Kubernetes** (same cluster) | `http://trendz-app:8888` | `http://tb-node:8080` |
| **Public network** (same domain) | `https://mydomain` | `https://mydomain` |

<ConditionalHeading level={3} id="actions" showFor="ce,pe">Actions</ConditionalHeading>

| Action | Description |
|--------|-------------|
| **Save Configuration** | Saves the URLs entered in the text fields |
| **Retry Discovery** | Initiates synchronization with Trendz using the saved configuration |
| **Retry Healthcheck** | Checks if the current Trendz sync contains any errors (visible only after successful sync) |
