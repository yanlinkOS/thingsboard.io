The **Notification center** is where you send, manage, and automate notifications. You can trigger notifications manually, via the REST API, or automatically in response to system events, with optional scheduling.

<ImageGallery images={[{ src: '/src/assets/schemas/notification-center-overview.svg', alt: 'Notification center overview: trigger event flows through a rule and template to resolve recipients and deliver via Web, Email, Slack, and other channels' }]} />

## Notification Delivery Methods

- **Web:** Notifications appear inside the ThingsBoard web interface.
- **Mobile app:** Push notifications via the {props.product === Products.CE ? <a href="/docs/mobile/" target="_blank">ThingsBoard Mobile Application</a> : <a href="/docs/mobile/pe/" target="_blank">ThingsBoard PE Mobile Application</a>}. Requires mobile app setup and mobile settings enabled in ThingsBoard.
- **SMS:** Delivered via SMS. Works without internet access. Requires an <DocLink product={props.product} path="user-guide/ui/sms-provider-settings" bold={false} target="_blank">SMS provider</DocLink>{props.product === Products.CE ? <> configured by a System Administrator</> : <> configured at your level or by your System Administrator</>}.
- **Email:** Delivered to the recipient's inbox. Requires an <DocLink product={props.product} path="user-guide/ui/mail-settings" bold={false} target="_blank">outgoing mail server</DocLink>{props.product === Products.CE ? <> configured by a System Administrator</> : <> configured at your level or by your System Administrator</>}.
- **Slack:** Messages to Slack users or channels. Requires <DocLink product={props.product} path="user-guide/ui/slack-settings" bold={false} target="_blank">Slack settings</DocLink>.
- **Microsoft Teams:** Messages to a Teams channel. Requires a <DocLink product={props.product} path="user-guide/ui/microsoft-teams-settings" bold={false} target="_blank">webhook URL</DocLink>.

## Navigation

The **Notification center** is accessible from the left sidebar and provides the following options:

- **Send notification:** Send a one-off message manually to selected recipients.
- **Inbox:** View received notifications.
- **Sent:** View sent notifications and their delivery status.
- **Recipients:** Manage who receives notifications.
- **Templates:** Define notification content and delivery method.
- **Rules:** Configure triggers for automated notifications.

## Automated Notifications

To send notifications automatically in response to system events, configure three components:

- **Recipients:** Who receives the notification — platform users, Slack channels, Teams channels, etc.
- **Templates:** What is sent — message content and delivery method (Web, Email, Slack, etc.).
- **Rules:** When it is sent — the trigger event and filter conditions that activate the notification.

For end-to-end examples, see the [Examples](#examples) section below.

## Recipients

The **Recipients** tab lists configured notification recipients. 

To add one:

<Steps>
  1. Click **+ Add recipients**.
  2. Enter a name for the recipients group.
  3. Select the notification **type**:
      - **Platform users** – send notifications to ThingsBoard users. Select a user filter.
      - **Slack** – send notifications to a Slack channel or direct message. Requires <DocLink product={props.product} path="user-guide/ui/slack-settings" bold={false} target="_blank">Slack settings</DocLink> configured first. Select the channel type and specify the conversation.
      - **Microsoft Teams** – send notifications to a Teams channel. Requires a <DocLink product={props.product} path="user-guide/ui/microsoft-teams-settings" bold={false} target="_blank">webhook URL</DocLink>. Enter the webhook URL and channel name.
  4. Configure the fields specific to the selected type.
  5. Click **Add** to create the recipients group.
</Steps>

You can also create recipients inline during the send wizard.

## Templates

Use the **Templates** tab to create and manage notification templates. Each template defines a set of delivery methods and message content per method.

<Steps>
  1. Click **+ Add template**.
  2. Enter a template name and select the required **type**.
  3. Choose one or more **delivery methods**.
  4. Click **Next** to configure message content.
  5. For each selected delivery method, enter the **subject** (if applicable) and **message**.
  6. (Optional) Configure additional options such as icons, action buttons, or report attachments.
  7. Click **Add** to create the template.
</Steps>

Templates support dynamic parameters wrapped in `${...}`, like `${recipientFirstName}`, with optional suffixes (`upperCase`, `lowerCase`, `capitalize`). See <DocLink product={props.product} path="reference/notification-system/template-parameters" bold={false} target="_blank">Notification template parameters</DocLink> for the full reference.

## Rules

Notification rules automate delivery based on trigger events. Each rule connects a system event with a matching template and recipients.

<Steps>
  1. Open the **Rules** tab and click **+ Add rule**.
  2. Provide a rule name and select the required **trigger type**.
  3. Choose an existing [template](#templates) or create a new one.
  4. Add one or more [recipients](#recipients) and click **Next**.
  5. Configure the trigger-specific settings and click **Add** to save the rule.
</Steps>

### Alarm Escalation Chain

The **Alarm** trigger uses an escalation chain instead of a simple recipient list. You configure multiple recipient stages with delays between them:

<ImageGallery images={[{ src: '/src/assets/schemas/alarm-escalation-chain.svg', alt: 'Alarm escalation chain: first recipient notified immediately, then second recipient after a delay, escalation stops when alarm is acknowledged or cleared' }]} />

If a recipient acknowledges or clears the alarm before the next stage's delay expires, subsequent recipients do not receive the notification. Configure which alarm statuses stop the chain in the **Stop escalation on the alarm status become** field.

## Manual Notifications

Use **Send notification** to send a one-off message immediately or at a scheduled time — suited for announcements, maintenance alerts, and direct messages to specific users.

<Steps>
  1. Go to **Notification center** and click **Send notification** in the top-right corner.
  2. Select recipients and delivery methods. Optionally schedule the notification{props.product !== Products.CE && <> or <DocLink product={props.product} path="user-guide/reporting/notifications" bold={false} target="_blank">attach a report</DocLink> (Email and Slack only)</>}. Click **Next**.
  3. Enter the subject and message for each delivery method. For Web notifications, optionally add an action button with a dashboard link or URL. Click **Next**.
  4. Review and click **Send**.
</Steps>

## Viewing Notifications

### Inbox

The **Inbox** tab shows all notifications you have received. By default, only **unread** messages appear. Click **All** to view the full history.  
The **bell** icon in the top-right corner shows the unread count and provides quick access to recent notifications.

<MultiProductImageGallery product={props.product} images={[
  { src: '/src/assets/images/user-guide/notifications/notification-center-inbox-1.png', alt: 'Inbox tab — received notifications with mark as read and delete actions', caption: 'The <strong>Inbox</strong> tab showing unread notifications. ### Sent

The **Sent** tab lists all notifications you have sent and their delivery status. Click **Notify again** to resend, or the **delete** icon to remove.

<MultiProductImageGallery product={props.product} images={[
  { src: '/src/assets/images/user-guide/notifications/notification-center-sent-1.png', alt: 'Sent tab — notification list with Notify again and Delete actions', caption: 'The <strong>Sent</strong> tab showing a sent notification with its status. <Aside type="caution">
  Deleting a Web notification removes it for all recipients. Delivery issues (e.g., incorrect email address or phone number) appear in the **Status** column.
</Aside>
