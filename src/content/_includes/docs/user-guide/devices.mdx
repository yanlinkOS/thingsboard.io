**Devices** are the primary IoT entities in ThingsBoard. They represent physical or virtual things — sensors, controllers, gateways, emulators — that publish telemetry, report attributes, and respond to RPC commands.

## Managing Devices

### Create Device

<Steps>
1. Navigate to **Entities &#8702; Devices**.
2. Click **+ Add device** and select **Add new device**.
3. Enter a **Name** (must be unique within the tenant).
4. Optionally set a <DocLink product={props.product} path='user-guide/device-profiles' target="_blank">Device profile</DocLink> (defaults to `default`), **Label**, or **Description**.
5. If this device will proxy data for other devices, enable **Is gateway** (see **[Gateway Devices](#gateway-devices)**).
6. Click **Add**.
</Steps>

{props.product !== Products.CE && (
<Aside type="tip">
  The creation form also includes <strong>Owner</strong> (Tenant or a specific Customer) and <strong>Groups</strong> fields for organization and access control.
</Aside>
)}

A connectivity check dialog opens automatically — use it to verify the connection or close it to continue later.

### Edit Device

<Steps>
1. Click the **pencil** icon next to the device in the list.
2. Update the name, profile, label, firmware, or software version.
3. Click **Apply changes**.
</Steps>

### Delete Device

Click the **trash** icon next to the device and confirm, or open the device details page and click **Delete device**.

### Manage Credentials

<Steps>
1. Open the device details page.
2. Click **Manage credentials**.
3. Select the credential type and fill in the required fields.
4. Click **Save**.
</Steps>

<Aside type="tip">
  Credential types (Access Token, X.509 Certificate, MQTT Basic) depend on the transport protocol used by your device. See <DocLink product={props.product} path='user-guide/connectivity-guide'>How to Connect Your Device</DocLink> for a full list of supported protocols and their credential requirements.
</Aside>

## Gateway Devices

A gateway device proxies data for multiple downstream devices over a single connection using the <DocLink product={props.product} path='reference/gateway-api/overview' target="_blank">Gateway API</DocLink>. Enable gateway mode when creating or editing a device.

| Setting | Description |
|---------|-------------|
| **Is gateway** | Grants the device access to the gateway topic namespace for proxying downstream devices. |
| **Overwrite activity time** | When enabled, the gateway's own connection activity keeps connected downstream devices in **Active** state. Useful when downstream devices are connected through the gateway but not actively sending telemetry — RPC commands and shared attribute updates can still be delivered to them. |

To enable gateway mode, toggle **Is gateway** in the device creation form (step 4 of **[create device](#create-device)**) or in the device edit form.

## Common Operations

### Check Connectivity

Verify that the device can communicate with ThingsBoard:

<Steps>
1. Open the device details page and click **Check connectivity**.
2. Select the protocol (MQTT, HTTP, or CoAP) and your operating system.
3. Copy the generated command and run it in your terminal.
</Steps>

On success, the device status changes from **Inactive &#8702; Active** and telemetry appears under the **Latest telemetry** tab.

## Where Devices Are Used

- <DocLink product={props.product} path='user-guide/calculated-fields' target="_blank">Calculated Fields</DocLink> — compute derived values from device data
- <DocLink product={props.product} path='user-guide/digital-twins/relations' target="_blank">Relations</DocLink> — model hierarchies (e.g., Gateway → Contains → Sensor)
- <DocLink product={props.product} path='reference/rest-api' target="_blank">REST API</DocLink> — query, create, and manage devices programmatically
