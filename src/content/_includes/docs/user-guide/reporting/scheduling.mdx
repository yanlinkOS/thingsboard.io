The **Scheduler** automates report generation and delivery based on a pre-defined template. A single template can be scheduled multiple times — for different customers, time zones, or recipient groups — without duplication.

**Prerequisites:** At least one <DocLink product={props.product} path="user-guide/reporting/report-templates" bold={true} target="_blank">report template</DocLink> must exist before creating a scheduler event.

---

## Create a scheduled report

<Steps>
1. Navigate to **Reporting &#8702; Scheduling**.
2. Click **+ Scheduled report**.
3. Configure the event (see fields below).
4. Click **Add**.
</Steps>

### Scheduler event fields

| Field | Description |
|-------|-------------|
| **Name** | Unique name for this scheduler event |
| **Event type** | Always *Generate Report* |
| **Report template** | The template to use for generation |
| **User** | The user account whose data access permissions are applied when the report runs. The report includes only the data accessible to this user. |
| **Time zone** | Time zone used for interpreting the schedule and for relative time windows in the report |
| **Recipients** | One or more user groups or individual users. Configure delivery channels (Web, Email, Mobile app, Slack, SMS, Microsoft Teams) per recipient. |
| **Notification template** | The notification template used to wrap the report delivery message |
| **Start time** | When the first report generation should occur |
| **Repeat** | *Daily*, *Weekly*, or *Monthly* |

<Aside type="note">
  Web UI delivery is available to **tenant users only**.
</Aside>

---

## View generated reports

All reports produced by the scheduler are stored and accessible on the **Reports** tab.

<Steps>
1. Navigate to **Reporting &#8702; Reports**.
</Steps>

Available actions per report:
- Download the generated PDF or CSV file
- Remove the report from storage

You can also filter reports by **report template** or by the **user whose credentials were used to generate the report**.
