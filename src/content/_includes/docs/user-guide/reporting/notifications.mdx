A report can be attached to any notification template and delivered automatically when a system event occurs — such as when an alarm is created or cleared.

This is useful for combining platform notifications with analytical context: instead of receiving a simple notification (for example, an alarm), recipients receive an email with a PDF attachment summarizing the event and related data.

**Prerequisites:** Familiarity with <DocLink product={props.product} path="user-guide/reporting/getting-started" bold={true} target="_blank">reporting concepts</DocLink> and <DocLink product={props.product} path="user-guide/notifications" bold={true} target="_blank">notifications</DocLink>.

<Aside type="note">
  Receiving and viewing reports through the web interface is available only at the Tenant level.  
  Customer users can receive report notifications only via email or Slack.
</Aside>

---

## How It Works

When a notification rule fires, ThingsBoard:
1. Generates the specified report using the configured user account's permissions.
2. Attaches the resulting PDF (or CSV) to the outgoing notification.
3. Delivers the notification via the channels configured in the notification template.

The report is generated fresh each time the notification fires — it reflects data at the moment the event occurred.
