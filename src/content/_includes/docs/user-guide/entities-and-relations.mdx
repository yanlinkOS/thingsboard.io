## Entities Overview

ThingsBoard provides the user interface and REST APIs to provision and manage multiple entity types and their relations in your IoT application.
Supported entities are:

 - <strong><a href={docsLink(props.product, 'user-guide/ui/tenants/')}>Tenants</a></strong> - you can treat the tenant as a separate business-entity: it's an individual or an organization who owns or produce devices and assets;
 Tenant may have multiple tenant administrator users and millions of customers, devices and assets;
 - <strong><a href={docsLink(props.product, 'user-guide/customers')}>Customers</a></strong> - the customer is also a separate business-entity: individual or organization who purchase or uses tenant devices and/or assets;
 Customer may have multiple users and millions of devices and/or assets;
 - <strong><a href={docsLink(props.product, 'user-guide/users')}>Users</a></strong> - users are able to browse dashboards and manage entities;
 - <strong><a href={docsLink(props.product, 'user-guide/devices')}>Devices</a></strong> - basic IoT entities that may produce telemetry data and handle RPC commands. For example, sensors, actuators, switches;
 - <strong><a href={docsLink(props.product, 'user-guide/assets')}>Assets</a></strong> - abstract IoT entities that may be related to other devices and assets. For example factory, field, vehicle;
 - <strong><a href={docsLink(props.product, 'user-guide/entity-views/')}>Entity Views</a></strong> - useful if you like to share only part of device or asset data to the customers;
 - <strong><a href={docsLink(props.product, 'user-guide/alarms/')}>Alarms</a></strong> - events that identify issues with your assets, devices, or other entities;
 - <strong><a href={docsLink(props.product, 'user-guide/dashboards/')}>Dashboards</a></strong> - visualization of your IoT data and ability to control particular devices through the user interface;
 - **Rule Node** - processing units for incoming messages, entity lifecycle events, etc;
 - <strong>Rule Chain</strong> - defines the flow of the processing in the <a href={docsLink(props.product, 'user-guide/rule-engine')}>Rule Engine</a>. May contain many rule nodes and links to other rule chains;

Each entity supports:

 - <strong><a href={docsLink(props.product, 'user-guide/digital-twins/attributes/')}>Attributes</a></strong> - static and semi-static key-value pairs associated with entities. For example serial number, model, firmware version;
 - <strong><a href={docsLink(props.product, 'user-guide/digital-twins/time-series-data/')}>Time-series data</a></strong> - time-series data points available for storage, querying and visualization. For example temperature, humidity, battery level;
 - **[Relations](#relations)** - directed connections to other entities. For example contains, manages, owns, produces.

Some entities support profiles:

  - <strong><a href={docsLink(props.product, 'user-guide/multi-tenancy')}>Tenant Profiles</a></strong> - contains common settings for multiple tenants: entity, API and rate limits, etc. Each Tenant has the one and only profile at a single point in time.
  - <strong><a href={docsLink(props.product, 'user-guide/device-profiles/')}>Device Profiles</a></strong> - contains common settings for multiple devices: processing and transport configuration, etc. Each Device has the one and only profile at a single point in time.
  - <strong><a href={docsLink(props.product, 'user-guide/asset-profiles/')}>Asset Profiles</a></strong> - contains common settings for multiple assets: processing configuration, etc. Each Asset has the one and only profile at a single point in time.

## Relations

Entity relation defines connection between two ThingsBoard entities that belong to the same <a href={docsLink(props.product, 'user-guide/ui/tenants/')}>Tenant</a>.
The relation has an arbitrary type: Contains, Manages, Supports, etc. The relation is also directional.
You may treat ThingsBoard relations as a [Has-a](https://en.wikipedia.org/wiki/Has-a) relationship from object-oriented programming.

Relations help to model physical world objects in ThingsBoard. The easiest way to understand them is using the example.
Let's assume we want to build an application that collects data from soil moisture and temperature sensors, visualize this data on the dashboard, detect issues, raise alarms and control the irrigation.
Let's also assume we want to support multiple fields with hundreds of sensors. Fields may be also grouped into the Geo regions.
