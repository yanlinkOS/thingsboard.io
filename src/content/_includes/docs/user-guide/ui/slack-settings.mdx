Integrating Slack with ThingsBoard lets users receive <DocLink product={props.product === Products.EDGE_PE ? Products.PE : props.product === Products.EDGE ? Products.CE : props.product} path="user-guide/notifications">notifications</DocLink> in Slack channels or direct messages based on notification rules — for example, device status changes or detected issues.

<Aside type="note">
  You can send Slack notifications to users from **your Slack workspace only**.
</Aside>

## Create a Slack app and get the API token

<Steps>
1. Open the [Slack API — Getting a token](https://api.slack.com/tutorials/tracks/getting-a-token) tutorial page.

2. Scroll down to **Create a pre-configured app** and click **Create app**.

3. In the dialog, select your workspace from the dropdown and click **Next**.

4. Review the summary and click **Create** to create the app.

5. On the app configuration page, click **Got it**.

6. Click **Install to Workspace**, then click **Allow** to grant permissions.

7. Navigate to **OAuth & Permissions** in the left sidebar.

8. Copy the **Bot User OAuth Token** — this is the Slack API token you will use in ThingsBoard.
</Steps>

## Configure Slack settings in ThingsBoard

{props.product !== Products.CE ? (
  <p>Both system administrators and tenant administrators can configure Slack settings. Tenant-level settings override the system defaults.</p>

  <p>A system administrator configures Slack settings for the entire platform.</p>

<Steps>
1. Navigate to **Settings** and open the **Notifications** tab.

2. Paste the **Bot User OAuth Token** into the **Slack api token** field.

3. Click **Save**.
</Steps>

Once configured, ThingsBoard will deliver Slack notifications according to your <DocLink product={props.product === Products.EDGE_PE ? Products.PE : props.product === Products.EDGE ? Products.CE : props.product} path="user-guide/notifications">notification rules</DocLink>.

## Troubleshooting

If the Slack API token expires or is revoked, notification deliveries will fail. You can see delivery failures in the **Sent** tab of the <DocLink product={props.product === Products.EDGE_PE ? Products.PE : props.product === Products.EDGE ? Products.CE : props.product} path="user-guide/notifications">Notification center</DocLink>. To resolve, generate a new token from the Slack app's **OAuth & Permissions** page and update the settings in ThingsBoard.
