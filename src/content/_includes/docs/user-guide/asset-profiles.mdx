**Asset profiles** let you centrally define common settings for a group of similar assets — rule chain and queue. Instead of configuring each asset individually, you define the profile once and assign it to as many assets as needed.

## Managing Profiles

### Create a Profile

<Steps>
1. Navigate to **Profiles &#8702; Asset profiles**.
2. Click **+ Add asset profiles** and select **Create new asset profile**.
3. Enter a **Name** (the only required field).
4. Optionally configure the rule chain and queue.
5. Click **Add**.
</Steps>

### Edit a Profile

<Steps>
1. Click the profile name or the **pencil** icon.
2. Update the desired settings across any tab.
3. Click **Apply changes**.
</Steps>

### Delete a Profile

Click the **trash** icon next to the profile and confirm deletion.

## Profile Details

Clicking a profile opens a details panel with the following tabs:

| Tab | Contents |
|-----|---------|
| **Details** | Profile name, default dashboard, rule chain, and queue |
| **Calculated Fields** | Formulas for computing derived metrics from asset telemetry or attributes |
| **Alarm Rules** | Conditions, severity levels, and actions applied to all assets in this profile |
| **Audit Log** | History of user actions performed on this profile |
| **Version Control** | Git-based export and restore for backup and collaboration |

## Key Settings

### Default Rule Chain

By default, the Root Rule Chain processes all incoming messages and events for every asset. As your system grows and you introduce different asset types, the Root Rule Chain can become complex.

Assign a dedicated <DocLink product={props.product} path='user-guide/rule-engine' target="_blank">rule chain</DocLink> to an asset profile to isolate processing logic — attribute updates and lifecycle events (Created/Updated/Deleted) — for that asset type without overloading the Root Rule Chain.

### Queue

By default, all messages use the **Main** queue. Assign a custom queue to isolate high-priority assets from high-volume ones, or to apply different submit and processing strategies.

<Aside type="note">
  Custom queues must be configured by a System Administrator before they can be assigned to a profile.
</Aside>

### Alarm Rules

Define conditions that apply to all assets in this profile — for example, trigger a Critical alarm when a computed value exceeds a threshold. Rules defined here are shared across every asset using the profile. See <DocLink product={props.product} path='user-guide/alarm-rules' target="_blank">Alarm Rules</DocLink>.
