We’re excited to introduce **ThingsBoard 4.1** — another step toward an even more flexible, secure, and intelligent IoT platform. This release focuses on improving data usability, expanding analytical capabilities, and giving you greater control over internal processes.

Let’s take a closer look at what’s new in ThingsBoard 4.1 — and how it helps you build more powerful and reliable solutions.

## Automatic unit conversion

Forget about manual conversions and duplicate widgets! Starting with version 4.1, ThingsBoard automatically converts data between metric and imperial units. Whether it’s Celsius or Fahrenheit, kilometers or miles — set it once and it works everywhere.

- Each user can select their preferred unit format in their profile.
- Dashboards automatically respect user preferences during data visualization.
- If needed, specific units can still be locked manually per widget or dashboard.

**Important:** unit conversion only applies at the **visualization level**. Original values received from devices are preserved in their native format — no transformation occurs in storage. This ensures accurate aggregation, data integrity, and compatibility with external systems, while providing a more convenient experience for end users.
This feature makes ThingsBoard even more friendly for global teams and clients across regions — the same dashboard can now easily adapt to local preferences with just a few clicks.

## Secure Secrets Storage

Working with sensitive information — passwords, tokens, private keys, or certificates? The new **Secrets Storage** module in ThingsBoard 4.1 allows you to securely store and use such data directly within the platform, without ever exposing it or copying it manually.

- **Centralized management**: all secrets are managed via the unified **Security → Secrets Storage** interface.
- **Seamless integration**: create secrets in advance for reuse, or generate them directly while setting up Rule Engine nodes or integrations.
- **Platform-level encryption**: strong encryption safeguards your secrets at every stage — from storage to usage.
- **Safe sharing**: share credentials with your team or customers securely and without compliance risks.

**Currently**, secrets are supported in **Rule Engine** and **Integrations** only. 
📌 Future releases will expand support for Secrets Storage to more entity types. This will unify sensitive data management across the platform and further strengthen end-to-end security.
ThingsBoard is becoming not just a powerful IoT platform — but a **trustworthy environment** where security is built into the architecture by design.

## Historical data reprocessing with Calculated Fields

Calculated Fields have become even more powerful — they now support processing not only real-time values but also **historical data**.

- Formulas and scripts can apply logic to data collected over past periods, not just live telemetry.
- This is ideal for retrospective analytics, metric recalculations, and automatic adjustments to business logic as new data becomes available.

For example, you can recalculate anomalies in historical data after updating threshold settings.

![](/images/blog/thingsboard-4-1-release-unit-conversion-historical-data-processing-task-manager-secrets-storage-and-more/historical_data_processing_with_calculated_fields.webp)

## New “current owner” entity type in Calculated Fields

Calculated Fields just became even more context-aware. In ThingsBoard 4.1, a new entity type — **Current owner** — is available in argument settings.
This allows referencing telemetry from the **parent entity** that owns the current one. 
Device profile → owner of current entity → 
→device A →customer A;
→device B → customer B.
It enables rich, multi-level logic — like factoring in external environmental values when calculating metrics for an entity based on its owner data. No additional logic in rule chains: simplifies telemetry calculations without the need for complex rule chain configurations!

![](/images/blog/thingsboard-4-1-release-unit-conversion-historical-data-processing-task-manager-secrets-storage-and-more/new_current_owner_.webp)

## Long-running Task Manager

ThingsBoard 4.1 introduces the **Task Manager** — a centralized tool for tracking and managing background tasks from the UI. The first supported task is **Calculated Field Reprocessing**. When triggered, a main task is created and split into subtasks per entity.

Users can:

- monitor task status (`PENDING`, `RUNNING`, `COMPLETED`, `FAILED`, `CANCELLED`)
- track progress in real time
- cancel execution if needed
- reprocess only the failed subtasks

Task Manager makes working with large volumes of data safer and more transparent.  Currently available for Calculated Fields, with future support planned for additional task types.

![](/images/blog/thingsboard-4-1-release-unit-conversion-historical-data-processing-task-manager-secrets-storage-and-more/new_task_manager-4.webp)

## Full MQTT version support in Rule Engine and integrations

ThingsBoard 4.1 adds support for **all current versions of the MQTT protocol** — 3.1, 3.1.1, and 5.0 — in both **Rule Engine nodes** and **Integrations.**
You can now select the appropriate version depending on your use case — from simple device connections using 3.1, to advanced scenarios with MQTT 5.0 features like Reason Codes, Session Expiry, and User Properties. This ensures maximum compatibility and fine-tuned control over connection behavior.

![](/images/blog/thingsboard-4-1-release-unit-conversion-historical-data-processing-task-manager-secrets-storage-and-more/full_mqtt_version_support_in_rule_engine_and_integrations.webp)

## Data Converter library now accessible in the converter menu

Setting up integrations just got easier. In ThingsBoard 4.1, the **Converter Library** is now available directly in the **Data Converters** menu.
Previously, the library could only be accessed during integration creation. Now, you can create converters independently via the menu and reuse them across multiple integrations.
This is another step toward ThingsBoard’s **modular architecture**, where components are easy to scale, share, and centrally manage.

![](/images/blog/thingsboard-4-1-release-unit-conversion-historical-data-processing-task-manager-secrets-storage-and-more/data_converter_library_now_accessible_in_the_converter_menu.webp)

## Compatibility with the latest versions of key components

ThingsBoard 4.1 is fully compatible with the latest stable releases of key backend technologies:

- **Valkey 8.x** — open-source successor to Redis, better suited for clustering and scalability.
- **Cassandra 5.0** — improved performance and support for new query types.
- **Apache Kafka 4.0 (KRaft mode)** — no more ZooKeeper; now with embedded coordination for simplified deployment and increased resilience.

These upgrades let you leverage modern tools without sticking to legacy dependencies — resulting in better performance and scalability.

- EDQS (External Data Quality Service) improvements
- Calculated fields performance improvements
