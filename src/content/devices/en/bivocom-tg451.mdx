This guide covers connecting a **Bivocom TG451** IoT Gateway to ThingsBoard via MQTT. The gateway collects Modbus RTU sensor data and publishes it as telemetry to the platform.

## Prerequisites

- ThingsBoard instance (Cloud, CE, or PE)
- Bivocom TG451 gateway with a Modbus RTU slave device or [Mbslave](http://www.modbustools.com/mbslave.html) simulator for testing
- LAN access to the gateway WebUI

## Step 1. Provision device on ThingsBoard

<Steps>
1. Go to **Entities > Devices**, click **+** and select **Add new device**.

   <ImageGallery images={[
     { src: '/src/assets/images/partners/bivocom/provision-1-devices-page.png', alt: 'Navigate to the Devices page' },
   ]} />

2. Enter a device name (e.g., `TG451`), select the default profile, and click **Next: Credentials**.

   <ImageGallery images={[
     { src: '/src/assets/images/partners/bivocom/provision-2-add-device.png', alt: 'Add new device dialog' },
     { src: '/src/assets/images/partners/bivocom/provision-3-device-name.png', alt: 'Enter device name and select profile' },
   ]} />

3. Select **MQTT Basic**, enter a unique **Client ID**, **Username**, and **Password**. Click **Add**.

   <ImageGallery images={[
     { src: '/src/assets/images/partners/bivocom/provision-4-mqtt-credentials.png', alt: 'Configure MQTT Basic credentials' },
     { src: '/src/assets/images/partners/bivocom/provision-5-device-inactive.png', alt: 'Device added — status is Inactive' },
   ]} />
</Steps>

<Aside type="note">
Save the MQTT **Client ID**, **Username**, and **Password** — you will need them in Step 2.5.
</Aside>

## Step 2. Configure gateway

Log in to the gateway WebUI via its LAN IP address (default credentials: `admin` / `admin`).

<ImageGallery images={[
  { src: '/src/assets/images/partners/bivocom/gateway-1-login.png', alt: 'Gateway WebUI login page' },
]} />

### 2.1 Basic Setting

Go to **Data Collect > Basic Setting**. Enable **Data Collect**, configure **Collect Period** and **Report Period**, then click **Save & Apply**.

<ImageGallery images={[
  { src: '/src/assets/images/partners/bivocom/gateway-2-basic-setting.png', alt: 'Basic Setting — enable data collection' },
]} />

### 2.2 Interface Setting

Go to **Data Collect > Interface Setting**. Select the COM port connected to your Modbus slave (e.g., COM2 for RS232). Enable the port and set serial parameters to match your device (baud rate, data bits, stop bits, parity). Set **Frame Interval** to `200`, **Protocol** to `Modbus`, and **Command Interval** to `1`. Click **Save & Apply**.

<ImageGallery images={[
  { src: '/src/assets/images/partners/bivocom/gateway-3-interface-setting.png', alt: 'Interface Setting page' },
  { src: '/src/assets/images/partners/bivocom/gateway-4-interface-com.png', alt: 'COM port and serial parameters' },
]} />

### 2.3 Modbus Rules Setting

Go to **Data Collect > Modbus Rules Setting**. Click **+** to add a rule. Set **Order**, **Device Name**, and factor name. Select the report center and click **Add**.

<ImageGallery images={[
  { src: '/src/assets/images/partners/bivocom/gateway-5-modbus-rules.png', alt: 'Modbus Rules list' },
  { src: '/src/assets/images/partners/bivocom/gateway-6-modbus-rule-add.png', alt: 'Add Modbus rule dialog' },
]} />

Click **Edit** on the created rule to configure register details: slave ID, function code, start address, data type, unit, and optional operator (e.g., `+10` to offset the raw value).

<ImageGallery images={[
  { src: '/src/assets/images/partners/bivocom/gateway-7-modbus-rule-edit.png', alt: 'Edit Modbus rule — register and operator configuration' },
]} />

### 2.4 IO Setting

Go to **Data Collect > IO Setting**. Click **+** to add DI channels. Select the channel, set a factor name, **Status Mode**, and the report center. Click **Add**.

<ImageGallery images={[
  { src: '/src/assets/images/partners/bivocom/gateway-8-io-setting.png', alt: 'IO Setting page' },
  { src: '/src/assets/images/partners/bivocom/gateway-9-io-channels.png', alt: 'DI channels configured' },
]} />

### 2.5 Server Setting

Go to **Data Collect > Server Setting**. Enable the server and fill in the fields:

| Field | Value |
|---|---|
| Protocol | `MQTT` |
| Encapsulation Type | `LUA` |
| Server Address | `mqtt.thingsboard.cloud` |
| Port | `1883` |
| Publish Topic | `v1/devices/me/telemetry` |
| Username | *(from Step 1)* |
| Password | *(from Step 1)* |
| Client ID | *(from Step 1)* |

<ImageGallery images={[
  { src: '/src/assets/images/partners/bivocom/gateway-10-server-setting.png', alt: 'Server Setting — enable server' },
  { src: '/src/assets/images/partners/bivocom/gateway-11-server-mqtt.png', alt: 'MQTT server address and port' },
  { src: '/src/assets/images/partners/bivocom/gateway-12-server-params.png', alt: 'MQTT topic and credentials' },
]} />

Click **Save & Apply**. The **Connection Status** changes to `CONNECTED`.

<ImageGallery images={[
  { src: '/src/assets/images/partners/bivocom/gateway-13-server-connected.png', alt: 'Server status: CONNECTED' },
]} />

<Aside type="caution">
The Bivocom gateway requires **LUA** encapsulation to format data correctly for ThingsBoard. Standard JSON encapsulation is not compatible.
</Aside>

## Step 3. Verify connection

On the ThingsBoard device page, click **Check connectivity**.

<ImageGallery images={[
  { src: '/src/assets/images/partners/bivocom/check-1-connectivity-button.png', alt: 'Check connectivity button on device details' },
]} />

Select **Linux** (the gateway runs OpenWrt), then run the displayed command to test MQTT connectivity.

<ImageGallery images={[
  { src: '/src/assets/images/partners/bivocom/check-2-select-os.png', alt: 'Select Linux OS and run MQTT test command' },
]} />

The device status changes to **Active** once the connection is established.

<ImageGallery images={[
  { src: '/src/assets/images/partners/bivocom/check-3-device-active.png', alt: 'Device status changed to Active' },
]} />

## Step 4. Verify data transmission

In the gateway WebUI, go to **Data Collect > Data Query** to confirm real-time Modbus data collection.

<ImageGallery images={[
  { src: '/src/assets/images/partners/bivocom/data-1-gateway-query.png', alt: 'Gateway data query — real-time Modbus values' },
]} />

On the ThingsBoard device page, click **Latest telemetry** to confirm incoming data.

<ImageGallery images={[
  { src: '/src/assets/images/partners/bivocom/data-2-latest-telemetry.png', alt: 'ThingsBoard latest telemetry from gateway' },
]} />

Modify values in Mbslave (or through the connected sensor) and confirm the telemetry updates on ThingsBoard.

<ImageGallery images={[
  { src: '/src/assets/images/partners/bivocom/data-3-telemetry-updated.png', alt: 'Telemetry updated after data change' },
]} />
