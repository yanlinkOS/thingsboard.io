## Introduction

<a href="https://www.milesight-iot.com/lorawan/sensor/ws202/" target="_blank" rel="noopener noreferrer">WS202 PIR & Light Sensor</a> is a PIR sensor based on passive infrared technology to detect a motion or occupancy.
WS202 can detect whether there is a movement within the range of 6-8 m.
Besides, WS202 equips with a light sensor which can link PIR detection results to trigger scenes.
WS202 can be widely used in smart homes, smart offices, schools, warehouses, etc.
Sensor data are transmitted in real-time using the standard LoRaWAN® protocol.
LoRaWAN® enables encrypted radio transmissions over long distances while consuming very little power.

<Banner variant="peFeature">This feature is available only for **Platform Integrations**.</Banner>

## Prerequisites

To continue with this guide we will need the following:

- <a href="https://www.milesight-iot.com/lorawan/sensor/ws202/" target="_blank">WS202 PIR & Light LoRaWAN® Sensor </a>
- <a href="https://resource.milesight.com/milesight/iot/document/ws202-user-guide-en.pdf" target="_blank" rel="noopener noreferrer">WS202 PIR & Light Sensor user manual</a>
- Smartphone with NFC-enabled and Milesight ToolBox application (<a href="https://play.google.com/store/apps/details?id=com.ursalinknfc" target="_blank" rel="noopener noreferrer">Android</a>/<a href="https://itunes.apple.com/app/id1518748039" target="_blank" rel="noopener noreferrer">iOS</a>)
- LoRaWAN® gateway (in our case <a href="/device-library/ug56-lorawan-gateway/?platform=pe" target="_blank" rel="noopener noreferrer">UG56 LoRaWAN® Gateway</a>)
- Configured integration on networks server and ThingsBoard
- [Network Server account](#device-connection)

- <a href="https://localhost" target="_blank" rel="noopener noreferrer">ThingsBoard account</a>

## Device connection

According to the <a href="https://resource.milesight.com/milesight/iot/document/ws202-user-guide-en.pdf" target="_blank" rel="noopener noreferrer">official user manual</a>, we need a smartphone with NFC enabled and the ToolBox application to connect the sensor.
Since this device can only be operated using a LoRaWAN® gateway, we must first connect it to a network server that has an integration configured with ThingsBoard.
Afterward, it can be provisioned to ThingsBoard.

### Device configuration

To connect and send data we should configure the device and network server.
At first we are going to configure the device, and save required information for network server configuration.
To add a device to network server and get information from it, we will need the following device parameters:
 - **Device EUI** - device identifier
 - **Application EUI** - Application identifier
 - **Application Key** - Application key to identify device. We recommend to use a generated key, not from the example!

The parameters above are required for connection.

Depending on the network server, you may also need to provide join type (OTAA), LoRaWAN version.

To configure device via NFC, you will need to hold your smartphone like on the picture below:

<DocImage src="/src/assets/devices-library/ready-to-go-devices/ws202-pir-and-light-sensor/nfc-area.png" alt="NFC zone" />

<br />

To read and write configuration on the device you may follow next steps on your smartphone:

<Steps>

1. Open **ToolBox** application.
2. Click on **NFC Read** button and hold your smartphone near the device.
3. Go to tab **Setting**, set and save required fields and any other configuration that you need.
4. Press **Write** button and hold your smartphone near the device.

</Steps>

## Introduction

<a href="https://www.milesight-iot.com/lorawan/sensor/ws202/" target="_blank" rel="noopener noreferrer">WS202 PIR & Light Sensor</a> is a PIR sensor based on passive infrared technology to detect a motion or occupancy.
WS202 can detect whether there is a movement within the range of 6-8 m.
Besides, WS202 equips with a light sensor which can link PIR detection results to trigger scenes.
WS202 can be widely used in smart homes, smart offices, schools, warehouses, etc.
Sensor data are transmitted in real-time using the standard LoRaWAN® protocol.
LoRaWAN® enables encrypted radio transmissions over long distances while consuming very little power.
