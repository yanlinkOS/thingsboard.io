<a href="https://www.elastel.com/products/industrial-raspberry-pi/eg500-edge-computing-gateway/" target="_blank" rel="noopener noreferrer">Elastel EG500</a> is an Industrial Edge Computing Gateway powered by **Raspberry Pi CM4**.

Perfect for use as an IoT Gateway, data acquisition device, Industrial Controller, or for Edge Computing applications.

Designed for IIoT applications, it features:
- **Industrial interfaces**: 2× GbE, 2× USB, 1× RS485, 1× RS232, 6× DI, 6× DO, and 3× ADC
- **Connectivity options**: 4G, LoRa, Wi-Fi/BLE, WiFi HaLow (802.11ah WiFi)
- **Enclosure**: Truly IPC-grade stability and ruggedness

In this tutorial we will use Elastel EG500 as edge gateway to collect data from RS485 Modbus temperature & humidity sensor, also collect the DI status value and control the DO on/off which wiring a LED light. EG500 will connect to ThingsBoard for data visualizing and controlling the LED.

## Prerequisites

To continue with this guide, you will need to have access to ThingsBoard Professional Edition. The easiest way is to use <a href="/installations/choose-region/" target="_blank" rel="noopener noreferrer">ThingsBoard Cloud</a> server.
The alternative option is to install ThingsBoard using <a href="/docs/pe/installation/" target="_blank" rel="noopener noreferrer">installation guide</a>.

Also, you will require the following items:

- An **Elastel EG500** device
- An **RS485 Modbus Temperature & Humidity sensor**
- A **LED light**

## Create device on ThingsBoard

**1. In our example, we use ThingsBoard Cloud**

Create an account and then <a href="https://thingsboard.cloud/login" target="_blank" rel="noopener noreferrer">log in</a>.

<DocImage src="/src/assets/devices-library/basic/single-board-computers/raspberry-pi-cm4/thingsboard_login.png" alt="thingsboard_login" />

<br />**2. Create a device on ThingsBoard**

Go to the "**Devices**" page of the "**Entities**" section, click on the "**+**" icon button in the top right corner of the table and then select "**Add new device**" from drop-down menu.

<DocImage src="/src/assets/devices-library/basic/single-board-computers/raspberry-pi-cm4/add_new_device.png" alt="add_new_device" />

<br />**3.** Enter a device name (e.g., **Elastel_EG500**) and click "**Add**".

<br />**4. Copy and save the Access token**

It will be needed in further device configuration for connecting to ThingsBoard cloud.

<DocImage src="/src/assets/devices-library/basic/single-board-computers/raspberry-pi-cm4/access_token.png" alt="access token" />

## Elastel device configuration

In this step, we will configure Elastel EG500 to collect data from RS485 Modbus sensor and DI/DO status value. Elastel devices has prebuilt <a href="https://www.elastel.com/iiot-gateway-software-elastpro/" target="_blank" rel="noopener noreferrer">Elastel IIoT gateway software ElastPro</a> which provide web-based programs for easy configuration and data collect setup.

**1. Launch ElastPro**

To launch ElastPro, follow <a href="https://docs.elastel.com/docs/ElastPro/Getting_Started/" target="_blank" rel="noopener noreferrer">this guide</a>.

**2. Enable Data Collect**

Go to the **Data Collect** -> **Basic**, enable the **Data collect** function, and configure the **Collect period**, **Report period**, then click the "**Apply settings**" button.

<DocImage src="/src/assets/devices-library/basic/single-board-computers/raspberry-pi-cm4/basic_setting.png" alt="enable data collect" />

<br />**3. Configure the RS485 Serial port for Modbus sensor data collect**

Go to the **Data Collect** -> **Interfaces** to enable the COM1 RS485 for connecting the RS485 **Temp&Humidity Sensor**, configure the related parameters, then "**Apply settings**".

<DocImage src="/src/assets/devices-library/basic/single-board-computers/raspberry-pi-cm4/rs485_interface.png" alt="rs485 interface setting" />

<br />Go to **Data Collect** -> **South Devices** -> **Modbus Rules** to configure the collect Modbus data from **Temp&Humidity Sensor**, then "**Apply settings**".

<DocImage src="/src/assets/devices-library/basic/single-board-computers/raspberry-pi-cm4/modbus_rule_settings.png" alt="Modbus rule settings" />

<br />**4. Configure the IO port for collect DI status and control DO status**

Go to **Data Collect** -> **South Devices** -> **IO**, add DI and DO Setting rules, configure the relevant parameters and click the "**Apply settings**".

<DocImage src="/src/assets/devices-library/basic/single-board-computers/raspberry-pi-cm4/io_settings.png" alt="IO settings" />

## Connect device to ThingsBoard

Go to the **Data Collect** -> **North Apps** -> **Reporting Center** to configure the connection to ThingsBoard platform via MQTT.

<DocImage src="/src/assets/devices-library/basic/single-board-computers/raspberry-pi-cm4/mqtt_thingsboard_connect.png" alt="Connect mqtt to ThingsBoard" />

> Note: If using custom MQTT credentials on ThingsBoard, set them under **Next: Credentials → MQTT Basic** during device creation, and use those here.

Refer to the <a href="https://docs.elastel.com/docs/ElastPro/intro/" target="_blank" rel="noopener noreferrer">**ElastPro User Manual**</a> for more detailed configuration options.

### Create Dashboard

Now let&#39;s display the data from the Elastel EG500 device on the dashboard. Learn more how to work with the dashboard <a href="/docs/pe/user-guide/dashboards/" target="_blank" rel="noopener noreferrer">here<span class="sr-only"> about dashboard</span></a>.

Go to the "**Dashboards**" page, click on the "**+**" icon button in the top right corner of the table and then select "**Create new dashboard**" from drop-down menu. Enter the Title of the dashboard, then click "**Add**".

<DocImage src="/src/assets/devices-library/basic/single-board-computers/raspberry-pi-cm4/add_dashboard.png" alt="add dashboard" />

<br /> On the Dashboard edit page, click the "**+ Add new widget**" button to add <a href="/docs/pe/user-guide/widgets/" target="_blank" rel="noopener noreferrer">widget</a> for displaying our telemetry data.

<DocImage src="/src/assets/devices-library/basic/single-board-computers/raspberry-pi-cm4/add_widget.png" alt="add widget" />

<br />Select the appropriate widget template and configure the data source accordingly. For example, choose a **Value card** to display the "Temperature" data and add the "Temperature" data key as a data source from the EG500 device we created.

<DocImage src="/src/assets/devices-library/basic/single-board-computers/raspberry-pi-cm4/add_temp_widget.png" alt="add temperature widget" />

<br />After completing the widget setup, click the "**Add**" button. We have now added a widget that displays the real-time temperature value.

<DocImage src="/src/assets/devices-library/basic/single-board-computers/raspberry-pi-cm4/temperature_widget.png" alt="temperature widget" />

<br />Similarly, add widgets to display **humidity** and **door state** data, and add a "**Power button**" widget to control turning on/off the indicator light.
After completing the dashboard configuration, click the "**Save**" button.

Now we have a visualizing dashboard for displaying data and controlling the DO (LED light).

<DocImage src="/src/assets/devices-library/basic/single-board-computers/raspberry-pi-cm4/other_widget.png" alt="add other widget" />
